Por que evitar tudo na Activity?

MainActivity.kt

class MainActivity : AppCompatActivity() {

// ERRO 1: Dados hardcoded na View

val frases = listOf("Bom dia", "Foco!")


override fun onCreate(saved: Bundle?) {

super.onCreate(saved)

setContentView(R.layout.activity_main)


// ERRO 2: Lógica de negócio misturada com UI

botao.setOnClickListener {

val aleatorio = frases.random()

textoView.text = aleatorio

}

}

}

Ciclo de Vida (Rotação)

Ao girar a tela, a Activity é destruída e recriada. Resultado: A frase atual seria perdida e voltaria ao estado inicial.

Difícil de Testar

Não tem como testar a lógica de sortear frases sem subir um emulador Android, pois está "colada" no botão.

God Activity

Se o app crescer, esse arquivo terá 2000 linhas, misturando Banco de Dados, UI, Rede e Lógica. Impossível de manter.
