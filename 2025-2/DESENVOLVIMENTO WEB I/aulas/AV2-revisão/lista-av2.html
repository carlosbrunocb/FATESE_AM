Layout CSS

1. Compare Flexbox e CSS Grid. Em que cenário você escolheria usar um em detrimento do outro?

Ver Resposta Esperada

Flexbox é um sistema unidimensional (linha OU coluna). É ideal para alinhar itens em uma barra de navegação, centralizar elementos ou distribuir espaço em componentes menores.

CSS Grid é bidimensional (linhas E colunas). É ideal para definir o layout estrutural da página inteira (header, sidebar, content, footer) ou galerias complexas.

Conclusão: Use Grid para a "macro" estrutura da página e Flexbox para o alinhamento "micro" dentro dos componentes.

Web Storage

2. Explique a utilidade da Web Storage API e diferencie LocalStorage de SessionStorage, citando um exemplo de uso para cada.

Ver Resposta Esperada

A Web Storage API permite armazenar dados (pares chave-valor) no navegador do cliente de forma mais eficiente que cookies. Ela não envia dados ao servidor a cada requisição.

  • LocalStorage: Persiste mesmo após fechar o navegador. Exemplo: Salvar a preferência de tema (Dark Mode) do usuário.
  • SessionStorage: Dados duram apenas enquanto a aba estiver aberta. Exemplo: Salvar o estado de um formulário multipáginas temporariamente ou filtro de busca atual.
Assincronismo

3. Descreva o funcionamento da Fetch API com async/await. Por que o tratamento de erros (try/catch) é importante nesse contexto?

Ver Resposta Esperada

A Fetch API é usada para fazer requisições de rede. Ao usar async/await, o código parece síncrono, tornando-o mais legível. A função deve ser marcada como async, e usamos await antes do fetch() para esperar a Promise resolver.

O bloco try/catch é crucial porque requisições de rede podem falhar (servidor fora do ar, sem internet). O try envolve a tentativa de conexão, e o catch captura e trata esses erros, evitando que a aplicação quebre e permitindo feedback ao usuário.

Manipulação DOM

4. Explique o que é o DOM e descreva como o JavaScript pode ser usado para manipular o conteúdo e o estilo de um elemento HTML (dê exemplos de propriedades).

Ver Resposta Esperada

O DOM (Document Object Model) é uma representação em árvore da estrutura HTML da página, servindo como interface entre o HTML e o JavaScript.

Para manipular, primeiro selecionamos o elemento (ex: document.getElementById). Depois, podemos usar:

  • innerHTML ou textContent para alterar o texto/conteúdo.
  • style (ex: elemento.style.color = 'red') para alterar CSS inline.
  • classList (ex: classList.add('ativo')) para adicionar classes CSS dinamicamente.
Boas Práticas

5. Por que o uso de var é desencorajado no JavaScript moderno em favor de let e const? Relacione com o conceito de escopo.

Ver Resposta Esperada

O var tem escopo de função (ou global) e sofre hoisting (é elevado ao topo), o que pode causar comportamentos imprevisíveis e vazamento de variáveis.

Já let e const (introduzidos no ES6) têm escopo de bloco (respeitam as chaves {} de um if ou for). Isso torna o código mais seguro e previsível. const ainda garante que a referência da variável não seja reatribuída, promovendo imutabilidade onde necessário.
