Guia do Projeto: Calculadora de Impostos CLT vs. PJ

Informações:

  1. Data de entrega: 15/11/2025.
    • Entrega da implementação e apresentação do trabalho.
  2. Trabalho em equipe: 3 a 2 membros por equipe.
    • Entregar uma calculadora de Imposto de Renda por equipe.

Objetivo do Projeto

Desenvolver um aplicativo web front-end (HTML/CSS/JS) que calcula e compara os encargos e impostos de um trabalhador CLT (Consolidação das Leis do Trabalho) e de um prestador de serviço PJ (Pessoa Jurídica). O objetivo é que o aluno aplique seus conhecimentos de JavaScript para manipular o DOM e implementar lógicas de negócio complexas, como tabelas progressivas de impostos.

Funcionalidades Requeridas

  1. Seleção de Modo: O usuário deve poder escolher claramente se deseja calcular os impostos para CLT ou para PJ.
  2. Formulário CLT:
    • Entradas: Salário Bruto Mensal, Número de Dependentes.
    • Saídas (Relatório): Valor do FGTS (pago pela empresa), Valor do INSS (descontado), Valor do IRRF (descontado), Salário Líquido.
  3. Formulário PJ:
    • Seleção de Regime: O usuário deve escolher entre "Simples Nacional" e "Lucro Presumido".
    • Entradas (Simples Nacional): Faturamento Bruto Mensal.
    • Entradas (Lucro Presumido): Faturamento Bruto Mensal (para este exercício, faremos um cálculo mensal simplificado).
    • Saídas (Relatório): Total de Impostos Pagos (pela empresa), Lucro Líquido (Faturamento - Impostos).
  4. Geração de Relatório: A página deve exibir os resultados de forma clara e organizada após o cálculo.

Orientações para o Aluno: O que Implementar

Este projeto é uma excelente oportunidade para organizar seu código em três partes claras: a estrutura (HTML), a apresentação (CSS) e a lógica (JavaScript).

1. O Front-End (HTML & CSS)

Seu HTML será a espinha dorsal do aplicativo. Pense em como estruturar a página para guiar o usuário.

Sugestão de Estrutura HTML

Você pode dividir sua página principal (index.html) nestas seções:

  • Cabeçalho: Com o título da aplicação.
  • Seleção de Modo: 
    <div id="seletor-modo">
    <button id="btn-modo-clt">Calcular para CLT</button>
    <button id="btn-modo-pj">Calcular para PJ</button>
</div>
  • Formulários (Invisíveis no Início):

    Crie três formulários ou seções (divs) separadas. Um para CLT, um para Simples Nacional e um para Lucro Presumido. Inicialmente, todos eles devem estar ocultos. 

    <div id="form-clt" style="display: none;">
    <label for="salario-bruto">Salário Bruto:</label>
    <input type="number" id="salario-bruto">
    <!-- ... outros inputs do CLT ... -->
    <button id="calcular-clt">Calcular</button>
</div>

<div id="form-pj" style="display: none;">
    <label for="regime-pj">Selecione o Regime:</label>
    <select id="regime-pj">
        <option value="simples">Simples Nacional</option>
        <option value="presumido">Lucro Presumido</option>
    </select>
    <!-- ... inputs de faturamento etc. ... -->
    <button id="calcular-pj">Calcular</button>
</div>
  • Área de Resultado:

    Um local dedicado para mostrar o relatório final. Comece com ele vazio.

    <div id="relatorio">
    <h3>Seu Relatório de Encargos</h3>
    <!-- Os resultados serão inseridos aqui via JavaScript -->
</div>

Estilização (CSS): Use o CSS para tornar a interface limpa. A principal propriedade que você usará em conjunto com o JavaScript é display: none; (para ocultar os formulários) e display: block; (para mostrá-los).

2. O "Back-End" (Lógica em JavaScript)

Todo o "cérebro" da sua aplicação estará no JavaScript. Seu script precisará lidar com eventos, coletar dados dos formulários, executar os cálculos e, finalmente, exibir os resultados.

A. Manipulação de Eventos (O Início)

Você precisará de addEventListener para gerenciar a interface:

  • Eventos nos Botões de Modo (CLT/PJ): Quando o usuário clicar em "Calcular para CLT", você deve mostrar o #form-clt e esconder o #form-pj. O inverso também é verdadeiro.
  • Eventos nos Botões de Cálculo: Quando o usuário clicar em "Calcular", seu script deve:
    1. Ler os valores dos inputs (ex: document.getElementById('salario-bruto').value).
    2. Importante: Converter os valores de texto para números (parseFloat()).
    3. Chamar as funções de cálculo que você criará.
    4. Exibir os resultados no #relatorio.

B. Funções de Cálculo (O Núcleo)

É aqui que a lógica do negócio acontece. Organize seu código em funções claras e reutilizáveis.

Desafio: Lógica de Cálculo CLT

Esta é a parte mais complexa. O cálculo é sequencial: você precisa do resultado de um para fazer o próximo.

  1. Função calcularFGTS(salario): Simples. O FGTS é 8% do salário bruto. (Lembre-se: não é descontado, é pago pela empresa).
  2. Função calcularINSS(salario): Complexo. Você deve implementar a lógica da tabela progressiva. O cálculo é feito "faixa por faixa".
  3. Função calcularIRRF(salario, inss, dependentes): Complexo. Primeiro, ache a "Base de Cálculo": Base = Salário Bruto - INSS - (Dependentes * Valor_Por_Dependente). Depois, aplique a tabela progressiva do IRRF sobre essa base.
  4. Função calcularSalarioLiquido(salario, inss, irrf): Simples. Líquido = Salário Bruto - INSS - IRRF.

Desafio: Lógica de Cálculo PJ (Simplificada)

Para este exercício, usaremos regras simplificadas.

  • Função calcularSimplesNacional(faturamento):
    • Assuma a alíquota inicial de 6% (para atividades de serviço comuns, Anexo III).
    • Imposto Pago = Faturamento * 0.06
    • Lucro Líquido = Faturamento - Imposto Pago
  • Função calcularLucroPresumido(faturamento):
    • Este regime paga impostos separados. Vamos simplificar as alíquotas:
    • PIS: 0,65% sobre o faturamento.
    • COFINS: 3% sobre o faturamento.
    • ISS: 5% sobre o faturamento (pode variar, mas use 5%).
    • IRPJ: 15% sobre uma "presunção de lucro" de 32% do faturamento. (Cálculo: Faturamento * 0.32 * 0.15).
    • CSLL: 9% sobre uma "presunção de lucro" de 32% do faturamento. (Cálculo: Faturamento * 0.32 * 0.09).
    • Imposto Total = PIS + COFINS + ISS + IRPJ + CSLL
    • Lucro Líquido = Faturamento - Imposto Total

C. Exibição dos Resultados (DOM)

Após ter os resultados (provavelmente em um objeto { }), você deve formatá-los como HTML e inseri-los na div de relatório.

// Exemplo de como exibir o resultado no DOM
const relatorioDiv = document.getElementById('relatorio');
const salario = 5000;
const inss = 500;
const irrf = 300;

// Você pode construir uma string HTML
const resultadoHTML = `
    <h4>Resultado do Cálculo CLT</h4>
    <p>Salário Bruto: R$ ${salario.toFixed(2)}</p>
    <p style="color: red;">Desconto INSS: R$ ${inss.toFixed(2)}</p>
    <p style="color: red;">Desconto IRRF: R$ ${irrf.toFixed(2)}</p>
    <p style="color: green; font-weight: bold;">Salário Líquido: R$ ${(salario - inss - irrf).toFixed(2)}</p>
`;

// E então injetá-la na página
relatorioDiv.innerHTML = resultadoHTML;

Dica: Para formatar os números como moeda (R$), pesquise sobre valor.toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' }).

Elucidação: O Cálculo Progressivo (INSS/IRRF)

Não mostre a implementação final, mas esta é a parte que os alunos mais têm dificuldade. O cálculo não é apenas encontrar a faixa e aplicar a alíquota.

Exemplo de Lógica (Não é a Solução Completa):

Imagine uma tabela simples de imposto com 2 faixas:

  • Até R$ 1000: 10%
  • Acima de R$ 1000: 20%

Se o salário for R$ 1500, o cálculo ERRADO é: R$ 1500 * 20% = R$ 300.

O cálculo CORRETO (progressivo) é:

  1. R$ 1000 (o teto da Faixa 1) * 10% = R$ 100
  2. R$ 500 (o que sobrou do salário: R$ 1500 - R$ 1000) * 20% = R$ 100
  3. Total do Imposto: R$ 100 + R$ 100 = R$ 200

Seu código JavaScript deverá implementar essa lógica de "fatiar" o salário e aplicar a alíquota correta para cada fatia, tanto para o INSS quanto para o IRRF (na base de cálculo dele).
