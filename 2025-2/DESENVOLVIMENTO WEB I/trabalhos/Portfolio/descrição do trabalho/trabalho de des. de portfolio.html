Objetivo do Projeto

Este projeto final tem como objetivo consolidar todo o conhecimento adquirido. Vocês irão pegar o "Portfólio Responsivo" (Projeto 1, focado em HTML/CSS) e elevá-lo a um nível profissional, injetando interatividade com JavaScript.

A principal tarefa é integrar seus dois projetos de calculadora (CLT vs. PJ e Calculadora RPN) dentro do seu portfólio, de uma forma que demonstre suas novas habilidades em lógica e manipulação do DOM. Além disso, o próprio portfólio ganhará vida com funcionalidades que só o JavaScript pode oferecer.

Funcionalidades Requeridas (Resumo)

  1. Reformular a Seção "Portfólio" para incluir seus dois projetos de calculadora.
  2. Implementar um Modal (Janela) em JavaScript puro para exibir os detalhes de cada projeto (sem sair da página).
  3. Implementar um Menu "Hambúrguer" funcional para dispositivos móveis.
  4. Implementar um "Toggle" de Dark/Light Mode que salva a preferência do usuário.
  5. (Obrigatório) Envio de Formulário de Contato com AJAX (Fetch API), para que a página não precise recarregar ao enviar uma mensagem.
  6. (Bônus) Filtros de Projeto: Botões para filtrar sua galeria (ex: "Todos", "Apps com JS", "Landing Pages").
  7. (Bônus) Animações de Scroll: Elementos que aparecem suavemente conforme o usuário rola a página.

Seção "Portfólio" (#portfolio)

Sua galeria de projetos (provavelmente um CSS Grid) precisa de uma pequena mudança. Cada "card" de projeto será um gatilho para o JavaScript.

Desafio: A Nova Galeria de Projetos e o Modal

1. A Estrutura dos Cards (HTML)

Seus cards de projeto agora precisam "guardar" informações que o JavaScript possa ler. A melhor forma de fazer isso é com data-attributes.

<!-- Exemplo de um card na sua galeria grid -->
<div class="projeto-card" 
     data-titulo="Calculadora CLT vs. PJ" 
     data-imagem="img/preview-calculadora-clt.jpg"
     data-descricao="Uma aplicação web para comparar custos de contratação CLT e PJ, implementando as tabelas progressivas de INSS e IRRF."
     data-tecnologias="HTML, CSS, JavaScript (DOM, Lógica de Negócio)"
     data-link-projeto="caminho/para/calculadora-impostos.html"
     data-link-github="https://github.com/seu-usuario/repo-calculadora">
     
    <img src="img/preview-calculadora-clt.jpg" alt="Preview da Calculadora de Impostos">
    <h4>Calculadora CLT vs. PJ</h4>
</div>

<!-- Repita para a Calculadora RPN e outros projetos -->

2. A Estrutura do Modal (HTML)

Em algum lugar do seu index.html (geralmente no final, antes de fechar o <body>), crie a "casca" do seu modal. Ele deve começar escondido com CSS (display: none ou visibility: hidden).

<!-- O Modal (Começa escondido) -->
<div id="modal-projeto" class="modal-overlay">
    <div class="modal-content">
        <button id="modal-close-btn" class="modal-close">×</button>
        <img id="modal-imagem" src="" alt="Preview do Projeto">
        <h3 id="modal-titulo"></h3>
        <p id="modal-descricao"></p>
        <p><strong>Tecnologias:</strong> <span id="modal-tecnologias"></span></p>
        <a id="modal-link-projeto" href="#" target="_blank" class="btn">Ver Projeto</a>
        <a id="modal-link-github" href="#" target="_blank" class="btn-outline">Ver Código</a>
    </div>
</div>

3. Estilização (CSS)

Você precisará de CSS para fazer o modal funcionar. Lembre-se das técnicas de posicionamento:

  • .modal-overlay: Deve cobrir a tela inteira (position: fixed, top: 0, left: 0, width: 100%, height: 100%). Use um fundo semi-transparente (ex: background: rgba(0, 0, 0, 0.7)).
  • .modal-content: Deve ficar centralizado sobre o overlay. Use Flexbox ou Grid para centralizar (display: flex, align-items: center, justify-content: center no overlay) ou position: absolute com top: 50%, left: 50%, transform: translate(-50%, -50%).
  • Crie uma classe .visivel (ou .ativo) que muda o display ou visibility do modal para que ele apareça. O JavaScript irá adicionar/remover esta classe.

Parte 2: O "Back-End" (Lógica em JavaScript)

Aqui é onde seu portfólio ganha vida. Organize seu script.js por funcionalidades.

Funcionalidade 1: Lógica do Modal de Projetos

Este script deve "ouvir" cliques nos cards de projeto e usar as informações dos data-attributes para preencher e exibir o modal.

Elucidação: A Lógica do Modal (Pseudo-código)

Você não deve criar um modal para cada projeto. Você tem um modal (a "casca") e o JavaScript o preenche dinamicamente.

// 1. Pegue as referências do DOM
const cards = document.querySelectorAll('.projeto-card');
const modal = document.getElementById('modal-projeto');
const btnFechar = document.getElementById('modal-close-btn');

// 2. Adicione um "ouvinte" de clique em CADA card
cards.forEach(card => {
    card.addEventListener('click', () => {
        // 3. Pegue os dados do card clicado
        const titulo = card.getAttribute('data-titulo');
        const descricao = card.getAttribute('data-descricao');
        // ... pegue todos os outros data-attributes ...

        // 4. Preencha o HTML do modal com esses dados
        document.getElementById('modal-titulo').textContent = titulo;
        document.getElementById('modal-descricao').textContent = descricao;
        // ... preencha a imagem, links, etc. ...

        // 5. Mostre o modal
        modal.classList.add('visivel');
    });
});

// 6. Adicione o evento para fechar o modal
btnFechar.addEventListener('click', () => {
    modal.classList.remove('visivel');
});

// Dica extra: Feche o modal se o usuário clicar no overlay (fundo)
modal.addEventListener('click', (event) => {
    if (event.target === modal) { // Verifica se o clique foi no fundo
        modal.classList.remove('visivel');
    }
});

Funcionalidade 2: Menu Hambúrguer

  • HTML: Adicione um botão <button id="btn-mobile"> no seu <header> (ele pode ser visível apenas em telas pequenas via Media Query).
  • JS: Adicione um evento de clique no #btn-mobile.
  • Lógica: O clique deve adicionar ou remover uma classe (ex: .nav-ativa) no <body> ou no <nav>.
  • CSS: Use a classe .nav-ativa para mostrar/esconder o menu (ex: com transform: translateX(0)).

Funcionalidade 3: Dark Mode com localStorage

O CSS (com prefers-color-scheme) é ótimo, mas um toggle manual que *salva* a preferência do usuário é mais profissional.

Elucidação: Usando localStorage

localStorage é um "pequeno banco de dados" no navegador. Ele permite salvar strings (texto) que persistem mesmo depois que o usuário fecha a aba.

  • localStorage.setItem('nomeDaChave', 'valor'): Salva um valor.
  • localStorage.getItem('nomeDaChave'): Lê um valor.

Sua lógica deve ser:

  1. No clique do botão "toggle":
    • Verifique se o <html> tem a classe .dark-mode.
    • Se tiver, remova a classe e salve no localStorage: localStorage.setItem('theme', 'light').
    • Se não tiver, adicione a classe e salve: localStorage.setItem('theme', 'dark').
  2. Quando a página carrega (fora de qualquer evento):
    • Verifique o que está salvo: const temaSalvo = localStorage.getItem('theme').
    • Se temaSalvo === 'dark', adicione a classe .dark-mode ao <html>.

Isso garante que, ao recarregar a página, o site "lembre" da escolha do usuário.

Parte 3: O Desafio Final - Formulário de Contato com AJAX

Um portfólio profissional não deve "quebrar" a experiência do usuário com um recarregamento de página ao enviar um formulário. Vamos usar JavaScript para enviar os dados de forma assíncrona.

Desafio: Envio de Formulário com Fetch API

Como não temos um back-end real, usaremos um serviço gratuito de "form endpoint" como o Formspree, Netlify Forms ou Getform. Eles fornecem uma URL para a qual você pode enviar seus dados.

1. O HTML (Pequena Mudança)

Seu formulário precisa de um id e, o mais importante, a action (a URL que o Formspree lhe dará) e o method="POST".

<form id="form-contato" action="https://formspree.io/f/SUA_ID_UNICA" method="POST">
    <!-- ... seus inputs (nome, email, mensagem) ... -->
    <button type="submit">Enviar Mensagem</button>
</form>
<!-- Adicione um local para a mensagem de sucesso/erro -->
<div id="form-status"></div>

2. O JavaScript (A Lógica Principal)

O objetivo é "interceptar" o envio padrão do formulário, pegar os dados e enviá-los nós mesmos usando a Fetch API.

// 1. Pegue as referências do formulário e do status
const form = document.getElementById('form-contato');
const formStatus = document.getElementById('form-status');

// 2. Adicione o "ouvinte" ao evento de SUBMIT
form.addEventListener('submit', (event) => {
    // 3. IMPEDE o comportamento padrão (recarregar a página)
    event.preventDefault(); 
    
    // 4. Crie um objeto FormData para coletar os dados
    const formData = new FormData(form);
    
    // Mostra um feedback de "enviando..."
    formStatus.innerHTML = "Enviando...";
    
    // 5. Use a Fetch API para enviar os dados
    fetch(form.action, {
        method: form.method,
        body: formData,
        headers: {
            'Accept': 'application/json' // O Formspree prefere isso
        }
    })
    .then(response => {
        // 6. Verifique se o envio foi bem-sucedido
        if (response.ok) {
            formStatus.innerHTML = "Obrigado! Sua mensagem foi enviada.";
            form.reset(); // Limpa o formulário
        } else {
            // Se o servidor retornar um erro
            response.json().then(data => {
                if (data.errors) {
                    // Erro de validação do Formspree
                    formStatus.innerHTML = data.errors.map(err => err.message).join(', ');
                } else {
                    formStatus.innerHTML = "Oops! Houve um problema ao enviar.";
                }
            });
        }
    })
    .catch(error => {
        // 7. Capture erros de rede (ex: sem internet)
        formStatus.innerHTML = "Erro de rede. Tente novamente.";
    });
});

Ao implementar isso, você estará demonstrando uma das habilidades mais requisitadas no desenvolvimento front-end moderno: a comunicação assíncrona com APIs.
