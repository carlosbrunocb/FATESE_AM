<p class="text-lg text-gray-700 leading-relaxed">

</p>
<p class="text-lg text-gray-700 leading-relaxed mt-3">

</p>

<span class="text-lg text-gray-700">Baseado no protocolo HTTP</span>
</li>
<li class="flex items-start">
<span class="text-blue-600 mr-2 mt-1"><i class="fas fa-cube"></i></span>
<span class="text-lg text-gray-700">Foco em recursos identificáveis por URLs</span>
</li>
<li class="flex items-start">
<span class="text-blue-600 mr-2 mt-1"><i class="fas fa-exchange-alt"></i></span>
<span class="text-lg text-gray-700">Manipulação de recursos via métodos HTTP padrão</span>
</li>
<li class="flex items-start">
<span class="text-blue-600 mr-2 mt-1"><i class="fas fa-stream"></i></span>

<h2 class="text-2xl font-semibold text-blue-700 mb-4">O que é GraphQL?</h2>
<p class="text-lg text-gray-700 leading-relaxed">
<strong>GraphQL</strong> é uma linguagem de consulta e manipulação de dados para APIs, desenvolvida pelo Facebook em 2015. Diferentemente do REST e SOAP, o cliente especifica exatamente quais dados deseja receber.
</p>
</div>
<div class="mb-6">
<h2 class="text-2xl font-semibold text-blue-700 mb-4">Características do GraphQL</h2>
<ul class="space-y-3">
<li class="flex items-start">
<span class="text-purple-600 mr-2 mt-1"><i class="fas fa-check-circle"></i></span>
<span class="text-lg text-gray-700">Consultas precisas: o cliente solicita exatamente o que precisa</span>
</li>
<li class="flex items-start">
<span class="text-purple-600 mr-2 mt-1"><i class="fas fa-check-circle"></i></span>
<span class="text-lg text-gray-700">Um único endpoint para todas as operações</span>
</li>
<li class="flex items-start">
<span class="text-purple-600 mr-2 mt-1"><i class="fas fa-check-circle"></i></span>
<span class="text-lg text-gray-700">Fortemente tipado com um schema definido</span>
</li>
<li class="flex items-start">
<span class="text-purple-600 mr-2 mt-1"><i class="fas fa-check-circle"></i></span>
<span class="text-lg text-gray-700">Resolução hierárquica de dados</span>
</li>
</ul>
</div>
</div>
<!-- Coluna da direita com diagrama e exemplos -->
<div class="w-1/2 pl-8">
<div class="content-box bg-purple-50 rounded-lg p-6 border border-purple-100 mb-6">
<h3 class="text-xl font-semibold text-purple-700 mb-3">Como o GraphQL se compara?</h3>
<div class="grid grid-cols-3 gap-3 mb-4 text-center">
<div class="bg-blue-100 rounded-lg p-3">
<p class="font-semibold">REST</p>
<p class="text-sm">Múltiplos endpoints</p>
<p class="text-sm">Over/under-fetching</p>
</div>
<div class="bg-green-100 rounded-lg p-3">
<p class="font-semibold">SOAP</p>
<p class="text-sm">XML verbose</p>
<p class="text-sm">Alta complexidade</p>
</div>
<div class="bg-purple-100 rounded-lg p-3">
<p class="font-semibold">GraphQL</p>
<p class="text-sm">Endpoint único</p>
<p class="text-sm">Dados precisos</p>
</div>
</div>
<p class="text-gray-700">O GraphQL resolve problemas comuns do REST como over-fetching (receber mais dados que o necessário) e under-fetching (precisar fazer múltiplas chamadas).</p>
</div>
<div class="content-box bg-gray-50 rounded-lg p-5 border border-gray-200">
<h3 class="text-xl font-semibold text-purple-700 mb-3">Exemplo de consulta GraphQL</h3>
<div class="flex space-x-4">
<div class="w-1/2 bg-gray-800 rounded p-3 text-white text-sm">
<pre>query {
  user(id: 1) {
    name
    email
    posts {
      title
    }
  }
}</pre>
</div>
<div class="w-1/2 bg-gray-800 rounded p-3 text-white text-sm">
<pre>{
  "data": {
    "user": {
      "name": "João",
      "email": "joao@email.com",
      "posts": [
        { "title": "API GraphQL" }
      ]
    }
  }
}</pre>
</div>
</div>
<p class="mt-3 text-sm text-gray-600">O cliente especifica a estrutura exata dos dados desejados na resposta.</p>
</div>
