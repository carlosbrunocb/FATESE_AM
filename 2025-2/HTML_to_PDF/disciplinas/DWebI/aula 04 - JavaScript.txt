<h1 class="text-4xl font-bold mb-4 text-gray-800 border-b-2 border-gray-200 pb-3">Debugging em JavaScript</h1>
<div class="mt-4">
<p class="text-lg mb-4">Debugging é o processo de encontrar e corrigir erros em código JavaScript. É uma parte essencial do desenvolvimento que ajuda a identificar problemas na lógica ou na implementação do código.</p>
<ul class="list-disc pl-6 mt-2 space-y-2">
<li><span class="font-medium text-blue-700">Propósito:</span> Encontrar e resolver erros que impedem o funcionamento correto do código</li>
<li><span class="font-medium text-blue-700">Quando usar:</span> Durante desenvolvimento ou quando o código não se comporta como esperado</li>
<li><span class="font-medium text-blue-700">Benefícios:</span> Economiza tempo, melhora a qualidade do código e ajuda no aprendizado</li>
</ul>
</div>
<div class="debug-tools">
<div class="debug-tool">
<h3 class="flex items-center"><i class="fas fa-terminal tool-icon"></i>console.log()</h3>
<p class="text-sm">Método mais simples para inspecionar valores durante a execução do código.</p>
</div>
<div class="debug-tool">
<h3 class="flex items-center"><i class="fas fa-map-pin tool-icon"></i>Breakpoints</h3>
<p class="text-sm">Pause a execução em pontos específicos usando as DevTools do navegador.</p>
</div>
<div class="debug-tool">
<h3 class="flex items-center"><i class="fas fa-bug tool-icon"></i>debugger</h3>
<p class="text-sm">Palavra-chave que força uma pausa na execução quando as DevTools estão abertas.</p>
</div>
</div>
<div class="code-block">
<pre class="text-white"><code>// Exemplo de técnicas de debugging
function calcularMedia(notas) {
  // Usando console.log para verificar valores
  console.log("Notas recebidas:", notas);
  
  // Palavra-chave debugger para pausar execução
  debugger;
  
  let soma = 0;
  for(let i = 0; i &lt; notas.length; i++) {
    soma += notas[i];
  }
  
  return soma / notas.length;
}</code></pre>

<h1 class="text-4xl font-bold mb-4 text-gray-800 border-b-2 border-gray-200 pb-3">console.log() e Debugger Keyword</h1>
<div class="mt-4">
<p class="text-lg mb-4">O debug é essencial para encontrar e resolver problemas no seu código JavaScript. Existem duas ferramentas fundamentais para essa tarefa:</p>
<div class="two-columns">
<div class="column">
<h3 class="font-semibold text-xl text-blue-700 mb-2"><i class="fas fa-terminal mr-2"></i>console.log()</h3>
<ul class="list-disc pl-6 mt-2 space-y-1">
<li>Exibe valores no console do navegador</li>
<li>Não interfere na execução do código</li>
<li>Útil para visualizar o estado de variáveis</li>
<li>Permite verificar o fluxo de execução</li>
</ul>
</div>
<div class="column">
<h3 class="font-semibold text-xl text-blue-700 mb-2"><i class="fas fa-bug mr-2"></i>debugger</h3>
<ul class="list-disc pl-6 mt-2 space-y-1">
<li>Palavra-chave que cria um breakpoint</li>
<li>Pausa a execução do código</li>
<li>Permite inspeção detalhada do estado</li>
<li>Funciona com as DevTools abertas</li>
</ul>
</div>
</div>
</div>
<div class="explanation-block">
<p class="font-medium text-blue-700 mb-1"><i class="fas fa-lightbulb mr-2"></i>Exemplos práticos:</p>
<p>Vejamos como utilizar estas ferramentas para depurar um código que calcula o preço final de um produto.</p>
</div>
<div class="code-block">
<pre class="text-white"><code>// Função para calcular preço com desconto
function calcularPrecoFinal(preco, desconto) {
    console.log('Preço original:', preco);
    console.log('Desconto aplicado:', desconto);
    
    // Debugger pausa a execução aqui quando as DevTools estão abertas
    debugger;
    
    const precoFinal = preco * (1 - desconto/100);
    console.log('Preço final calculado:', precoFinal);
    
    return precoFinal;
}

// Exemplo de uso
const precoProduto = 150;
const descontoBlackFriday = 15;
const valorFinal = calcularPrecoFinal(precoProduto, descontoBlackFriday);</code></pre>
</div>

<div class="mt-4">
<p class="text-lg mb-4">O escopo (scope) em JavaScript determina a visibilidade e acessibilidade de variáveis, funções e objetos em seu código. Entender o escopo é fundamental para escrever código eficiente e evitar bugs.</p>
</div>
<div class="scope-grid">
<div class="scope-card global-card">
<h3 class="text-xl font-bold mb-2 text-gray-800">Global Scope</h3>
<p class="mb-2">Variáveis declaradas fora de funções são acessíveis em qualquer parte do código.</p>
<p class="text-sm mb-1 text-gray-600"><i class="fas fa-info-circle mr-1"></i>Características:</p>
<ul class="list-disc pl-5 text-sm">
<li>Acessível de qualquer lugar</li>
<li>Pode causar conflitos de nomes</li>
<li>Permanece na memória durante toda execução</li>
</ul>
</div>
<div class="scope-card function-card">
<h3 class="text-xl font-bold mb-2 text-gray-800">Function Scope</h3>
<p class="mb-2">Variáveis declaradas dentro de uma função só são acessíveis naquela função.</p>
<p class="text-sm mb-1 text-gray-600"><i class="fas fa-info-circle mr-1"></i>Características:</p>
<ul class="list-disc pl-5 text-sm">
<li>Visível apenas dentro da função</li>
<li>Protege variáveis de acesso externo</li>
<li>Libera memória após execução</li>
</ul>
</div>
<div class="scope-card block-card">
<h3 class="text-xl font-bold mb-2 text-gray-800">Block Scope</h3>
<p class="mb-2">Variáveis let/const declaradas em blocos {} só são visíveis dentro do bloco.</p>
<p class="text-sm mb-1 text-gray-600"><i class="fas fa-info-circle mr-1"></i>Características:</p>
<ul class="list-disc pl-5 text-sm">
<li>Introduzido com ES6</li>
<li>Funciona com let/const (não com var)</li>
<li>Reduz risco de conflito de variáveis</li>
</ul>
</div>
</div>
<div class="code-block">
<pre class="text-white"><code>// Global Scope
let globalVar = "Sou global";

function exemploFunction() {
  // Function Scope
  let functionVar = "Sou local da função";
  console.log(globalVar);  // Acessível
  
  if (true) {
    // Block Scope
    let blockVar = "Sou local do bloco";
    const blockConst = "Também sou do bloco";
    var naoRespeitaBloco = "Visível na função inteira";
    
    console.log(blockVar);       // Acessível
    console.log(functionVar);    // Acessível
  }
  
  console.log(naoRespeitaBloco); // Acessível (var ignora bloco)
  // console.log(blockVar);      // Erro! Não acessível
}</code></pre>
</div>

<h1 class="text-4xl font-bold mb-4 text-gray-800 border-b-2 border-gray-200 pb-3">JavaScript Hoisting</h1>
<div class="mt-4">
<p class="text-lg mb-4"><strong>Hoisting</strong> é o comportamento padrão do JavaScript de mover declarações para o topo do escopo atual antes da execução do código.</p>
<ul class="list-disc pl-6 mt-2 space-y-2">
<li>Declarações de <span class="font-medium text-blue-700">variáveis</span> e <span class="font-medium text-blue-700">funções</span> são elevadas</li>

<li>Comportamento diferente entre <span class="font-medium text-blue-700">var</span> vs. <span class="font-medium text-blue-700">let/const</span></li>

<p class="font-medium text-blue-700 mb-1"><i class="fas fa-lightbulb mr-2"></i>Diferenças importantes:</p>
<p>Variáveis declaradas com <code>var</code> são içadas e inicializadas como <code>undefined</code>.</p>
<p>Variáveis declaradas com <code>let</code> e <code>const</code> são içadas mas não são inicializadas (Temporal Dead Zone).</p>
</div>

let sobrenome = "Bruno";

// Hoisting em funções
saudacao(); // "Olá, Mundo!"
function saudacao() {

}

// Expressões de função não são hoisted
// bemVindo(); // Erro: bemVindo is not a function
var bemVindo = function() {

function somar(a, b) {
  return a + b;
}

const multiplicar = function(a, b) {

console.log(saudacao('Carlos'));     // Saída: Olá, Carlos!
console.log(somar(5, 3));            // Saída: 8
