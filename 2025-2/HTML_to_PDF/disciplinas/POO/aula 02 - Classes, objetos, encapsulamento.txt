<p>Classes definidas pelo usuário para modelar objetos do domínio da aplicação.</p>

<div class="key-point">
<p class="text-lg font-semibold">Classes Abstratas</p>
<p>Criadas com o módulo <code>abc</code> (Abstract Base Classes), definem interfaces sem implementação completa.</p>
</div>
<div class="key-point">
<p class="text-lg font-semibold">Classes Built-in</p>
<p>Classes nativas do Python como <code>list</code>, <code>dict</code>, <code>str</code> e <code>int</code>.</p>
</div>
<div class="key-point">
<p class="text-lg font-semibold">Classes de Dados (dataclass)</p>
<p>Introduzidas no Python 3.7 para simplificar a criação de classes que armazenam principalmente dados.</p>
</div>

<pre><span class="keyword">class</span> <span class="function">Carro</span>:

    quantidade_rodas = <span class="string">4</span>
    
    <span class="comment"># Método construtor</span>
    <span class="keyword">def</span> <span class="function">__init__</span>(self, marca, modelo, ano, cor=<span class="string">"branco"</span>):
        <span class="comment"># Atributos de instância - específicos para cada objeto</span>

        self.<span class="variable">modelo</span> = modelo
        self.<span class="variable">ano</span> = ano
        self.<span class="variable">cor</span> = cor
        self.<span class="variable">ligado</span> = <span class="keyword">False</span>
        self.<span class="variable">velocidade</span> = <span class="string">0</span>
    
    <span class="comment"># Métodos da classe</span>
    <span class="keyword">def</span> <span class="function">ligar</span>(self):
        self.<span class="variable">ligado</span> = <span class="keyword">True</span>

    <span class="keyword">def</span> <span class="function">desligar</span>(self):
        <span class="keyword">if</span> self.<span class="variable">velocidade</span> &gt; <span class="string">0</span>:
            <span class="keyword">return</span> <span class="string">"Pare o carro antes de desligar!"</span>
        self.<span class="variable">ligado</span> = <span class="keyword">False</span>
        <span class="keyword">return</span> <span class="string">f"{self.marca} {self.modelo} desligado."</span>
        
    <span class="keyword">def</span> <span class="function">acelerar</span>(self, incremento=<span class="string">10</span>):

<!-- Header -->
<div class="px-16 pt-12">
<h1 class="text-4xl font-bold python-accent">Atributos de Classe vs de Instância</h1>
<div class="accent-bar"></div>
</div>
<!-- Main Content -->
<div class="px-16 py-6 flex-grow">
<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    Uma classe pode ter dois tipos de atributos: <span class="font-semibold python-accent">atributos de classe</span>, que são compartilhados por todas as instâncias, e <span class="font-semibold python-accent">atributos de instância</span>, que são exclusivos para cada objeto.
</p>
</div>
<div class="grid grid-cols-5 gap-6">
<div class="col-span-3">
<div class="code-container">
<div class="code-header flex justify-between items-center">
<span>atributos.py</span>
<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="keyword">class</span> <span class="function">Funcionario</span>:
    <span class="comment"># Atributo de classe - compartilhado por todas instâncias</span>
    empresa = <span class="string">"Tech Solutions"</span>
    contador = <span class="string">0</span>
    
    <span class="keyword">def</span> <span class="function">__init__</span>(self, nome, salario):
        <span class="comment"># Atributos de instância - exclusivos para cada objeto</span>
        self.<span class="variable">nome</span> = nome
        self.<span class="variable">salario</span> = salario
        <span class="function">Funcionario</span>.contador += <span class="string">1</span>
    
<span class="comment"># Criando instâncias</span>
f1 = <span class="function">Funcionario</span>(<span class="string">"Ana"</span>, <span class="string">5000</span>)
f2 = <span class="function">Funcionario</span>(<span class="string">"Carlos"</span>, <span class="string">6000</span>)

<span class="comment"># Acessando atributos de instância</span>
print(f1.nome)        <span class="comment"># Ana</span>
print(f2.nome)        <span class="comment"># Carlos</span>

<span class="comment"># Acessando atributos de classe</span>
print(f1.empresa)     <span class="comment"># Tech Solutions</span>
print(f2.empresa)     <span class="comment"># Tech Solutions</span>
print(<span class="function">Funcionario</span>.contador) <span class="comment"># 2</span>

<span class="comment"># Alterando um atributo de classe</span>
<span class="function">Funcionario</span>.empresa = <span class="string">"Nova Tech"</span>
print(f1.empresa)     <span class="comment"># Nova Tech</span>
print(f2.empresa)     <span class="comment"># Nova Tech</span></pre>
</div>
</div>
</div>
<div class="col-span-2">
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">Atributos de classe</span>: Definidos diretamente na classe, fora de métodos. São compartilhados por todas as instâncias.</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">Atributos de instância</span>: Definidos dentro de métodos usando o parâmetro <code>self</code>. Cada instância tem sua própria cópia.</p>
</div>
<div class="note-bubble p-4">
<p class="text-lg">A alteração em um <span class="font-semibold python-accent">atributo de classe</span> afeta todas as instâncias, enquanto alterações em <span class="font-semibold python-accent">atributos de instância</span> são específicas para o objeto modificado.</p>
</div>
</div>
</div>

<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    Instanciar objetos significa criar exemplares concretos a partir de uma classe. Isso é feito chamando o nome da classe como se fosse uma função.
</p>
</div>
<div class="grid grid-cols-5 gap-6">
<div class="col-span-3">
<div class="code-container">
<div class="code-header flex justify-between items-center">

<div class="code-content text-lg">
<pre><span class="comment"># Definição da classe Carro</span>
<span class="keyword">class</span> <span class="function">Carro</span>:
    <span class="keyword">def</span> <span class="function">__init__</span>(self, marca, modelo, ano):
        self.<span class="variable">marca</span> = marca
        self.<span class="variable">modelo</span> = modelo
        self.<span class="variable">ano</span> = ano
        self.<span class="variable">ligado</span> = False
        
    <span class="keyword">def</span> <span class="function">ligar</span>(self):
        self.<span class="variable">ligado</span> = True
        <span class="keyword">return</span> <span class="string">f"{self.marca} {self.modelo} está ligado!"</span>

<span class="comment"># Instanciando objetos da classe Carro</span>
carro1 = <span class="function">Carro</span>(<span class="string">'Ford'</span>, <span class="string">'Ka'</span>, <span class="string">2020</span>)
carro2 = <span class="function">Carro</span>(<span class="string">'Chevrolet'</span>, <span class="string">'Onix'</span>, <span class="string">2021</span>)

<span class="comment"># Acessando atributos e métodos</span>
print(carro1.<span class="variable">marca</span>)  <span class="comment"># Saída: Ford</span>
print(carro2.<span class="function">ligar</span>())  <span class="comment"># Saída: Chevrolet Onix está ligado!</span></pre>

<div class="col-span-2">
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">Instanciar</span>: Criar um objeto específico a partir do molde (classe)</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">Construtor</span>: O método <span class="font-mono">__init__</span> é chamado automaticamente durante a instanciação</p>

<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">Múltiplas instâncias</span>: É possível criar vários objetos independentes da mesma classe</p>
</div>
<div class="note-bubble p-4">
<p class="text-lg"><span class="font-semibold python-accent">Parâmetros</span>: Os valores passados são atribuídos aos atributos do objeto</p>

        self.<span class="variable">ativo</span> = True

<div class="px-16 pt-12">
<h1 class="text-4xl font-bold python-accent">O método __new__</h1>
<div class="accent-bar"></div>
</div>
<!-- Main Content -->
<div class="px-16 py-6 flex-grow">
<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    O método <span class="font-semibold python-accent">__new__</span> é responsável pela <span class="font-semibold">criação</span> do objeto, enquanto o <span class="font-semibold python-accent">__init__</span> inicializa o objeto criado. É raramente sobrescrito diretamente, mas útil em casos específicos.
</p>
</div>
<div class="grid grid-cols-5 gap-6">
<div class="col-span-3">
<div class="code-container">
<div class="code-header flex justify-between items-center">
<span>singleton_exemplo.py</span>
<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="comment"># Exemplo de Singleton usando __new__</span>
<span class="keyword">class</span> <span class="function">Singleton</span>:
    <span class="variable">_instancia</span> = <span class="keyword">None</span>
    
    <span class="keyword">def</span> <span class="function">__new__</span>(cls, *args, **kwargs):
        <span class="comment"># Verifica se já existe uma instância</span>
        <span class="keyword">if</span> cls.<span class="variable">_instancia</span> <span class="keyword">is None</span>:
            <span class="comment"># Cria uma nova instância</span>
            cls.<span class="variable">_instancia</span> = <span class="function">super</span>().<span class="function">__new__</span>(cls)
        <span class="comment"># Retorna a instância existente</span>
        <span class="keyword">return</span> cls.<span class="variable">_instancia</span>
        
    <span class="keyword">def</span> <span class="function">__init__</span>(self, nome):
        <span class="comment"># Inicializa apenas na primeira chamada</span>
        <span class="keyword">if not</span> <span class="function">hasattr</span>(self, <span class="string">'nome'</span>):
            self.<span class="variable">nome</span> = nome</pre>
</div>
</div>
</div>
<div class="col-span-2">
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">__new__</span>: Recebe a classe (cls) e cria o objeto</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">__init__</span>: Recebe a instância (self) e inicializa</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold">Quando usar:</span> Subclassar tipos imutáveis, implementar singletons, controle de criação</p>
</div>
<div class="note-bubble p-4">
<p class="text-lg"><span class="font-semibold">Ordem:</span> Primeiro __new__ cria, depois __init__ inicializa</p>
</div>
</div>

<!-- Main Content -->
<div class="px-16 py-6 flex-grow">
<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    Em Python, os atributos de um objeto são armazenados em um <span class="font-semibold">dicionário interno</span> chamado <span class="font-semibold python-accent">__dict__</span>. Este mecanismo permite adicionar e acessar atributos dinamicamente durante a execução do programa.
</p>
</div>
<div class="grid grid-cols-5 gap-6">
<div class="col-span-3">
<div class="code-container">
<div class="code-header flex justify-between items-center">
<span>atributos_dinamicos.py</span>
<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="keyword">class</span> <span class="function">Pessoa</span>:
    <span class="keyword">def</span> <span class="function">__init__</span>(self, nome, idade):
        self.<span class="variable">nome</span> = nome
        self.<span class="variable">idade</span> = idade

<span class="comment"># Criando um objeto da classe Pessoa</span>
joao = <span class="function">Pessoa</span>(<span class="string">"João"</span>, <span class="string">30</span>)

<span class="comment"># Visualizando o dicionário interno</span>
print(joao.<span class="special">__dict__</span>)  <span class="comment"># {'nome': 'João', 'idade': 30}</span>

<span class="comment"># Adicionando um atributo dinamicamente</span>
joao.<span class="variable">profissao</span> = <span class="string">"Programador"</span>

<span class="comment"># Dicionário atualizado</span>
print(joao.<span class="special">__dict__</span>)  <span class="comment"># {'nome': 'João', 'idade': 30, 'profissao': 'Programador'}</span>

<span class="comment"># Acessando atributo pelo dicionário</span>
print(joao.<span class="special">__dict__</span>[<span class="string">'nome'</span>])  <span class="comment"># João</span>

<span class="comment"># Modificando atributo pelo dicionário</span>
joao.<span class="special">__dict__</span>[<span class="string">'idade'</span>] = <span class="string">31</span>
print(joao.<span class="variable">idade</span>)  <span class="comment"># 31</span></pre>
</div>
</div>
</div>
<div class="col-span-2">
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">__dict__</span>: Armazena todos os atributos de instância como um dicionário</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold">Atributos dinâmicos</span>: Podem ser adicionados em tempo de execução</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold">Acesso direto</span>: É possível manipular atributos diretamente pelo __dict__</p>
</div>
<div class="note-bubble p-4">
<p class="text-lg"><span class="font-semibold">Observação</span>: Classes com __slots__ não usam __dict__, economizando memória</p>

<div class="px-16 py-4 flex-grow">
<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    Em Python, cada objeto tem um endereço único na memória. A função <span class="font-semibold python-accent">id()</span> retorna um inteiro que representa o endereço de memória do objeto e permanece constante durante sua vida útil.
</p>
</div>
<div class="grid grid-cols-2 gap-6">
<div class="col-span-1">
<div class="code-container mb-4">
<div class="code-header flex justify-between items-center">
<span>enderecamento.py</span>
<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="comment"># Objetos diferentes têm ids diferentes</span>
a = <span class="number">10</span>
b = <span class="number">10</span>
c = <span class="string">"Olá"</span>

<span class="comment"># Imprime endereço dos objetos</span>
print(<span class="keyword">id</span>(a))  <span class="comment"># Ex: 140720345228752</span>
print(<span class="keyword">id</span>(b))  <span class="comment"># Mesmo id de a</span>
print(<span class="keyword">id</span>(c))  <span class="comment"># Diferente dos anteriores</span>

<span class="comment"># Variáveis referenciando o mesmo objeto</span>
lista1 = [<span class="number">1</span>, <span class="number">2</span>, <span class="number">3</span>]
lista2 = lista1

print(<span class="keyword">id</span>(lista1) == <span class="keyword">id</span>(lista2))  <span class="comment"># True</span></pre>
</div>
</div>
<div class="memory-box p-4 bg-white">
<h3 class="font-bold python-accent text-lg mb-2">Entendendo o Output:</h3>
<p class="mb-2">Objetos imutáveis com mesmo valor podem compartilhar endereço:</p>
<div class="flex items-center mb-2">
<span class="memory-address">0x7ff...</span> <code>a = 10</code> e <code>b = 10</code>
</div>
<p class="mb-2">Objetos diferentes sempre têm endereços diferentes:</p>
<div class="flex items-center">
<span class="memory-address">0x7fa...</span> <code>c = "Olá"</code>
</div>
</div>
</div>
<div class="col-span-1">
<div class="code-container mb-4">

<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="comment"># Comparação de identidade vs igualdade</span>
x = [<span class="number">1</span>, <span class="number">2</span>, <span class="number">3</span>]
y = [<span class="number">1</span>, <span class="number">2</span>, <span class="number">3</span>]

print(x == y)      <span class="comment"># True - mesmos valores</span>
print(x <span class="keyword">is</span> y)      <span class="comment"># False - objetos diferentes</span>
print(<span class="keyword">id</span>(x) == <span class="keyword">id</span>(y))  <span class="comment"># False - ids diferentes</span>

<span class="comment"># Modificar uma lista afeta todas as referências</span>
a = [<span class="number">4</span>, <span class="number">5</span>, <span class="number">6</span>]
b = a
a.append(<span class="number">7</span>)

print(b)  <span class="comment"># [4, 5, 6, 7] - b também muda!</span>
print(<span class="keyword">id</span>(a) == <span class="keyword">id</span>(b))  <span class="comment"># True</span></pre>
</div>
</div>
<div class="bg-white p-4 rounded-lg border-l-4 border-yellow-500">
<h3 class="font-bold text-lg mb-2">Dicas importantes:</h3>
<ul class="list-disc ml-5 space-y-2">
<li>O operador <code>is</code> compara identidade (id dos objetos)</li>
<li>O operador <code>==</code> compara valores (igualdade)</li>
<li>Objetos mutáveis alterados afetam todas suas referências</li>
<li>Entender endereçamento é crucial para evitar bugs sutis</li>
</ul>

<div class="px-16 py-6 flex-grow">
<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    Uma cópia rasa (shallow copy) cria um <span class="font-semibold">novo objeto</span> mas mantém as <span class="font-semibold python-accent">referências</span> para os objetos aninhados. Alterações nos objetos internos afetam ambas as cópias.
</p>
</div>
<div class="grid grid-cols-5 gap-6">
<div class="col-span-3">
<div class="code-container">

<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="keyword">import</span> copy

<span class="comment"># Lista com outra lista aninhada</span>
original = [<span class="string">1</span>, <span class="string">2</span>, [<span class="string">3</span>, <span class="string">4</span>]]

<span class="comment"># Criando uma cópia rasa</span>
copia = copy.<span class="function">copy</span>(original)

print(<span class="string">f"Original: {</span>original<span class="string">}"</span>)    <span class="comment"># Original: [1, 2, [3, 4]]</span>
print(<span class="string">f"Cópia: {</span>copia<span class="string">}"</span>)        <span class="comment"># Cópia: [1, 2, [3, 4]]</span>

<span class="comment"># Verificando se são objetos diferentes</span>
print(<span class="string">f"Mesma lista? {</span>original <span class="keyword">is</span> copia<span class="string">}"</span>)  <span class="comment"># Mesma lista? False</span>

<span class="comment"># Porém, os objetos internos SÃO os mesmos</span>
print(<span class="string">f"Mesmo objeto interno? {</span>original[<span class="string">2</span>] <span class="keyword">is</span> copia[<span class="string">2</span>]<span class="string">}"</span>)  <span class="comment"># Mesmo objeto interno? True</span>

<span class="comment"># Modificando a lista interna no original</span>
original[<span class="string">2</span>].<span class="function">append</span>(<span class="string">5</span>)

print(<span class="string">f"Original após mudança: {</span>original<span class="string">}"</span>)  <span class="comment"># [1, 2, [3, 4, 5]]</span>
print(<span class="string">f"Cópia após mudança: {</span>copia<span class="string">}"</span>)      <span class="comment"># [1, 2, [3, 4, 5]]</span></pre>

</div>

<p class="text-lg"><span class="font-semibold python-accent">copy.copy()</span>: Cria uma cópia rasa do objeto</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">Objetos diferentes</span>: A lista principal é duplicada (novo objeto)</p>
</div>
<div class="note-bubble p-4 mb-4">

<div class="px-16 pt-12">
<h1 class="text-4xl font-bold python-accent">Cópia Profunda (Deep Copy)</h1>
<div class="accent-bar"></div>
</div>
<!-- Main Content -->
<div class="px-16 py-6 flex-grow">
<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    A <span class="font-semibold">cópia profunda (deep copy)</span> cria uma duplicação completa de um objeto e de todos os objetos aninhados dentro dele. Isso garante <span class="font-semibold python-accent">independência total</span> entre o objeto original e sua cópia.
</p>
</div>
<div class="grid grid-cols-5 gap-6">
<div class="col-span-3">
<div class="code-container">
<div class="code-header flex justify-between items-center">
<span>deep_copy_exemplo.py</span>
<i class="fab fa-python"></i>
</div>
<div class="code-content text-lg">
<pre><span class="keyword">import</span> copy

<span class="comment"># Objeto com estrutura aninhada</span>
original = [{<span class="string">'nome'</span>: <span class="string">'João'</span>, <span class="string">'notas'</span>: [7.5, 8.0, 6.5]}, 
           {<span class="string">'nome'</span>: <span class="string">'Maria'</span>, <span class="string">'notas'</span>: [9.0, 8.5, 10.0]}]

<span class="comment"># Criando uma cópia profunda</span>
copia_profunda = copy.<span class="function">deepcopy</span>(original)

<span class="comment"># Modificando um objeto aninhado na cópia</span>
copia_profunda[0][<span class="string">'notas'</span>][0] = 9.5

<span class="comment"># Verificando o resultado</span>
print(<span class="string">"Original:"</span>, original[0][<span class="string">'notas'</span>])
print(<span class="string">"Cópia profunda:"</span>, copia_profunda[0][<span class="string">'notas'</span>])

<span class="comment"># Saída:
# Original: [7.5, 8.0, 6.5]
# Cópia profunda: [9.5, 8.0, 6.5]</span></pre>
</div>
</div>
</div>
<div class="col-span-2">
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold python-accent">copy.deepcopy()</span>: Cria uma cópia recursiva de todo o objeto</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold">Independência Total:</span> Modificações em objetos aninhados não afetam o original</p>
</div>
<div class="note-bubble p-4 mb-4">
<p class="text-lg"><span class="font-semibold">Uso:</span> Ideal para estruturas complexas ou quando a independência completa é necessária</p>
</div>
<div class="note-bubble p-4">
<p class="text-lg"><span class="font-semibold">Desvantagem:</span> Consome mais memória e é mais lenta que a cópia rasa</p>
</div>
</div>

    <span class="keyword">def</span> <span class="function">__init__</span>(self, titular, saldo):

    <span class="keyword">def</span> <span class="function">__init__</span>(self, celsius=0):
        self.<span class="variable">_celsius</span> = celsius  <span class="comment"># atributo não-público</span>
    
    <span class="decorator">@property</span>
    <span class="keyword">def</span> <span class="function">celsius</span>(self):
        <span class="comment"># Getter - Acessa o valor</span>
        <span class="keyword">return</span> self.<span class="variable">_celsius</span>
    
    <span class="decorator">@celsius.setter</span>
    <span class="keyword">def</span> <span class="function">celsius</span>(self, valor):
        <span class="comment"># Setter - Valida e atribui o valor</span>
        <span class="keyword">if</span> valor &lt; -273.15:
            <span class="keyword">raise</span> <span class="function">ValueError</span>(<span class="string">"Temperatura abaixo do zero absoluto!"</span>)

    <span class="keyword">def</span> <span class="function">fahrenheit</span>(self):

        <span class="comment"># Atributo privado com dois underscores</span>
        self.<span class="variable">__nome</span> = nome
        
    <span class="comment"># Getter - Método para acessar o atributo privado</span>
    <span class="keyword">def</span> <span class="function">get_nome</span>(self):

<span class="comment"># Usando a classe</span>

pessoa.<span class="function">set_nome</span>(<span class="string">"Ana Maria"</span>)

<div class="explanation-box p-5 mb-6">
<p class="text-xl leading-relaxed">
    O decorator <span class="font-semibold python-accent">@property</span> permite criar atributos com comportamento de método, mas com sintaxe de acesso a atributos. Isso torna o código mais elegante e <span class="font-semibold">pythônico</span>.

        self.<span class="variable">nome</span> = nome
        self.<span class="variable">_preco</span> = 0  <span class="comment"># Atributo protegido</span>
        self.preco = preco  <span class="comment"># Chama o setter</span>
    
    <span class="decorator">@property</span>

        <span class="keyword">if</span> valor &gt; 0:
            self.<span class="variable">_idade</span> = valor
        <span class="keyword">else</span>:
            <span class="keyword">raise</span> <span class="function">ValueError</span>(<span class="string">"Idade deve ser positiva"</span>)
            
<span class="comment"># Usando a classe</span>
pessoa = <span class="function">Pessoa</span>(<span class="string">"João"</span>, 25)
print(pessoa.<span class="function">get_idade</span>())  <span class="comment"># 25</span>
pessoa.<span class="function">set_idade</span>(30)  <span class="comment"># Valor válido</span>
<span class="comment"># pessoa.set_idade(-5)  # Erro!</span></pre>
</div>
</div>
<div class="code-container">
<div class="code-header flex justify-between items-center">
<span class="approach-header">Abordagem 3: Property</span>
</div>
<div class="code-content text-sm">
<pre><span class="keyword">class</span> <span class="function">Pessoa</span>:
    <span class="keyword">def</span> <span class="function">__init__</span>(self, nome, idade):
        self.<span class="variable">nome</span> = nome
        self.<span class="variable">_idade</span> = idade  <span class="comment"># protegido</span>
        
    <span class="comment"># Getter</span>
    <span class="variable">@property</span>
    <span class="keyword">def</span> <span class="function">idade</span>(self):
        <span class="keyword">return</span> self.<span class="variable">_idade</span>
        
    <span class="comment"># Setter</span>
    <span class="variable">@idade.setter</span>
    <span class="keyword">def</span> <span class="function">idade</span>(self, valor):
        <span class="keyword">if</span> valor &gt; 0:
            self.<span class="variable">_idade</span> = valor
        <span class="keyword">else</span>:
            <span class="keyword">raise</span> <span class="function">ValueError</span>(<span class="string">"Idade deve ser positiva"</span>)
            
<span class="comment"># Usando a classe</span>
pessoa = <span class="function">Pessoa</span>(<span class="string">"João"</span>, 25)
print(pessoa.<span class="variable">idade</span>)  <span class="comment"># 25</span>
pessoa.<span class="variable">idade</span> = 30  <span class="comment"># Usa setter implicitamente</span>
<span class="comment"># pessoa.idade = -5  # Erro!</span></pre>
</div>
</div>
</div>
<div class="grid grid-cols-3 gap-4">
<div class="note-bubble p-3">
<p class="text-base font-semibold python-accent mb-2">Acesso Direto</p>
<ul class="list-disc pl-5 text-sm">
<li>Simples e direto</li>
<li>Sem validação dos dados</li>
<li>Não segue o princípio de encapsulamento</li>
<li>Difícil de modificar implementação depois</li>
</ul>
</div>
<div class="note-bubble p-3">
<p class="text-base font-semibold python-accent mb-2">Getters/Setters Tradicionais</p>
<ul class="list-disc pl-5 text-sm">
<li>Permite validação dos dados</li>
<li>Comum em outras linguagens (Java)</li>
<li>Menos pythônico (verboso)</li>
<li>Requer alteração do código cliente</li>
</ul>
</div>
<div class="note-bubble p-3">
<p class="text-base font-semibold python-accent mb-2">Property</p>
<ul class="list-disc pl-5 text-sm">
<li>Abordagem pythônica recomendada</li>
<li>Preserva interface de atributo</li>
<li>Permite validação de dados</li>
<li>Facilita refatoração de código existente</li>
</ul>
</div>
