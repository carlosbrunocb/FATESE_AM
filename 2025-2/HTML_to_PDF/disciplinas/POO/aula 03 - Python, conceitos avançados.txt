<h2 class="subtitle text-2xl mb-4">O que são?</h2>
<ul class="space-y-3 text-lg">
<li><span class="font-semibold text-blue-700">Atributos de Classe:</span> Compartilhados por todas as instâncias de uma classe. São definidos no escopo da classe, fora de qualquer método.</li>
<li><span class="font-semibold text-blue-700">Atributos de Instância:</span> Específicos para cada objeto. São definidos no método __init__ ou em outros métodos usando self.</li>
</ul>
<h2 class="subtitle text-2xl mt-6 mb-4">Características</h2>
<div class="flex space-x-4">
<div class="w-1/2 bg-gray-100 p-3 rounded-lg">
<h3 class="font-semibold text-center border-b border-gray-300 pb-1 mb-2">Atributo de Classe</h3>
<ul class="list-disc pl-5 space-y-1">
<li>Compartilhado por todas instâncias</li>
<li>Acesso via Classe.atributo</li>
<li>Memória eficiente</li>
<li>Ideal para constantes</li>
</ul>
</div>
<div class="w-1/2 bg-gray-100 p-3 rounded-lg">
<h3 class="font-semibold text-center border-b border-gray-300 pb-1 mb-2">Atributo de Instância</h3>
<ul class="list-disc pl-5 space-y-1">
<li>Exclusivo para cada objeto</li>
<li>Acesso via objeto.atributo</li>
<li>Independente entre instâncias</li>
<li>Ideal para propriedades individuais</li>
</ul>
</div>
</div>
</div>

    <span class="comment"># Atributo de classe</span>
    empresa = <span class="string">"TechCorp"</span>
    contador = 0

    <span class="keyword">def</span> <span class="function">__init__</span>(self, nome, salario):
        <span class="comment"># Atributos de instância</span>
        self.nome = nome
        self.salario = salario
        <span class="attr">Funcionario</span>.contador += 1

<span class="comment"># Criando instâncias</span>
func1 = <span class="attr">Funcionario</span>(<span class="string">"Ana"</span>, 5000)
func2 = <span class="attr">Funcionario</span>(<span class="string">"Carlos"</span>, 6000)

<span class="comment"># Acessando atributos</span>
print(func1.nome)          <span class="comment"># Ana</span>
print(func2.nome)          <span class="comment"># Carlos</span>
print(func1.empresa)       <span class="comment"># TechCorp</span>
print(func2.empresa)       <span class="comment"># TechCorp</span>
print(<span class="attr">Funcionario</span>.empresa)  <span class="comment"># TechCorp</span>
print(<span class="attr">Funcionario</span>.contador) <span class="comment"># 2</span></pre>

<div class="mb-6 space-y-4">

<p class="font-semibold text-xl text-blue-700">Sobrecarga (Overloading)</p>
<p class="mt-1">Capacidade de definir múltiplos métodos com o mesmo nome, mas com parâmetros diferentes (quantidade ou tipo).</p>
</div>
<div class="bg-green-50 border-l-4 border-green-500 p-3 rounded-r">
<p class="font-semibold text-xl text-green-700">Sobreescrita (Overriding)</p>
<p class="mt-1">Reimplementação de um método da classe pai em uma classe filha, mantendo a mesma assinatura.</p>
</div>
</div>
<h2 class="subtitle text-2xl mt-6 mb-4">Python e Sobrecarga</h2>
<div class="bg-gray-100 p-4 rounded-lg">
<p class="mb-2"><i class="fas fa-info-circle text-blue-600 mr-2"></i> <span class="font-semibold">Diferente de outras linguagens</span>, Python não suporta sobrecarga nativa de métodos.</p>
<p>No entanto, pode-se emular sobrecarga usando:</p>
<ul class="list-disc pl-6 mt-2">
<li>Parâmetros padrão</li>
<li>Argumentos variáveis (*args)</li>
<li>Argumentos de palavra-chave variáveis (**kwargs)</li>
<li>Verificação de tipos em runtime</li>
</ul>
</div>

<pre><span class="comment"># Exemplo de sobreescrita em Python</span>
<span class="keyword">class</span> <span class="function">Animal</span>:
    <span class="keyword">def</span> <span class="function">fazer_som</span>(self):
        <span class="keyword">return</span> <span class="string">"Som genérico de animal"</span>

<span class="keyword">class</span> <span class="function">Cachorro</span>(<span class="attr">Animal</span>):
    <span class="comment"># Sobreescrita do método fazer_som</span>
    <span class="keyword">def</span> <span class="function">fazer_som</span>(self):
        <span class="keyword">return</span> <span class="string">"Au Au!"</span>

<span class="comment"># Exemplo de emulação de sobrecarga</span>
<span class="keyword">def</span> <span class="function">calcular_area</span>(forma, *args):

<p class="mb-3 text-gray-700">Ocorre quando uma classe filha redefine um método presente na classe pai, modificando seu comportamento.</p>
<div class="code-block text-sm">
<pre><span class="keyword">class</span> <span class="function">Animal</span>:
    <span class="keyword">def</span> <span class="function">fazer_som</span>(self):
        <span class="keyword">return</span> <span class="string">"Som genérico de animal"</span>
        
<span class="keyword">class</span> <span class="function">Cachorro</span>(Animal):
    <span class="comment"># Sobrescrita do método fazer_som</span>
    <span class="keyword">def</span> <span class="function">fazer_som</span>(self):
        <span class="keyword">return</span> <span class="string">"Au au!"</span>
        
<span class="keyword">class</span> <span class="function">Gato</span>(Animal):
    <span class="comment"># Sobrescrita do método fazer_som</span>
    <span class="keyword">def</span> <span class="function">fazer_som</span>(self):
        <span class="keyword">return</span> <span class="string">"Miau!"</span>
        
animal = <span class="attr">Animal</span>()
cachorro = <span class="attr">Cachorro</span>()
gato = <span class="attr">Gato</span>()

<h2 class="subtitle text-2xl mb-4">Sobrecarga (Overloading)</h2>
<p class="mb-3 text-gray-700">Python não suporta nativamente sobrecarga como outras linguagens, mas podemos emulá-la usando parâmetros padrão ou *args/**kwargs.</p>
<div class="code-block text-sm">
<pre><span class="keyword">class</span> <span class="function">Calculadora</span>:
    <span class="comment"># Emulando sobrecarga com valores padrão</span>

    <span class="keyword">def</span> <span class="function">multiplica</span>(self, *args):
        resultado = 1
        <span class="keyword">for</span> num <span class="keyword">in</span> args:
            resultado *= num
        <span class="keyword">return</span> resultado

<li>Parâmetros com valores padrão</li>
<li>Argumentos variáveis (*args)</li>
<li>Argumentos de palavra-chave (**kwargs)</li>
<li>Verificação de tipos dentro da função</li>

<div class="bg-gray-100 p-4 rounded-lg">
<ul class="list-disc pl-5 space-y-2">

<li>Para criar funções mais flexíveis que aceitam múltiplos valores</li>
<li>Em funções que processam coleções de tamanho variável</li>
<li>Para emular sobrecarga de funções em Python</li>
<li>Para encaminhar argumentos posicionais para outras funções (forwarding)</li>

<thead>
<tr>
<th class="w-1/4 text-left rounded-tl-lg">Conceito</th>
<th class="w-2/5 text-left">Primeira Opção</th>
<th class="w-2/5 text-left rounded-tr-lg">Segunda Opção</th>
</tr>
</thead>
<tbody>
<tr>
<td class="font-semibold bg-gray-100">Atributos</td>
<td class="accent-border">
<span class="highlight">Atributo de Classe:</span><br/>
                            • Compartilhado por todas instâncias<br/>
                            • Definido no nível da classe<br/>
                            • Acesso: <code>Classe.atributo</code><br/>
                            • Ideal para valores constantes
                        </td>
<td>
<span class="highlight">Atributo de Instância:</span><br/>
                            • Específico para cada objeto<br/>
                            • Definido com <code>self</code> no <code>__init__</code><br/>
                            • Acesso: <code>objeto.atributo</code><br/>
                            • Ideal para propriedades individuais
                        </td>
</tr>
<tr>
<td class="font-semibold bg-gray-100">Sobrecarga vs Sobreescrita</td>
<td class="accent-border">
<span class="highlight">Sobrecarga (Overloading):</span><br/>
                            • Múltiplas versões do mesmo método<br/>
                            • Diferenciadas pelos parâmetros<br/>
                            • Em Python: simulado com parâmetros<br/>
                              padrão ou <code>*args</code>/<code>**kwargs</code>
</td>
<td>
<span class="highlight">Sobreescrita (Overriding):</span><br/>
                            • Redefine método da classe pai<br/>
                            • Mesma assinatura (nome e parâmetros)<br/>
                            • Implementa polimorfismo<br/>
                            • Usa <code>super()</code> para acessar original
                        </td>
</tr>
<tr>
<td class="font-semibold bg-gray-100">Argumentos Variáveis</td>
<td class="accent-border">
<span class="highlight">*args:</span><br/>
                            • Argumentos posicionais variáveis<br/>
                            • Acessados como tupla dentro da função<br/>
                            • Ideal para número variável de valores<br/>
                            • Exemplo: <code>def func(*args):</code>
</td>
<td>
<span class="highlight">**kwargs:</span><br/>
                            • Argumentos de palavra-chave variáveis<br/>
                            • Acessados como dicionário na função<br/>
                            • Ideal para opções nomeadas<br/>
                            • Exemplo: <code>def func(**kwargs):</code>
</td>
</tr>
</tbody>
</table>
<div class="mt-6 p-4 bg-blue-50 rounded-lg accent-border">
<h3 class="subtitle text-xl mb-2">Dicas de Aplicação:</h3>
<div class="grid grid-cols-3 gap-4">
<div>
<p class="font-semibold text-center mb-1">Atributos</p>
<p class="text-sm text-center">Use atributos de classe para constantes e contadores compartilhados, e atributos de instância para dados específicos de cada objeto.</p>
</div>
<div>
<p class="font-semibold text-center mb-1">Métodos</p>
<p class="text-sm text-center">Prefira sobreescrita para polimorfismo em classes herdadas e use parâmetros padrão para simular sobrecarga.</p>
</div>
<div>
<p class="font-semibold text-center mb-1">Argumentos</p>
<p class="text-sm text-center">Combine *args e **kwargs para criar funções e métodos altamente flexíveis que aceitam múltiplos tipos de entrada.</p>
</div>
