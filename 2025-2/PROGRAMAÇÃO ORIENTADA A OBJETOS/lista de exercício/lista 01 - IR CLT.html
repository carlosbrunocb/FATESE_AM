Lista de Exercícios: Cálculo de Imposto CLT com POO

Objetivo

Esta lista de exercícios tem como objetivo guiá-lo na construção de um sistema orientado a objetos para calcular os impostos e o salário líquido de um funcionário CLT. Utilizaremos os conceitos de Composição vistos na Aula 04 para estruturar nosso código de forma coesa e modular.

O sistema deve receber como entrada o salário bruto e o número de dependentes e calcular/armazenar os seguintes atributos:

  • FGTS pago pela empresa (fgts_empresa)
  • INSS retido (inss_retido)
  • Dedução por dependentes (deducao_dependentes)
  • Base de cálculo do IRPF (base_calculo_irpf)
  • Imposto de Renda Retido na Fonte (irpf_retido)
  • Salário Líquido (salario_liquido)

A cada questão, construiremos partes da solução final, culminando em um conjunto de classes coesas que geram um relatório em string.

Informações Teóricas Essenciais (Valores Fictícios/Exemplos para 2024)

Para resolver os exercícios, você precisará destas tabelas e valores:

1. FGTS (Fundo de Garantia por Tempo de Serviço):

  • Alíquota: 8% sobre o salário bruto (pago pela empresa, não descontado do funcionário).

2. Dedução por Dependente (para base do IRPF):

  • Valor: R$ 189,59 por dependente.

3. Tabela Progressiva do INSS (Contribuição Previdenciária):

O cálculo é feito faixa por faixa sobre o salário bruto.

Salário de Contribuição (R$) Alíquota
Até R$ 1.412,00 7,5%
De R$ 1.412,01 até R$ 2.666,68 9%
De R$ 2.666,69 até R$ 4.000,03 12%
De R$ 4.000,04 até R$ 7.786,02 (Teto) 14%

Exemplo Cálculo INSS (Salário R$ 3.000,00):
- Faixa 1: R$ 1.412,00 * 7,5% = R$ 105,90
- Faixa 2: (R$ 2.666,68 - R$ 1.412,00) * 9% = R$ 1.254,68 * 9% = R$ 112,92
- Faixa 3: (R$ 3.000,00 - R$ 2.666,68) * 12% = R$ 333,32 * 12% = R$ 39,99
- Total INSS: R$ 105,90 + R$ 112,92 + R$ 39,99 = R$ 258,81

4. Tabela Progressiva do IRPF (Imposto de Renda Retido na Fonte):

A base de cálculo é: Salário Bruto - INSS Retido - (Nº Dependentes * 189,59).

Base de Cálculo (R$) Alíquota Parcela a Deduzir (R$)
Até R$ 2.259,20 0% R$ 0,00
De R$ 2.259,21 até R$ 2.826,65 7,5% R$ 169,44
De R$ 2.826,66 até R$ 3.751,05 15% R$ 381,44
De R$ 3.751,06 até R$ 4.664,68 22,5% R$ 662,77
Acima de R$ 4.664,68 27,5% R$ 896,00

Exemplo Cálculo IRPF:
- Base de Cálculo: R$ 3.000,00 (Salário) - R$ 258,81 (INSS) - 0 (Dependentes) = R$ 2.741,19
- Cálculo (Faixa 7,5%): (R$ 2.741,19 * 7,5%) - R$ 169,44 = R$ 205,59 - R$ 169,44 = R$ 36,15 (IRPF Retido)

    class MinhaClasse:

  3. Questão 3: O Primeiro Cálculo (Método Simples)

    Adicione um método privado _calcular_fgts à classe CalculoCLT. Este método deve calcular 8% do salario_bruto e armazenar no atributo self.fgts_empresa.

    Crie também um método público processar_calculo que, por enquanto, apenas chamará o _calcular_fgts. Altere o __init__ para chamar self.processar_calculo() no final.

    Dica: Métodos privados em Python são indicados com um underscore (_) no início. O cálculo é self.salario_bruto * 0.08. O método processar_calculo servirá para orquestrar todas as nossas chamadas de cálculo futuramente.

  4. Questão 4: Aplicando Composição (Aula 04) - Calculador de INSS

    Aqui aplicaremos o conceito de Composição (Aula 04). O cálculo do INSS é complexo (progressivo) e não queremos que a classe CalculoCLT tenha essa responsabilidade (Princípio da Responsabilidade Única).

    Crie uma nova classe chamada CalculadorINSS. Esta classe deve ter um único método público calcular(salario_bruto) que recebe o salário bruto e retorna o valor total do INSS retido, com base na tabela progressiva fornecida.

    Dica (Conceito da Aula 04): Estamos modelando uma relação "parte-de" (Composição). O CalculadorINSS é uma "parte" essencial do CalculoCLT. A classe CalculadorINSS não precisa de um __init__ complexo, ela pode ter apenas o método calcular (ou pode ser um método estático, se preferir).

    Dica (Lógica): Você precisará de vários if/elif/else para calcular o valor de cada faixa e somá-los, conforme o exemplo na tabela de INSS.

  5. Questão 5: Integrando o Componente INSS (Composição)

    Modifique a classe CalculoCLT:

    1. No __init__, crie uma instância do CalculadorINSS e armazene-a em um atributo (ex: self._calculador_inss = CalculadorINSS()). Isso é a Composição!
    2. Crie um método privado _calcular_inss.
    3. Dentro de _calcular_inss, use o objeto composto (self._calculador_inss) para calcular o imposto (self._calculador_inss.calcular(self.salario_bruto)) e armazene o resultado em self.inss_retido.
    4. Adicione a chamada self._calcular_inss() dentro do método processar_calculo().
    Dica: A classe CalculoCLT não sabe como o INSS é calculado, ela apenas delega essa tarefa ao seu componente (CalculadorINSS). Isso torna o código muito mais limpo e fácil de manter.

  6. Questão 6: Cálculo da Base do IRPF

    Agora que temos o INSS, podemos calcular a base do IRPF.

    Crie dois métodos privados em CalculoCLT:

    1. _calcular_deducao_dependentes: Deve calcular self.numero_dependentes * 189.59 e salvar em self.deducao_dependentes.
    2. _calcular_base_irpf: Deve calcular self.salario_bruto - self.inss_retido - self.deducao_dependentes e salvar em self.base_calculo_irpf.

    Adicione a chamada desses métodos ao processar_calculo(). Importante: A ordem importa! Eles devem ser chamados *depois* do cálculo do INSS.

    Dica: O método processar_calculo está se tornando um orquestrador. A ordem deve ser: _calcular_fgts, _calcular_inss, _calcular_deducao_dependentes, _calcular_base_irpf.

  7. Questão 7: Aplicando Composição (Aula 04) - Calculador de IRPF

    Similar à Questão 4, o cálculo do IRPF também é complexo. Vamos usar Composição novamente.

    Crie uma nova classe chamada CalculadorIRPF. Ela deve ter um método calcular(base_calculo) que recebe a base de cálculo (calculada na Q6) e retorna o valor do imposto retido, com base na tabela progressiva e nas parcelas a deduzir.

    Dica: A lógica aqui é diferente do INSS. Você encontra a faixa correta (if/elif/else) e aplica a fórmula: (base_calculo * aliquota) - parcela_a_deduzir.

  8. Questão 8: Integrando o Componente IRPF e Calculando o Salário Líquido

    Vamos finalizar os cálculos na classe CalculoCLT:

    1. No __init__, crie a instância do CalculadorIRPF (self._calculador_irpf = CalculadorIRPF()). (Composição!)
    2. Crie o método privado _calcular_irpf. Nele, use o componente (self._calculador_irpf.calcular(self.base_calculo_irpf)) e salve em self.irpf_retido.
    3. Crie o método privado _calcular_salario_liquido. Ele deve calcular self.salario_bruto - self.inss_retido - self.irpf_retido e salvar em self.salario_liquido.
    4. Adicione as chamadas _calcular_irpf() e _calcular_salario_liquido() ao método processar_calculo(), na ordem correta.
    Dica: Seu método processar_calculo agora deve orquestrar todos os passos, garantindo que um cálculo que depende de outro (ex: IRPF depende da Base, que depende do INSS) seja executado na sequência correta.

    Parte B: Dicas de Formatação do Relatório
    Como você pode tornar seu relatório (a string gerada na Questão 9) mais agradável e alinhado? O usuário do seu sistema (ex: um funcionário) precisa entender o que é desconto e o que é provento.

    Dica de Formatação (Parte B):

    f-strings permitem controlar o alinhamento. {valor:10.2f} alinha o número em 10 espaços. {texto:<20} alinha o texto à esquerda em 20 espaços. {texto:>20} alinha à direita.

    Você pode tentar um formato de "Holerite", separando Proventos (Salário Bruto) de Descontos (INSS, IRPF) e mostrando os Resumos (Líquido, FGTS).

    Exemplo de formatação avançada (tente implementar):

    def gerar_relatorio_formatado(self):
    # Definindo larguras
    desc_width = 30
    val_width = 12

    # Função auxiliar para formatar moeda
    def f_moeda(valor):
        return f"R$ {valor:>{val_width - 3},.2f}"

    # Montando a string
    rel = "=" * (desc_width + val_width + 1) + "\n"
    rel += f"{'Holerite de Pagamento':^{desc_width + val_width}}\n"
    rel += "=" * (desc_width + val_width + 1) + "\n"

    # Entradas
    rel += f"{'Salário Bruto:':<{desc_width}}{f_moeda(self.salario_bruto)}\n"
    rel += f"{'Dependentes:':<{desc_width}}{self.numero_dependentes:>{val_width}}\n"
    rel += "-" * (desc_width + val_width + 1) + "\n"
    
    # Descontos
    rel += f"{'Descontos':^{desc_width + val_width}}\n"
    rel += f"{'  (-) INSS Retido:':<{desc_width}}{f_moeda(self.inss_retido)}\n"
    rel += f"{'  (-) IRPF Retido:':<{desc_width}}{f_moeda(self.irpf_retido)}\n"
    rel += "-" * (desc_width + val_width + 1) + "\n"

    # Líquido
    rel += f"{'SALÁRIO LÍQUIDO:':<{desc_width}}{f_moeda(self.salario_liquido)}\n"
    rel += "=" * (desc_width + val_width + 1) + "\n"

    # Resumo (Valores da Empresa)
    rel += f"{'Resumo (Empresa)':^{desc_width + val_width}}\n"
    rel += f"{'FGTS (8%):':<{desc_width}}{f_moeda(self.fgts_empresa)}\n"
    rel += "-" * (desc_width + val_width + 1) + "\n"

    return rel
