APIs como "Contrato" entre sistemas

A Analogia do Contrato

O contrato especifica de forma clara os endpoints, métodos, formatos de dados e códigos de retorno.

O Consumidor (client) e o Provedor (server) negociam e interagem baseados estritamente nessa especificação.

Exemplo de padrão de mercado para documentação de contratos: OpenAPI (Swagger).

Boas Práticas

Versionamento: Manter retrocompatibilidade (ex: v1, v2).

Idempotência: Requisições repetidas devem gerar o mesmo resultado final.

Políticas de Autenticação: Garantir que apenas clientes autorizados acessem os dados.

Limites (Rate Limit): Proteger a API contra abusos ou sobrecarga de requisições.
