GET vs POST

Característica
GET
POST
Propósito
Leitura/Recuperação de dados
Criação de recursos ou execução de ações
Corpo (Body)
Não recomendado / Sem corpo por convenção
Envia dados (JSON, formulários, arquivos)
Parâmetros
Na URL (Query String: ?id=1&sort=asc)
No corpo da requisição (payload)
Cache / Idempotência
Pode ser cacheado / Idempotente (seguro)
Geralmente não cacheado / Não idempotente

# Exemplo cURL: GET

curl -i "https://jsonplaceholder.typicode.com/posts?userId=1"

# Exemplo cURL: POST

curl -i -X POST \
-H "Content-Type: application/json" \
-d '{"title":"foo","body":"bar","userId":1}' \
https://jsonplaceholder.typicode.com/posts
