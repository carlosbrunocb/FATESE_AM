Boas Práticas de URI

  • Use substantivos no plural para recursos.
    Correto: /clientes | Incorreto: /getCliente

  • Mantenha uma hierarquia clara e lógica.
    Exemplo: /clientes/{id}/pedidos

  • Use Path Params para IDs e Query Params para filtros.
    Prefira: /produtos?categoria=livros em vez de URLs complexas.

  • Utilize kebab-case (hífens) para legibilidade.
    Use /user-profiles e evite camelCase ou underscores.

  • Defina uma política consistente para a barra final (trailing slash).

  • Implemente o versionamento de forma explícita.
    URI (ex: /v1/) ou via Header para clientes maduros.
