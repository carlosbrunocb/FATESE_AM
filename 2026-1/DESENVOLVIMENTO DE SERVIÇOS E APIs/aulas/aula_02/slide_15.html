Boas Práticas Design de API

Padrões de URIs RESTful

Hierarquia

URIs devem refletir a estrutura dos dados. Use barras / para indicar hierarquia entre recursos pai e filho.

Filtragem

Não crie novas URLs para filtros. Use Query Parameters (?) para ordenação, paginação e buscas.

Antipattern vs Pattern

/getCliente?id=123

RPC Style (Evite)

/clientes/123

REST Style (Prefira)

api-design-patterns.http

# 1. Coleção: Listar todos os clientes
GET /v1/clientes
# 2. Recurso Único: Buscar cliente específico pelo ID
GET /v1/clientes/{clienteId}
# 3. Sub-recurso: Pedidos de um cliente específico
GET /v1/clientes/{clienteId}/pedidos
# 4. Filtragem complexa, ordenação e paginação
GET /v1/pedidos?status=aberto&sort=-data&page=2&limit=20
# 5. Busca parametrizada
GET /v1/produtos?categoria=livros&preco_min=50
