Aula 02 Arquiteturas de API

Verbos HTTP e CRUD

  • GET Leitura de recursos. É Seguro e Idempotente.

  • POST Criação de novos recursos. Geralmente Não Idempotente.

  • PUT Substituição total de um recurso. É Idempotente.

  • PATCH Atualização parcial. Modifica apenas os campos enviados.

  • DELETE Remoção de recursos. O efeito final é Idempotente.

  • HEAD / OPT Metadados de cabeçalho e negociação de capacidades.
