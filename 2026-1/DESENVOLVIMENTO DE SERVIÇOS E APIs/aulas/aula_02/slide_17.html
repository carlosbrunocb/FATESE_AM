Estudo de Caso Modelagem de API

Exemplos Práticos: Biblioteca

Recursos Principais

Identificamos as entidades principais do sistema como substantivos no plural: Livros e Usuários.

Relacionamentos

Para expressar relacionamentos, utilizamos a hierarquia da URI. Empréstimos pertencem a um usuário específico.

Antipattern vs Pattern

/buscarLivroPorId?id=10

Verbo na URI (Evite)

/livros/10

Identificador no Path (Prefira)

library-api.http
123456789101112131415
# 1. Catálogo: Listar todos os livros disponíveis
GET /v1/livros
# 2. Detalhe: Buscar dados de um livro específico
GET /v1/livros/{livroId}
# 3. Histórico: Listar todos os empréstimos de um leitor
GET /v1/usuarios/{usuarioId}/emprestimos
# 4. Ação: Realizar um novo empréstimo
POST /v1/emprestimos
{ "usuario_id": 123, "livro_id": 456 }
# 5. Processo: Registrar devolução (Atualização parcial)
PATCH /v1/emprestimos/{emprestimoId}
{ "status": "devolvido" }
UTF-8 HTTP/1.1 JSON Payload
