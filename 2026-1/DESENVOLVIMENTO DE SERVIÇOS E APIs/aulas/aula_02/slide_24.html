Aula 02 Arquiteturas de API

Ferramentas para a Prática

  • Postman / Insomnia

    Essenciais para realizar requisições HTTP manuais, organizar coleções, criar variáveis de ambiente e testes automatizados básicos.

  • Swagger / OpenAPI

    Padrão de mercado para documentação interativa. Permite visualizar endpoints, modelos de dados e testar chamadas diretamente no navegador.

  • SoapUI

    Ferramenta clássica e robusta focada em testes funcionais avançados, especialmente para serviços SOAP e validação de contratos WSDL.

  • WSDL Browser / Extensões IDE

    Úteis para inspeção rápida da estrutura rígida de contratos SOAP (Tipos, Mensagens, PortTypes) sem necessidade de ferramentas pesadas.

  • Dicas de Produtividade

    Utilize variáveis de ambiente para alternar entre dev/prod, salve exemplos de request/response e mantenha coleções versionadas.
