ARQUITETURA • V8

Arquitetura: Motor V8

O componente que compila JavaScript (JIT) e entrega performance ao Node.js.

Em uma frase

JIT + GC

Compilação: Just-In-Time (JIT)

Memória: Garbage Collector (GC)

Integração: bindings com APIs do Node

O que ele faz no Node.js

O V8 é o motor JavaScript do Google (Chrome) que interpreta e compila seu código para rodar com eficiência.

No Node.js, o V8 cuida do JavaScript; o restante do runtime (como libuv) cuida do I/O assíncrono.

JIT (TurboFan)

Otimiza caminhos “quentes” e gera código nativo.

Garbage Collector

Gerencia memória automaticamente (objetos e heap).

Heap & Stack

Estruturas de memória usadas durante a execução.

Bindings (C/C++)

Conectam JS às APIs do Node (FS, rede, etc.).

Observação importante

O V8 não “faz” o I/O assíncrono; ele executa JavaScript. O I/O é orquestrado pelo Node (libuv).
