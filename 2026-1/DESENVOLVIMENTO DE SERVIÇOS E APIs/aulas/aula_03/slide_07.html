ARQUITETURA • EVENT LOOP

Event Loop e I/O Não Bloqueante

Como o Node.js lida com muitas conexões sem “travar” esperando rede ou disco.

Em uma frase

JS + I/O

Thread principal: executa JS

I/O: delegado (libuv)

Retorno: callbacks/promises entram na fila

Ideia central

O Node.js usa um loop de eventos para coordenar tarefas assíncronas, mantendo o JavaScript responsivo.

Em vez de “esperar” rede/arquivo, a aplicação registra a operação e continua. Quando o resultado fica pronto, uma função é agendada para execução.

Fila & agendamento

Tarefas prontas entram em filas e são processadas por turnos.

Não bloqueante

O JS não fica “parado” aguardando o I/O terminar.

Ideal para APIs

Muitas requisições concorrentes com custo baixo de threads.

Atenção com CPU

Código pesado bloqueia o loop; use workers/filas se necessário.

Regra prática

Prefira APIs assíncronas (fs.promises, requisições HTTP, DB). Use versões sync apenas em scripts simples.

Fluxo simplificado

Execução assíncrona

1) JS dispara uma operação

request

Ex.: ler arquivo, consultar banco, chamar API externa

2) libuv processa o I/O

kernel/pool

Rede: eventos do SO

FS/DNS: thread pool

3) Callback/Promise é enfileirada

queue

Timers / I/O callbacks / microtasks

4) Event Loop executa o próximo

tick

O JS volta a rodar "quando tiver algo pronto".
