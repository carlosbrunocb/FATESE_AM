Node.js vs Navegador

Mesmo JavaScript, mas com APIs, globals e restrições bem diferentes.

Em 1 minuto

RUNTIME

Navegador: DOM + Web APIs

Node.js: FS + rede + processos

Implicação: código não é 100% portátil

Ambiente

Node.js

Servidor / CLI

Objeto global

Ex.: global

Possui process, Buffer, timers e APIs do runtime.

Acesso ao sistema

Core: fs, path

Pode ler/escrever arquivos, abrir sockets e criar servidores HTTP.

Módulos

CommonJS + ESM

Suporta require (CJS) e import (ESM), dependendo da configuração.

Modelo de segurança

Permissivo

Executa com permissões do usuário do SO (cuidado com dependências e credenciais).

Quando usar

APIs, microserviços, automação, CLIs, integrações e streaming.

Ambiente

Navegador

Front-end

Objeto global

Ex.: window

Ambiente com document, eventos de UI e APIs visuais.

DOM e Web APIs

fetch, storage

Manipula interface (DOM), faz requisições e usa APIs como fetch.

Módulos

ESM nativo

Suporte nativo a import/export (com regras de origem/URL).

Sandbox

Restrito

Sem acesso direto ao sistema de arquivos/local, com políticas como CORS e permissões.

Quando usar

Interfaces, SPA, consumo de APIs, renderização e interação com o usuário.
