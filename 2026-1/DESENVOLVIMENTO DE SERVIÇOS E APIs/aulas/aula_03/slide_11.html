NPM • MANIFESTO

package.json — Estrutura Básica

O “manifesto” do projeto: metadados, scripts e dependências que definem como sua aplicação roda.

Por que importa?

npm

Scripts: start, dev, test...

Dependências: libs do app

Reprodutibilidade: + lockfile

Cria: npm init -y

Exemplo mínimo (com campos mais comuns)

package.json

/meu-projeto/package.json

{
  "name": "meu-projeto",
  "version": "1.0.0",
  "description": "API de exemplo em Node.js",
  "main": "index.js",
  "type": "commonjs",
  "scripts": {
    "start": "node index.js"
  },
  "dependencies": {
    "date-fns": "^3.6.0"
  },
  "devDependencies": {
    "nodemon": "^3.1.0"
  }
}

Identidade

name, version, description

Automação

scripts (npm start, npm run dev)

Pacotes

dependencies e devDependencies

Entrada

main e type (CJS/ESM)

Guia rápido dos campos

o que cai na prática

name

string

Nome do pacote/projeto (usado pelo NPM).

version

semver

Versão do projeto (major.minor.patch).

scripts

obj

Atalhos para comandos: npm start, npm run dev.

dependencies

prod

Bibliotecas necessárias para rodar em produção.

devDependencies

dev

Ferramentas de desenvolvimento (lint, testes, reload).

Regra prática

Se você apagar node_modules, basta rodar npm install e o NPM recria tudo a partir do package.json (+ lockfile).
