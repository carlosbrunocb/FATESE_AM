NPM • DEPENDÊNCIAS

Dependências vs DevDependencies

Organize o que vai para produção e o que existe só no ambiente de desenvolvimento.

Em 1 minuto

package.json

dependencies: necessárias para rodar em produção

devDependencies: ferramentas de build/test/lint

Regra: app falha em prod? então é dependencies

Tipo

dependencies

Produção

Quando usar

Roda em prod

Bibliotecas que o app precisa para executar: servidor HTTP, validação, acesso a banco, manipulação de datas, autenticação, etc.

Instalação

Cmd

npm i express ou npm install date-fns

Exemplos

runtime

express, fastify, date-fns, jsonwebtoken, bcrypt

Sinal de erro comum

deploy

“Cannot find module” em produção após deploy — geralmente a lib foi colocada por engano em devDependencies.

Regra prática

Se o código importa no runtime, é dependencies.

Tipo

devDependencies

Desenvolvimento

Quando usar

Ferramentas

Ferramentas que ajudam a desenvolver e manter o projeto: lint, formatação, testes, hot reload, build, TypeScript.

Instalação

Cmd

npm i -D nodemon eslint

Exemplos

tooling

nodemon, eslint, prettier, jest, typescript

Impacto em produção

deploy

Em servidores, costuma-se instalar só deps de produção (ex.: npm ci --omit=dev). Por isso, ferramentas de dev não devem ser necessárias para o app rodar.

Regra prática

Se só é usado em scripts e pipeline, é devDependencies.
