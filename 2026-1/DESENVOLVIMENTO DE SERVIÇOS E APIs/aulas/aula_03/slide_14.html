NPM Scripts — Automação

Crie atalhos no package.json para rodar tarefas do projeto de forma padronizada.

Essencial no dia a dia

npm run

Produtividade: menos digitação e menos erros

Padronização: mesmo comando em toda máquina

Workflow: dev, lint, test, build

Atalho: npm start

Exemplo de scripts no package.json

scripts

/meu-projeto/package.json

{
  "name": "meu-projeto",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "lint": "eslint .",
    "test": "jest"
  }
}

start

Entrada padrão do projeto

dev

Reload automático (ex.: nodemon)

lint

Qualidade e padrão de código

test

Executa a suíte de testes

Como executar

comandos

Rodar o padrão

start

npm start (atalho para npm run start).

Rodar script custom

run

Use npm run dev, npm run lint…

Passar argumentos

--

npm run build -- --watch (o -- separa args do npm).

Ver scripts

ls

npm run lista os scripts disponíveis no projeto.

Dica prática

Combine scripts para criar um fluxo: lint → test → start. Em projetos reais, é comum ter também build e format.
