Gerenciador de Versões: NVM

Instale, troque e padronize versões do Node.js por projeto — evitando “funciona na minha máquina”.

Quando usar?

nvm

Múltiplas versões: LTS e atuais

Troca rápida: nvm use

Padrão por projeto: .nvmrc

Dica: nvm install --lts

Instalação e uso (comandos essenciais)

CLI

Terminal

# macOS / Linux (nvm-sh)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Recarregar a sessão (pode variar por shell)
source ~/.bashrc  # ou ~/.zshrc

# Instalar e usar a versão LTS
nvm install --lts
nvm use --lts

# Ver versões instaladas / ativa
nvm ls
node -v
npm -v

# Instalar e usar uma versão específica
nvm install 20
nvm use 20

# Definir padrão
nvm alias default --lts

Inventário

nvm ls / nvm ls-remote

Alternância

nvm use 18 | nvm use --lts

Por projeto

.nvmrc + nvm use

Padrão

nvm alias default

Guia rápido

boas práticas

.nvmrc

arquivo

Crie na raiz do projeto com a versão (ex.: 18 ou lts/*).

Windows

nvm-windows

No Windows, use o projeto nvm-windows (Corey Butler). Comandos são similares.

Troubleshooting

PATH

Se nvm não aparecer, reabra o terminal e confira seu arquivo de shell (.bashrc/.zshrc).

Reprodutibilidade

time

Fixe a versão do Node por projeto para evitar inconsistências em times e CI.

Fluxo recomendado

Instale a LTS, crie .nvmrc no projeto e use nvm use antes de rodar npm install.
