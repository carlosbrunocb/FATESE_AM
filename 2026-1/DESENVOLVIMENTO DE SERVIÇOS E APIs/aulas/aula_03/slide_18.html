NPM • CLI

Comandos Básicos do NPM

O essencial para iniciar projetos, instalar pacotes, inspecionar dependências e manter tudo em dia.

Dica de rotina

terminal

Rodar: npm start / npm run dev

Instalar: npm i (prod) e npm i -D (dev)

Auditar: npm audit e corrigir com critério

Atalho: npm i = npm install

“Cheat sheet” (comandos do dia a dia)

npm

Terminal

# iniciar um projeto
npm init -y

# instalar dependências
npm i express
npm i date-fns

# instalar dependências de desenvolvimento
npm i -D nodemon eslint prettier

# remover pacotes
npm uninstall date-fns

# ver o que está instalado
npm ls --depth=0

# checar atualizações
npm outdated
npm update

# segurança
npm audit
npm audit fix

Início rápido

npm init -y + scripts no package.json

Instalação

npm i / npm i -D / npm uninstall

Diagnóstico

npm ls, npm outdated, npm audit

Manutenção

update com cuidado + commits consistentes

Quando usar cada comando

boas práticas

npm install

setup

Baixa dependências a partir do package.json (e usa o lockfile se existir).

npm ci

CI

Instalação limpa e reprodutível baseada no package-lock.json (ideal em pipelines).

npm run <script>

dev

Executa scripts definidos no package.json (ex.: dev, test, lint).

npm audit

seg

Verifica vulnerabilidades. Use audit fix com cuidado e preferencialmente com testes.

npm outdated

ver

Mostra o que está desatualizado (atualize por etapas e registre em commits).

Regra prática

No laboratório: npm init -y → npm i date-fns → criar npm start para rodar o projeto.
