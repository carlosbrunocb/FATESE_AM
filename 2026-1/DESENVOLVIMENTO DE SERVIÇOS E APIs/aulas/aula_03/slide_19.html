PRÁTICA • PRIMEIRO PROJETO

Criando o Primeiro Projeto

Objetivo: inicializar um projeto Node.js, criar um arquivo index.js e executar no terminal.

Checklist rápido

node + npm

Criar pasta: meu-projeto

Manifesto: package.json

Executar: node index.js

Dica: use Git desde o início

Passo a passo (terminal)

Inicialize e execute

npm init

1

Crie a pasta e acesse

Organize seus arquivos em uma pasta dedicada ao projeto.

Terminal

mkdir meu-projeto
cd meu-projeto

mkdir

cd

2

Inicialize o projeto com npm

Gere o package.json com configurações padrão.

Terminal

npm init -y

Resultado esperado

Um arquivo package.json aparece na pasta do projeto.

3

Crie e execute o index.js

Escreva um script simples e rode com o Node.

index.js

console.log('Olá, Node.js!')

Terminal

node index.js

Critério de sucesso

Mensagem exibida no terminal sem erros.

Próximo: npm install

Boa prática

Após npm init, já crie o .gitignore e faça o primeiro commit.

Arquivos: package.json • index.js

Estrutura mínima do projeto

recomendado

Pasta

meu-projeto/
  package.json
  index.js
  node_modules/   # após instalar pacotes

Observação

O node_modules só aparece depois do primeiro npm install.

Comandos úteis (opcional)

Git

Se estiver usando versionamento desde o início:

Terminal

git init
echo node_modules/ > .gitignore
git add .
git commit -m "chore: init projeto node"

Por que isso ajuda?

Mantém o repositório limpo e evita subir dependências.
