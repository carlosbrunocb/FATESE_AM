Instalando Pacotes Externos (date-fns)

Como adicionar bibliotecas ao projeto e importar funções para resolver problemas comuns (datas e horários).

Checklist rápido

npm install

Instalar: npm i date-fns

Importar: require (CJS) ou import (ESM)

Executar: node index.js

Dica: cheque o package.json

Instalação + uso no index.js (CommonJS)

date-fns

Terminal + /meu-projeto/index.js

# 1) Instale a dependência (produção)
npm i date-fns

# 2) Execute seu arquivo
node index.js

// index.js (CommonJS)
const { format } = require('date-fns')

const agora = new Date()
const dataFormatada = format(agora, 'dd/MM/yyyy HH:mm')

console.log('Agora:', dataFormatada)

Guia prático: importação e organização

na vida real

Onde registrar

package.json

Após npm i date-fns, o pacote entra em dependencies.

CommonJS vs ESM

type

Se "type": "module", use import { format } from 'date-fns'.

Importe só o que usar

tree-shaking

Prefira { format } em vez de importar tudo: ajuda na organização e no tamanho do código.

Versões e ranges

semver

A versão aparece como ^x.y.z. Atualize com cuidado e teste.

Reprodutibilidade

package-lock

O package-lock.json registra versões exatas instaladas na sua máquina.

Dica de laboratório

Se o require falhar por causa de ESM, confira o campo type no package.json e ajuste o padrão de importação.
