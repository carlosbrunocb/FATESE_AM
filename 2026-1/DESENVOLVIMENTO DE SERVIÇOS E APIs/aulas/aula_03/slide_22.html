node_modules e .gitignore

Evite commitar dependências: mantenha o repositório leve, reprodutível e seguro.

Ideia central

repo limpo

node_modules: gerado localmente

.gitignore: define o que não versionar

lockfile: garante versões exatas

Recria: npm install

Configuração mínima para projetos Node

.gitignore

/meu-projeto/.gitignore

# Dependências instaladas localmente
node_modules/

# Logs
npm-debug.log*
yarn-debug.log*
yarn-error.log*

# Variáveis de ambiente
.env

# Build/Cache (quando existir)
dist/
.cache/

Por que ignorar?

Pesado, muda a todo momento e é gerado automaticamente.

Como recriar

Use npm install a partir do package.json (+ lockfile).

Reprodutibilidade

Commite o package-lock.json para instalar as mesmas versões.

Dica

Se você commitou por engano, remova com git rm -r --cached node_modules.

Checklist de versionamento

o que vai pro Git

Commitar

sim

package.json, package-lock.json, src/, README.md.

Ignorar

não

node_modules/, .env, logs e artefatos de build.

.env

segredo

Nunca publique credenciais. Use .env.example para documentar variáveis.

CI/CD

dica

Prefira npm ci em pipelines: instala a partir do lockfile, mais previsível.

Regra prática

Se o arquivo pode ser gerado (dependências, build, cache), ele tende a ficar fora do Git. Versione apenas o que define e documenta o projeto.
