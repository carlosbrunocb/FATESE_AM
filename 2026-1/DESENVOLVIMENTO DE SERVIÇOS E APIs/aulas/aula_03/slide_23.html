NPM • LOCKFILE

package-lock.json

O arquivo que registra as versões exatas instaladas e torna builds reprodutíveis.

Resumo rápido

REPRODUÇÃO

Garante: versões idênticas

Ajuda: rastrear integridade

Base do: npm ci

O que é

Um “lockfile” gerado automaticamente que descreve a árvore completa de dependências.

Ele complementa o package.json: enquanto o manifesto aceita faixas de versão (ex.: ^1.2.3), o lockfile registra o que foi realmente instalado.

Reprodutibilidade

Mesmas versões em todas as máquinas (dev, CI e produção).

Integridade

Registra metadados (ex.: origem e checks) para instalação confiável.

Trabalho em equipe

Evita “na minha máquina funciona”.

Deve ir ao Git

Versione o lockfile junto com o código do projeto.

Regra prática

Use npm install no dia a dia e npm ci em CI para instalações rápidas e determinísticas.

Como ele muda o fluxo

npm install / npm ci

Comparando

// package.json (faixa)
"date-fns": "^3.0.0"

// package-lock.json (exato)
"date-fns": "3.6.0"

Boas práticas

Commitar o package-lock.json

Mantém o time e o CI sincronizados.

Evitar editar manualmente

Deixe o NPM atualizar ao instalar/remover pacotes.

Atualizar com controle

Revise mudanças em PR e teste antes de publicar.

Em uma frase

“O lockfile fixa o que foi instalado para o projeto ser reprodutível e confiável.”
