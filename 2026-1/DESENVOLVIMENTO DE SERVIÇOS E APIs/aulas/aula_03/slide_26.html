Atividade Prática 2 — Primeiro Script e NPM

Objetivo: instalar uma biblioteca, criar um script Node.js e automatizar a execução com npm start.

Entregáveis

OK

Projeto: package.json + index.js

Pacote: date-fns instalado

Execução: npm start

Critério: data formatada no terminal

Passo a passo

Do zero até o npm start

Terminal + VS Code

1

Criar (ou usar) um projeto Node

Na pasta do projeto, inicialize o manifesto (se ainda não existir).

Terminal

mkdir aula03-date-fns
cd aula03-date-fns
npm init -y

2

Instalar a biblioteca date-fns

Adicione a dependência que fará a formatação da data.

Terminal

npm i date-fns

dependencies

node_modules/

package-lock.json

3

Criar o index.js e imprimir a data

Use a função format para mostrar a data no padrão dd/MM/yyyy.

index.js

const { format } = require('date-fns')

const hoje = new Date()
console.log(format(hoje, 'dd/MM/yyyy'))

4

Criar o script start no package.json

Adicione (ou ajuste) a seção scripts para executar o arquivo.

package.json

{
  "scripts": {
    "start": "node index.js"
  }
}

Executar e validar

Rode npm start e verifique a data formatada no terminal.

Comando: npm start

Dicas rápidas

boa prática

CommonJS (require)

O exemplo usa require(). Isso funciona no Node por padrão em projetos CommonJS.

Formato de data

Use 'dd/MM/yyyy' (dia/mês/ano). Letras maiúsculas mudam o significado no date-fns.

Erros comuns

Se npm start falhar: confira o nome do arquivo (index.js) e o script no package.json.

Checklist de sucesso

validação

1) date-fns aparece em dependencies

2) Existe index.js na raiz do projeto

3) npm start imprime uma data como 11/03/2026

Comando final 

npm start
