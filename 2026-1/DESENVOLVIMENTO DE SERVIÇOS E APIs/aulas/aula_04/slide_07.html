Arquitetura em Camadas

Separando responsabilidades para escalar: Routes, Controllers e Services

Camada de Rotas

/routes

Responsável apenas por definir os endpoints (URLs) e os métodos HTTP (GET, POST, etc.). Ela recebe a requisição inicial e a encaminha para o Controller correspondente, sem conter lógica.

Camada de Controllers

/controllers

Atua como o orquestrador. Recebe a requisição, valida os dados de entrada, chama os serviços necessários e decide qual Status Code e formato de resposta devolver ao cliente.

Camada de Services

/services

Onde reside a inteligência. Contém toda a Regra de Negócio e interações com banco de dados. É agnóstica ao protocolo HTTP, facilitando testes unitários e reutilização.

Principais Benefícios

Código desacoplado, facilidade de manutenção, testes simplificados e possibilidade de trocar a interface (ex: Web para CLI) sem alterar a regra de negócio.
