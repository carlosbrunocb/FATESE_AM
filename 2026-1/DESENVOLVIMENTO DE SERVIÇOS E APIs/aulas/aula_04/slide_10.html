Business Logic
src/services/tasks.service.js
01// Simulação de Banco de Dados (Memória)
02let tasks = [];
03let nextId = 1;
04
05function create(data) {
06 // Regra de negócio: criar objeto Task
07 const newTask = {
08 id: nextId++,
09 title: data.title,
10 done: false
11 };
12
13 // Persistência (simulada)
14 tasks.push(newTask);
15 return newTask;
16}
17
18function list() {
19 return tasks;
20}
21
22module.exports = { create, list };

Conceitos Chave

Inteligência da Aplicação

Aqui residem as validações complexas, cálculos e interações com o banco de dados. É o "como fazer" da aplicação.

Agnóstico ao Framework

Observe que não há req ou res. O Service é puro JavaScript e pode ser reutilizado em outros contextos (CLI, Jobs, Testes).

Testabilidade

Por serem funções independentes da camada HTTP, são muito mais fáceis de testar com Testes Unitários.
