Testando a API

Validação via Terminal (cURL) ou Ferramentas Gráficas

GET / POST / PUT
bash / terminal
$npm run dev
>Server running on port 3000...
# 1. Listar Tarefas (GET)
$curl -s http://localhost:3000/tasks
>[ {"id":1,"title":"Estudar Express","done":false} ]
# 2. Criar Tarefa (POST) - Header JSON obrigatório
$curl -X POST http://localhost:3000/tasks \
-H 'Content-Type: application/json' \
-d '{"title":"Testar API","done":true}'
>{"id":2,"title":"Testar API","done":true}
# 3. Teste de Erro (400 Bad Request)
$curl -X POST http://localhost:3000/tasks -d '{}'
>{"message":"title é obrigatório"}

Pontos de Atenção

Servidor em Execução

Antes de testar, garanta que o comando npm run dev (ou node app.js) esteja rodando em um terminal separado.

Postman & Insomnia

Embora o cURL seja rápido, ferramentas visuais como Postman facilitam criar collections, salvar requisições e visualizar JSON formatado.

Header Content-Type

Para POST e PUT, é obrigatório enviar o header Content-Type: application/json para que o express.json() consiga ler o req.body.
