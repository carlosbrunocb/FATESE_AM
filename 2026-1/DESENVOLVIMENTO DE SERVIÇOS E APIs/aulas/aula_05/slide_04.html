APIs REST com Express.js

Visão geral do ciclo: Request → Middleware → Controller → Response

REST + JSON

Express.js

Fluxo de uma requisição

Como o Express recebe dados, processa e devolve uma resposta consistente.

Stateless (sem estado)

Request

Cliente → API

req

URL & rota

/api/livros/2

Parâmetros

params, query, body

Verbo HTTP

GET • POST • PUT • DELETE

Middleware

Pré-processamento

next()

Parse do JSON

express.json()

Exemplos comuns

logs, autenticação, validação, CORS

Objetivo

Padronizar e proteger o fluxo

Controller

Regra de negócio

res

Extrair dados

req.params • req.query • req.body

Validação básica

campos obrigatórios, ID existente

Retorno

res.status().json()

Response

API → Cliente

JSON

Status codes

200 • 201 • 204 • 400 • 401 • 404

Formato

Content-Type: application/json

Contrato

Resposta previsível para o frontend

Dica prática

Mantenha o server.js apenas para subir o servidor e plugar rotas; deixe a regra de negócio no controller.

Exemplo de endpoint

Requisição

GET /api/livros/2

Route param

req.params.id → "2"

Resposta

200 + { id, titulo, autor, ano }

Princípios REST (essenciais)

Endpoints previsíveis

Recursos com URIs claras: /livros, /livros/:id.

Verbo certo, efeito certo

GET lê, POST cria, PUT atualiza, DELETE remove.

Representação em JSON

Dados trafegam como objetos, com status codes para indicar resultado.
