Parâmetros de requisição: panorama

Onde os dados chegam na API: URL (route/query) e corpo (JSON)

Express.js

req.params • req.query • req.body

Três formas comuns de enviar dados

Escolha pelo propósito

Route Params (req.params)

Identificam um recurso específico na própria rota (geralmente um ID).

GET /livros/:id

req.params.id

Query Params (req.query)

Usados para filtro, busca, ordenação e paginação.

GET /livros?autor=Orwell&pagina=2

req.query.autor

Body Params (req.body)

Dados no corpo (normalmente JSON) para criar ou atualizar recursos.

POST /livros

req.body.titulo

Como decidir?

Identificar um item

Use Route Param (ex.: /livros/2).

Filtrar/Buscar/Listar

Use Query Param (ex.: ?autor=...).

Enviar dados “grandes”

Use Body para JSON (POST/PUT/PATCH).

Erros comuns (e como evitar)

Body vazio sem middleware

Inclua app.use(express.json()).

Tipos (string vs número)

Converta ID: parseInt(req.params.id).

Sem validação

Retorne 400 para campos obrigatórios ausentes.

Boa prática

Status + JSON consistente
