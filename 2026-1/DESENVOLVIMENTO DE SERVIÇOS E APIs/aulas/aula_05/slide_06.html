Route Params (req.params)

Parâmetros na URL para identificar um recurso específico

REST • rotas

Express.js

Quando usar e como extrair

Boa prática

Identificar um recurso

Use em GET/PUT/DELETE para localizar um item: /livros/2.

Extração no Express

O Express monta um objeto: req.params (strings por padrão).

Validação mínima

Converta e valide: parseInt(id) e retorne 404 se não existir.

Padronização

Mantenha nomes claros: :id, :livroId — e sempre documente.

Exemplos de URL

Detalhe de livro

GET /api/livros/2

Atualização

PUT /api/livros/2

Exclusão

DELETE /api/livros/2

Snippet (Express)

Rota + leitura do parâmetro

app.get('/livros/:id', (req, res) => {
  const id = parseInt(req.params.id)
  // buscar e responder...
  return res.status(200).json({ id })
})
