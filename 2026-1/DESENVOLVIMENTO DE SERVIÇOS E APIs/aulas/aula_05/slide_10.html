Express.js

O que um Controller deve fazer

Separação de camadas

Receber e extrair dados do req

Capturar params, query e body de forma clara (desestruturação).

Validar o básico antes de processar

Checar campos obrigatórios, tipos e formatos simples; retornar 400 quando faltar/invalidar.

Aplicar regras de negócio e decisões

Ex.: encontrar recurso, impedir operações inválidas, montar o objeto final a ser retornado (ou persistido).

Responder via res.status().json()

Padronizar mensagens e status: 200, 201, 204, 404.

Anti‑padrões comuns

Acoplar retorno ao “formato do front”

Prefira contratos previsíveis: { mensagem, dados } (quando fizer sentido).

Ignorar status code

Usar 200 para tudo dificulta debug e testes.

Mutação sem checar existência

Em PUT/DELETE, trate 404 quando o recurso não existir.
