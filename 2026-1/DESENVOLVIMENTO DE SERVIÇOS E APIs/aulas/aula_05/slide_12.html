Status Codes HTTP essenciais

Como sinalizar resultado da operação para o cliente (frontend/mobile)

REST

Express.js

Quando usar (com exemplos de endpoints)

Prático

2xx

Sucesso

200 OK (GET/PUT com retorno), 201 Created (POST criou), 204 No Content (DELETE ou operação sem corpo).

4xx

Erros do cliente

400 Bad Request (validação), 401 Unauthorized (credenciais), 404 Not Found (recurso não existe), 409 Conflict (estado conflitou).

5xx

Erros do servidor

500 Internal Server Error (erro inesperado). Evite vazar detalhes; registre logs no servidor.

Padrão de resposta recomendado

res.status(code).json({ mensagem, dados }) para erros e sucesso com corpo. Use res.status(204).send() quando não houver conteúdo.

Guia rápido

GET (lista/detalhe)

200 / 404

POST (criar)

201 / 400 / 409

PUT (atualizar)

200 / 400 / 404

DELETE (remover)

204 / 404

Cuidado comum

Não retorne 200 para tudo. O status code ajuda o cliente a tomar decisões (ex.: mostrar erro de validação vs. “não encontrado”).

Em 204, não envie JSON no corpo da resposta.

Em 500, registre logs e retorne mensagem genérica.
