Projeto Livraria: server.js (ponto de entrada)

Subindo o servidor, habilitando JSON e plugando rotas em /api

Express.js

/api

Arquivo de inicialização

Sem regra de negócio: apenas middlewares, rotas e listen.

Pronto para rodar

src/server.js

const express = require('express')
const livroRoutes = require('./routes/livroRoutes')

const app = express()
const PORT = 3000

// Middleware nativo para reconhecer requisições com corpo em JSON
app.use(express.json())

// Plugar as rotas da livraria no caminho base '/api'
app.use('/api', livroRoutes)

app.listen(PORT, () => {
  console.log(`Servidor rodando na porta ${PORT}`)
  console.log(`Acesse: http://localhost:${PORT}/api/livros`)
})

Boa prática

Mantenha o server.js “magro”: sem find, push ou regras — isso fica no controller.

Como executar

Modo desenvolvimento

npm run dev

Modo produção (local)

npm start

Dependências

Instale: express e nodemon (dev).

Endpoints iniciais

Listar livros

GET /api/livros

Buscar por ID

GET /api/livros/:id

Criar

POST /api/livros

Dica: no Postman/Insomnia, para POST/PUT use Body → raw → JSON e header Content-Type: application/json.
