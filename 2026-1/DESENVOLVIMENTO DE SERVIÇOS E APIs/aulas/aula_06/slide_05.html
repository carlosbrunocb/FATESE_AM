Anatomia de um Middleware

req (Objeto de Requisição)

Representa a solicitação HTTP. Contém propriedades essenciais como parâmetros de rota, query strings, cabeçalhos (headers) e o corpo da requisição (body).

res (Objeto de Resposta)

Representa a resposta HTTP que será enviada. Fornece métodos para definir status codes (404, 200), enviar JSON ou renderizar views.

next (Função de Controle)

Função de callback que passa o controle para o próximo middleware da pilha. Se não for chamada, a requisição ficará "pendurada" sem resposta.
