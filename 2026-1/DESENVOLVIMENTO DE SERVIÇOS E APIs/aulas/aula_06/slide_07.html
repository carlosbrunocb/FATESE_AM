A Função next()

Encadeamento Obrigatório

O next() é a "cola" que une os middlewares. Se sua função não envia uma resposta (res.send), ela deve chamar next() para evitar que a requisição fique travada (hanging).

Cuidado com Múltiplas Respostas

Chamar next() não encerra a execução da função atual. Use return next() ou return res.send() para garantir que o código pare ali e evitar erros de "Headers already sent".

Sinalização de Erros

Se você passar qualquer argumento para next(err) (exceto 'route'), o Express pulará todos os middlewares normais e irá direto para o Middleware de Tratamento de Erros.

Exemplo de Uso
const validaToken = (req, res, next) => { if (!req.headers.token) { // Erro: pula para o error handler return next(new Error('Sem token!')); } // Sucesso: passa para o próximo console.log('Token OK'); next(); }; app.use(validaToken); app.get('/', (req, res) => { res.send('Acesso permitido'); });
