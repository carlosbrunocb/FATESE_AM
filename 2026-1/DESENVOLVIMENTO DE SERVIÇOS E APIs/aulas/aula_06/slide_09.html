Comparativo de Escopo

Tipos de Middleware no Express

Diferenciando o escopo de execução global vs específico.

VS

Nível de Aplicação

Escopo Global (app.use)

  • Executado em todas as requisições que chegam ao servidor, independentemente da rota.

  • Ideal para configurações gerais: parsing de JSON, CORS, Logs, Segurança (Helmet).

  • Define o comportamento padrão da API inteira.

// Aplica a TODAS as rotas
app.use(logger);

// Aplica a caminhos específicos
app.use('/api', apiRoutes);

Nível de Roteador

Escopo Específico (router.use)

  • Limitado a um grupo de rotas específico ou a uma única rota.

  • Ideal para regras de negócio específicas: Validação de dados, Autenticação de admin.

  • Permite modularizar a aplicação em mini-apps independentes.

const router = express.Router();

// Só afeta rotas deste router
router.use(authMiddleware);

// Apenas nesta rota específica
router.post('/', validaLivro, criar);
