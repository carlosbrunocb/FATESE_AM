Middleware de Tratamento de Erros

Assinatura Especial (4 Parâmetros)

Diferente dos outros middlewares, este exige 4 argumentos: (err, req, res, next). O Express usa a contagem de argumentos para identificá-lo como um handler de erro.

Ordem de Execução Crítica

Deve ser definido após todas as rotas e outros middlewares. Se for definido antes, ele não conseguirá capturar erros lançados pelas rotas subsequentes.

Centralização e Padronização

Permite centralizar a lógica de logs e garante que o cliente receba uma resposta JSON formatada (ex: status 500) em vez de travar a requisição ou expor o stack trace.

Global Error Handler

Exemplo de implementação:

const errorHandler = (err, req, res, next) => { // Loga o erro no servidor console.error(err.stack); // Responde ao cliente res.status(500).json({ erro: 'Erro Interno', msg: 'Tente novamente mais tarde.' }); }; // Uso no app (SEMPRE POR ÚLTIMO) app.use(errorHandler);

Dica: Use next(err) nos seus controllers para encaminhar erros para este middleware.
