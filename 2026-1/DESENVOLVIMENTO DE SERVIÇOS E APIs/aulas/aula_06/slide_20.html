// Middleware Especial: recebe 4 argumentos
const errorHandler = (err, req, res, next) => {
// 1. Loga o erro para o desenvolvedor (console do servidor)
console.error(err.stack);
// 2. Retorna uma resposta segura para o cliente
res.status(500).json({
erro: "Erro Interno no Servidor",
mensagem: "Algo deu errado. Tente novamente mais tarde."
});
};
module.exports = errorHandler;

4 Parâmetros Obrigatórios

O Express só reconhece como tratamento de erro se a função tiver exatamente 4 argumentos: (err, req, res, next).

Segurança na Resposta

Nunca envie err.message ou stack trace diretamente ao usuário em produção. Use mensagens genéricas para evitar vazar dados.

Log do Desenvolvedor

Use console.error (ou libs como Winston) para registrar o erro real no servidor e facilitar a depuração posterior.
