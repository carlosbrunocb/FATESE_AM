Testes com Postman / Insomnia

Plano de Testes Manual

POST Simular Dados Inválidos

Enviar requisição sem o campo obrigatório 'titulo'. Esperar status 400 Bad Request e JSON descritivo.

GET Conferir Logs no Console

Fazer qualquer requisição e verificar no terminal do servidor se o formato [DATA] MÉTODO URL foi impresso.

Simular Erro Interno

Forçar um erro (ex: throw new Error) em uma rota temporária. Verificar se retorna 500 Internal Server Error padronizado.

Dica: Testes Automatizados

Para projetos reais, utilize bibliotecas como Supertest e Jest para automatizar essas verificações.

Postman - New Request
POST http://localhost:3000/api/livros
Body (JSON)
"autor": "George Orwell", "ano": 1949 // Faltando "titulo"
Response
400 Bad Request
{
"erro": "Dados inválidos",
"mensagem": "Os campos 'titulo', 'autor' e 'ano' são obrigatórios."
}
