Middlewares são poderosos, mas exigem cuidado

VS

O Que Fazer

Modularização por Domínio

Separe seus middlewares em arquivos pequenos e focados. Evite funções gigantescas. Tenha um middleware para Log, outro para Auth, outro para Validação.

Use Bibliotecas de Validação

Não reinvente a roda com if/else. Use bibliotecas robustas como Zod, Joi ou Yup para validar esquemas de dados complexos.

Tratamento de Erros Async

Em funções assíncronas, use try/catch e chame next(err) no catch. Ou use a lib express-async-errors para evitar try/catch repetitivo.

Segurança em Camadas

Aplique middlewares de segurança como Helmet (headers HTTP) e Rate Limit (anti-DDOS) logo no início da aplicação.

O Que Evitar (Pitfalls)

Esquecer o next()

Se você não responde (res.send) e não chama next(), a requisição fica "pendurada" (hanging) até dar timeout no cliente.

Headers Already Sent

Erro clássico: enviar uma resposta res.json() e depois tentar chamar next() ou enviar outra resposta. O ciclo deve acabar na resposta.

Ordem Incorreta

Colocar o middleware de 404 antes das rotas, ou o Error Handler no início. A ordem de definição no código é a ordem de execução!

Bloquear o Event Loop

Evite processamento síncrono pesado (ex: criptografia complexa, loops gigantes) no middleware, pois isso trava todas as outras requisições.
