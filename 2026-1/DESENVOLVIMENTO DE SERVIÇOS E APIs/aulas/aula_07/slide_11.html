Quando usar JSON ou XML?
Critérios de decisão, casos de uso e estratégias recomendadas
Aula 07
Formatos de Dados
Critérios de Decisão
Use JSON quando:
• Latência importa (mobile, SPAs)
• Consumidores web/JS predominam
• APIs REST públicas
• Observabilidade simples
Use XML quando:
• Precisa de contratos rígidos (XSD)
• Integrações legadas/SOAP
• Documentos ricos
• Assinaturas digitais
Estratégia Recomendada
• JSON por padrão + Content Negotiation para XML quando necessário
• Conversões na camada de Serviços (separação de responsabilidades)
• Cache de resultados convertidos para performance
Checklist de Decisão
Quem consome? Ferramentas disponíveis?
Requisitos legais? Auditoria necessária?
Transformação? XSLT/XPath necessário?
Compatibilidade? Sistemas legados?
Performance? Payload, parsing?
