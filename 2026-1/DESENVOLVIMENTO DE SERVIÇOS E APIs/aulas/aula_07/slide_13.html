O que é Content Negotiation?
Mecanismo HTTP para negociação de formato de dados entre cliente e servidor
Aula 07
Content Negotiation
Conceito e Funcionamento
Content Negotiation é o mecanismo HTTP pelo qual o cliente informa ao servidor quais formatos de dados ele aceita receber, permitindo que o servidor escolha a melhor representação disponível.
Cliente
Envia requisição
Servidor
Analisa Accept
Resposta
Formato negociado
Accept Header: Cliente informa preferências (application/json, application/xml)
Content-Type: Servidor responde com formato escolhido
406 Not Acceptable: Retornado quando formato não é suportado
Headers HTTP Importantes
Exemplo de requisição GET /api/livros
GET /api/livros HTTP/1.1
Host: localhost:3000
Accept: application/json, application/xml
Accept-Language: pt-BR
Por que é Importante?
Permite que APIs REST sejam flexíveis e atendam a diferentes tipos de clientes.
Flexibilidade: Suporte a múltiplos formatos
Compatibilidade: Integração com legados
Evolução: Mudanças sem quebrar clientes
Boas Práticas
Padrão JSON: Defina como formato padrão
Documentação: Liste formatos suportados na API
Vary Header: Inclua no cabeçalho para controle de cache
Validação: Verifique sempre o Accept header enviado
