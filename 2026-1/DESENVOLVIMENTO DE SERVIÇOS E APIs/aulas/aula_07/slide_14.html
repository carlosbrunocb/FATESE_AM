Content Negotiation
Headers HTTP - Funcionamento
Accept (Requisição)
Indica formatos aceitos na resposta
Accept: application/json
Content-Type (Resposta)
Descrição do formato do corpo
Content-Type: application/xml
// Exemplo de requisição com Accept
GET /api/livros HTTP/1.1
Host: localhost:3000
Accept: application/json, application/xml
Accept-Language: pt-BR
 
// Resposta do servidor
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Exemplos com curl
# Requisição JSON
curl -H "Accept: application/json" http://localhost:3000/api/livros
 
# Requisição XML
curl -H "Accept: application/xml" http://localhost:3000/api/livros
Implementação Express
const listarLivros = (req, res) => {
  const acceptHeader = req.headers['accept'];
   
  if (acceptHeader === 'application/xml') {
    const xmlData = livroService.listarTodosComoXML();
    res.set('Content-Type', 'application/xml');
    return res.status(200).send(xmlData);
  }
   
  // Comportamento padrão: JSON
  const livros = livroService.listarTodos();
  res.status(200).json(livros);
};
Boas Práticas
Validação: Verifique Content-Type de entrada
Charset: Sempre defina UTF-8
Fallback: JSON por padrão
Erro 406: Quando não suportado
