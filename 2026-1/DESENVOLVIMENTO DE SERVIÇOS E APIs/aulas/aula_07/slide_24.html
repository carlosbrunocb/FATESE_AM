Cenários de Teste
GET /api/livros (JSON) 200 OK
Content-Type: application/json Payload: Lista de livros
GET /api/livros (XML) 200 OK
Accept: application/xml Payload: XML formatado
GET /api/livros/fornecedor 200 OK
Integração SOAP → JSON Dados normalizados
Validações Importantes
Status Code: Verificar 200, 201, 404, 406
Headers: Content-Type, Accept, Vary
Payload: Estrutura e tipos de dados
Comandos cURL
# Teste JSON (padrão)
curl -i -H "Accept: application/json" \
http://localhost:3000/api/livros
 
# Teste XML
curl -i -H "Accept: application/xml" \
http://localhost:3000/api/livros
 
# Teste fornecedor
curl -i http://localhost:3000/api/livros/fornecedor
Checklist de Testes
Status Code
200 OK

application/json

< 500ms

< 10KB
