Por que usar Async/Await?
O acesso ao banco de dados é uma operação de I/O (Input/Output) que leva tempo. Usar async/await evita bloquear o event loop do Node.js, permitindo que outras requisições sejam processadas enquanto aguardamos a resposta do banco.
Exemplo de Controller com Async/Await
const livroService = require('../services/livroService'); const listarLivros = async (req, res, next) => { try { const livros = await livroService.listarTodos(); res.status(200).json(livros); } catch (error) { next(error); } };
Benefícios do Async/Await
Código mais limpo e legível
Tratamento de erros simplificado
Execução sequencial clara
Melhor debugging

Pontos de Atenção
• Sempre use try/catch para tratar erros
• Use await em chamadas ao banco
• Não esqueça de marcar funções como async
• Centralize erros com next(error)
