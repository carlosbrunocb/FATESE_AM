Pattern: Todas as funções são async e usam await para operações do Service
Boas Práticas
Try/Catch: Sempre use try/catch para capturar erros e repassar com next(err)
Async/Await: Evite callbacks, use async/await para código mais limpo
Tratamento de Erros: Retorne status HTTP adequados (404, 500, etc.)
Fluxo da Requisição
1. Cliente HTTP → Router
2. Controller (async) → Service
3. Service → Model → DB
