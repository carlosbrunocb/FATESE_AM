SQL - Schema Rígido
ALTER TABLE & Migrações
Rígido
ALTER TABLE necessário
Para adicionar novos campos, é necessário executar ALTER TABLE, o que pode causar downtime em bases grandes.
Migrações complexas
Cada mudança no schema requer migrações planejadas, janelas de manutenção e testes extensivos.
-- Adicionar coluna em tabela SQL
ALTER TABLE livros 
ADD COLUMN formato_digital BOOLEAN DEFAULT FALSE;

-- Migração de dados
UPDATE livros SET formato_digital = 'false' 
WHERE formato_digital IS NULL;
NoSQL - Schema Flexível
Adição Dinâmica de Campos
Flexível
Sem ALTER TABLE
Novos campos podem ser adicionados dinamicamente no código da aplicação sem modificações no banco.
Schema evolutivo
Documentos na mesma coleção podem ter estruturas diferentes, permitindo evolução rápida e iterativa.
// Adicionar campo dinamicamente no MongoDB
db.livros.updateOne(
  { _id: "123" },
  { $set: { formatoDigital: true, metadata: { fonte: "promo" } }}
);

// Ou com Mongoose (strict: false)
await Livro.findByIdAndUpdate(id, 
  { $set: { formatoDigital: true }}, 
  { new: true, strict: false }
);
Evite drift de schema com validações na borda da aplicação (DTOs e Schema Validation)
