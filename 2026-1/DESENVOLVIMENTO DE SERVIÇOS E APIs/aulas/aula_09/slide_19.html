Best Practices
Embed vs Referência: Quando Usar
Escolha a abordagem correta para cada cenário
Embed (Incorporar)
Quando usar:
• 1:N pequeno
• Leitura conjunta frequente
• Alta coesão do agregado
• Escritas atômicas
Referência
Quando usar:
• N grande
• Reuso alto
• Acessos independentes
• Dados compartilhados
One-to-Few:
Embed avaliações curtas em livros (poucos documentos, acesso conjunto)
One-to-Many:
Referência autorId quando autor é reutilizado em muitos livros
Many-to-Many:
Coleção de junção ou arrays de ids com limites
Boas Práticas Globais
Modelagem orientada ao acesso
Índices: Conforme queries (simples/compostos/texto/TTL)
Arrays: Evite arrays "infinitos" (limite 1000 itens)
Profundidade: Limite documentos aninhados
Timestamps: Use createdAt/updatedAt
Padrões de Modelagem
Estruturas comuns e suas aplicações
Padrão Uso Status
One-to-Few Embed (avaliações, metadados) Recomendado
One-to-Many Referência (autores, editoras) Recomendado
Many-to-Many Coleção de junção Cuidado
Árvores Materialized paths Recomendado
Polimorfismo Discriminators Avaliar
Dica: Modele orientado ao acesso. Defina índices conforme as queries mais frequentes.
Limitações e Cuidados
Evite problemas comuns na modelagem
Arrays muito grandes:
Documentos > 16MB ou arrays > 1000 itens causam problemas de performance
Profundidade excessiva:
Subdocumentos aninhados muito profundos dificultam consultas
Duplicação de dados:
Embed quando dados mudam juntos, referencie quando reutilizados
