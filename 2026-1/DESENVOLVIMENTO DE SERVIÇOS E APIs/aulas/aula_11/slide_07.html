Padrão MVC - Conceito
Model-View-Controller: Separação de Responsabilidades
Arquitetura MVC
Três componentes interconectados
Model
Gerencia dados, lógica e regras do aplicativo
Responsabilidades:
• Validar dados
• Aplicar regras de negócio
• Persistência no banco
Atualiza
Notifica
View
Interface do usuário e apresentação
Responsabilidades:
• Exibir dados
• Capturar interações
• Formatação visual
Comanda
Retorna
Controller
Aceita entrada e converte em comandos
Responsabilidades:
• Receber requisições
• Processar entradas
• Coordenar Model e View
Model
Gerencia dados e regras de negócio. É independente da interface do usuário.
View
Apresenta dados ao usuário. Em APIs RESTful, retorna JSON/XML.
Controller
Coordena as interações entre Model e View, processando requisições.
