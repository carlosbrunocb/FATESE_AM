Service-Repository - Introdução
Evolução do MVC para APIs RESTful
O que é o Padrão Service-Repository?
Arquitetura em 4 camadas
O Service-Repository é uma evolução do padrão MVC que separa melhor as responsabilidades em APIs RESTful. Ele divide a aplicação em camadas distintas: Rotas, Controller, Service e Repository, permitindo maior testabilidade, manutenibilidade e escalabilidade do código.
Arquitetura em Camadas
Fluxo de dados
1. Rotas (Routes)
Define endpoints (URLs) e verbos HTTP. Apenas mapeia requisições para controllers.
2. Controller
Lida com o tráfego HTTP, extrai req.body, chama o serviço e devolve res.status.
3. Service
Contém a Regra de Negócio pura. Não sabe de onde vem a requisição nem como os dados são salvos.
4. Repository
Isola a lógica de acesso a dados. É o único que conversa com o ORM (Sequelize/Mongoose).
Vantagens
Separação de Responsabilidades
Cada camada tem uma função específica
Testabilidade
Fácil de testar cada componente isoladamente
Flexibilidade
Fácil trocar tecnologias de banco de dados
