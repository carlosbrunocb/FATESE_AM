Stateful vs Stateless

Comparação entre autenticação baseada em sessões e JWT

Arquitetura de Autenticação
Stateful (Sessões)
1
Login do Cliente
Usuário envia credenciais para o servidor
2
Criação de Sessão
Servidor cria sessão em memória/banco de dados
3
Cookie de Sessão
ID de sessão enviado ao cliente via cookie
Características
• Servidor mantém estado
• Escalabilidade complexa
• Sessão pode expirar
Stateless (JWT)
1
Login do Cliente
Usuário envia credenciais para o servidor
2
Geração de Token
Servidor gera JWT assinado com claims
3
Armazenamento
Cliente armazena token (localStorage/sessionStorage)
Características
• Sem estado no servidor
• Escalabilidade horizontal
• Token auto-contido
