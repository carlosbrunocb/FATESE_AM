Anatomia do JWT

Estrutura de 3 partes: Header, Payload e Signature

JWT Token
Estrutura do Token JWT
Formato: header.payload.signature
1
Header (Cabeçalho)
Contém metadados do token: tipo (JWT) e algoritmo de assinatura (HS256)
{"alg": "HS256", "typ": "JWT"}
2
Payload (Carga Útil)
Contém as claims: dados do usuário, expiração, emissor
{"sub": "123", "name": "João", "exp": 1234567890}
3
Signature (Assinatura)
Garantia de integridade: HMAC-SHA256(header + payload + secret)
HMACSHA256(base64(header) + "." + base64(payload), secret)

Informações Importantes
  • Não coloque dados sensíveis no Payload (é decodificável)
  • Defina expiração adequada (exp)
  • Use HTTPS para transmissão
  • Valide a assinatura em todas as requisições
Atenção: O JWT é assinado, não criptografado. Não armazene senhas no payload.
