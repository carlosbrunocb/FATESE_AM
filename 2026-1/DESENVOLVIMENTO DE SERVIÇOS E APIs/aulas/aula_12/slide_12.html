JWT: Header (Cabeçalho)

Estrutura e campos do cabeçalho JWT

JWT Header
O que é o Header?
O Header é a primeira parte do token JWT e contém metadados sobre o token, incluindo o tipo de token e o algoritmo de assinatura utilizado. É codificado em Base64Url.

Campos Principais
O header contém dois campos essenciais: alg (algoritmo de assinatura) e typ (tipo de token). O campo "alg" deve corresponder ao algoritmo usado para assinar o token.
Campos do Header
alg (Algorithm)
Algoritmo de assinatura
typ (Type)
Tipo do token
kid (Key ID)
ID da chave
cty (Content Type)
Tipo de conteúdo
Atenção!
Nunca use "none" como algoritmo em produção, pois isso desativa a verificação de assinatura.
