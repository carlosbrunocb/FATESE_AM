JWT: Payload (Carga Útil)

Claims registradas, públicas e customizadas

JWT Structure
Estrutura do Payload
Exemplo de Payload JWT
{
  "sub": "usuario123",
  "name": "João Silva",
  "email": "joao@email.com",
  "iat": 1516239022,
  "exp": 1516242622,
  "role": "user"
}
sub (Subject)
Identificador único do usuário
iat (Issued At)
Timestamp de emissão do token
exp (Expiration)
Timestamp de expiração
iss (Issuer)
Emissor do token
Tipos de Claims
Claims Registradas
Padrão RFC 7519 (sub, iat, exp, iss, aud)
RFC 7519
Claims Públicas
Definidas em registro público (name, email)
Públicas
Claims Customizadas
Definidas pela aplicação (role, permissions)
Custom
Atenção: Não coloque dados sensíveis no payload - qualquer pessoa pode decodificar Base64.
