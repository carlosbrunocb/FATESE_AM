Criptografia: Hash vs Encriptação

Entenda as diferenças fundamentais entre hash e encriptação

Segurança de Dados
Hash (Unidirecional)
Processo que converte dados em valor fixo
Características: Transforma dados em uma string de tamanho fixo (ex: 64 caracteres)
Uso principal: Verificação de integridade e armazenamento de senhas
Vantagem: Não pode ser revertido (unidirecional)
Senha original:
minhaSenha123
Hash SHA-256:
5f4dcc3b5aa765d61d8327deb882cf99...
Encriptação (Bidirecional)
Processo que protege dados para leitura posterior
Características: Transforma dados em formato ilegível que pode ser revertido
Uso principal: Proteção de dados sensíveis em trânsito ou armazenados
Vantagem: Pode ser descriptografado com a chave correta

• Encriptação: Para proteger dados que precisam ser lidos posteriormente
