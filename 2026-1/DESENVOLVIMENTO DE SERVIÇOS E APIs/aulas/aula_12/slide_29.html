Boas Práticas de Segurança JWT
Recomendações para proteger APIs e tokens de autenticação
Segurança Ativa
1. Armazenamento Seguro da Chave Secreta
Nunca armazene JWT_SECRET diretamente no código. Use variáveis de ambiente:
process.env.JWT_SECRET // em vez de string hardcoded
2. Configurar Expiração Adequada
Tokens devem ter expiração curta (15-60 minutos). Considere refresh tokens para sessões longas.
3. Usar HTTPS
Sempre use HTTPS em produção para proteger tokens em trânsito contra interceptação.
4. Validar Payload
Nunca confie cegamente no payload do JWT. Valide todos os dados antes de usar no backend.
5. Evitar Dados Sensíveis
Nunca inclua senhas, tokens de API ou dados pessoais identificáveis no payload do JWT.
6. Implementar Revogação
Tenha uma estratégia clara para invalidar tokens comprometidos através de listas de revogação.
Atenção: Tokens JWT são decodificáveis por qualquer um que tenha o token. Não armazene segredos ou dados sensíveis no payload.
7. Rate Limiting
Implemente limites de requisição por IP ou usuário para prevenir ataques de força bruta e DoS.
