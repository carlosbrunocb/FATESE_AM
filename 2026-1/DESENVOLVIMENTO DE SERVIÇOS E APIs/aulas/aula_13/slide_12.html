Vulnerabilidade: Injection (SQL/NoSQL)

Análise de vetores de ataque, sintomas e mitigações
Alta Severidade
Vetor de Ataque
SQL/NoSQL Injection
O atacante envia comandos maliciosos em campos de entrada para manipular o banco de dados. Exemplo: ' OR '1'='1 ou { "$ne": null }
Severidade: Crítica
Sintomas Comuns
Consultas lentas ou falhas
Dados inconsistentes
Erros de sintaxe SQL
Acesso não autorizado
Mitigações
Proteções Recomendadas
1. Validação de Entrada: Sanitizar todos os inputs
2. Queries Parametrizadas: Usar prepared statements
3. ORM/ODM: Utilizar frameworks de mapeamento
4. Princípio do Menor Privilégio: Limitar permissões do DB
// Exemplo de proteção contra SQL Injection
const query = 'SELECT * FROM users WHERE id = ?';
const result = await db.query(query, [userId]);
