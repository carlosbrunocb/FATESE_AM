Proteção: Rate Limiting

Mitigação de ataques de força bruta e DDoS através de limitação de requisições
O que é Rate Limiting?

O Rate Limiting é uma técnica que limita o número de requisições que um cliente pode fazer em um período de tempo específico.

  • Previne ataques de força bruta em endpoints de autenticação
  • Mitiga ataques DDoS de volume massivo
  • Protege contra scraping e abuso de API
  • Melhora a disponibilidade do serviço
Configuração Básica
// Configuração do rate limiter const limiter = rateLimit({ windowMs: 15 * 60 * 1000, // 15 minutos max: 100, // máximo 100 requisições message: { erro: "Muitas requisições deste IP" } }); // Aplicar a todas as rotas de API app.use('/api/', limiter);

Boas Práticas
  • Defina limites baseados em rota (login vs leitura)
  • Implemente backoff exponencial para retries
  • Adicione whitelist para IPs confiáveis
  • Monitore e alerte sobre bloqueios
