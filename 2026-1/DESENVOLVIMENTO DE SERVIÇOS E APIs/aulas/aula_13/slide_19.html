JWT com Role no Payload (Auth Service)

Atualização do serviço de autenticação para incluir roles no token JWT
Implementação Completa
Auth Service - Login com Role
// src/services/authService.js
const login = async (email, senha) => {
const usuario = await usuarioRepository.buscarPorEmail(email);
if (!usuario) throw new Error("Credenciais inválidas.");
const senhaCorreta = await bcrypt.compare(senha, usuario.senha);
if (!senhaCorreta) throw new Error("Credenciais inválidas.");
// NOVO: Adicionando o "role" no Payload do Token
const token = jwt.sign(
{
id: usuario._id,
email: usuario.email,
role: usuario.role // <-- O cargo viaja no token
},
CHAVE_SECRETA,
{ expiresIn: '2h' }
);
return {
usuario: { nome: usuario.nome, role: usuario.role },
token: token
};
};
Estrutura do Token JWT
Payload do Token
id:
"507f1f77bcf86cd799439011"
email:
"usuario@email.com"
role:
"ADMIN"
iat:
1704067200
exp:
1704070800
Importante
O role no payload permite que o middleware de autorização verifique permissões sem consultar o banco de dados.
Fluxo de Autorização
Login: Usuário autentica
Token: JWT gerado com role
Verificação: Middleware checa role
