Padrões de Design e Boas Práticas

Arquitetura de segurança e padrões de implementação para APIs
Recomendado
Separação de Responsabilidades
Middlewares Finos
Cada middleware deve ter uma única responsabilidade (autenticação, autorização, validação)
Controladores Enxutos
Lógica de negócio delegada para serviços, não no controller
Regras de Role Claras
Definir permissões de forma explícita e centralizada
// Exemplo de middleware de autorização
const autorizaRole = (roles) => {
return (req, res, next) => {
if (!roles.includes(req.user.role)) {
return res.status(403).json({ error: 'Acesso negado' });
}
next();
};
};
