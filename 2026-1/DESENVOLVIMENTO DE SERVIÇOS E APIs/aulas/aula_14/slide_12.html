Query vs Path Parameters: Comparação Prática
Entenda as diferenças e quando usar cada tipo de parâmetro em suas APIs

Path Parameters

Identificação de recursos

Identificação Única

Usado para identificar um recurso específico

Obrigatório na URL

Faz parte do caminho da requisição

GET /livros/{id}
GET /livros/64f1...

Uso: Identificar recursos específicos

Query Parameters

Filtros e controle

Filtragem e Busca

Usado para filtrar, ordenar ou paginar

Após o ?

Aparece após a interrogação na URL

GET /livros?titulo=Node
GET /livros?pagina=2&limite=20

Uso: Filtrar e ordenar coleções

Quando usar Path?
Para identificar recursos únicos
Quando usar Query?
Para filtrar e buscar em coleções
Combinação
GET /livros/{id}/comentarios?page=1
