O que é Contrato de API?
Acordo entre cliente e servidor
O contrato de API define como o cliente e o servidor devem se comunicar, incluindo endpoints, formatos de resposta, tipos de dados, status HTTP e headers. É a base para integrações estáveis entre sistemas.
Endpoints: URLs e métodos HTTP disponíveis
Formato: JSON, XML, etc. e estrutura de dados
Tipos: String, Number, Boolean, Array, Object
Impacto de Breaking Changes
Uma quebra de contrato ocorre quando uma alteração na API força o consumidor a modificar seu código para continuar funcionando. Isso pode causar incidentes em produção, necessidade de rollbacks e perda de confiança.
