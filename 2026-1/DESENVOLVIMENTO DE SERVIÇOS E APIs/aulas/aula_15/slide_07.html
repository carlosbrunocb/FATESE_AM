O que NÃO Quebra vs O que QUEBRA
Comparação de alterações que mantêm ou quebram o contrato da API
Desenvolvimento de APIs
Aula 15 - Breaking Changes
O que NÃO Quebra o Contrato
Alterações compatíveis com versões anteriores
Adicionar novo endpoint Seguro
Adicionar um novo endpoint não afeta os existentes. Clientes antigos continuam funcionando normalmente.
Adicionar campo opcional Seguro
Adicionar um novo campo opcional na resposta JSON não quebra clientes existentes.
Otimizações internas Seguro
Melhorias de performance ou refatoração de código interno sem mudar o contrato.
Novos recursos Seguro
Adicionar novos recursos ou funcionalidades sem alterar endpoints existentes.
O que QUEBRA o Contrato
Alterações que exigem mudanças nos clientes
Renomear campo existente Breaking
Alterar titulo para nomeLivro quebra todos os clientes que usam o campo antigo.
Remover endpoint Breaking
Remover um endpoint existente faz com que clientes que o utilizavam parem de funcionar.
Mudar formato de resposta Breaking
Alterar de array simples para objeto paginado exige mudanças no cliente.
Alterar tipo de dado Breaking
Mudar ano de Number para String quebra validações existentes.
