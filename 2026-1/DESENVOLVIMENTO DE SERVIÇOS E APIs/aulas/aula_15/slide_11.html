Aula 15 - Versionamento
O que é Query Parameter Versioning?
Versão passada como parâmetro na URL
A versão da API é especificada como um parâmetro de consulta na URL, permitindo que diferentes clientes escolham a versão desejada através de um parâmetro opcional.
Exemplo:
https://api.livraria.com/livros?version=1
https://api.livraria.com/livros?version=2
Implementação em Express
const express = require('express');
const app = express();
app.get('/livros', (req, res) => {
  const version = req.query.version || '1';
  if (version === '2') {
    res.json({ version: 'v2', data: [] });
  } else {
    res.json({ version: 'v1', data: [] });
  }
});
Prós e Contras
Flexível
Fácil de alternar entre versões
Simples
Fácil de implementar
Cache
Pode afetar cacheabilidade
URL
Menos explícita que URI
Quando usar?
Ideal para APIs internas, protótipos e cenários onde a URL não pode mudar. Também útil para testes A/B.
Boas Práticas
  • Sempre forneça uma versão padrão
  • Documente os parâmetros aceitos
  • Valide a versão recebida
  • Erro 400 para versões inválidas
