Header Versioning
Content Negotiation
O versionamento por header utiliza o cabeçalho Accept para negociar a versão da API, mantendo a URL estável e permitindo negociação de mídia elegante.
// Requisição GET /api/livros HTTP/1.1 Accept: application/vnd.livraria.v1+json // Resposta HTTP/1.1 200 OK Content-Type: application/vnd.livraria.v1+json
Quando Usar
Ideal para APIs com forte foco em hipermídia e media types, onde a URL deve permanecer estável e a versão é negociada via cabeçalhos.
Exemplo de Negociação
Accept: application/vnd.livraria.v1+json
Content-Type: application/vnd.livraria.v1+json
Prós e Contras
Análise da estratégia
Prós
URL estável, negociação elegante, suporte a media types
Contras
Maior complexidade, menor visibilidade, caching complexo
Dica: Use headers Deprecation e Sunset para comunicar descontinuação.
