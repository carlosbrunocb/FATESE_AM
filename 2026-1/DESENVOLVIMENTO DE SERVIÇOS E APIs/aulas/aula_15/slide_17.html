Aula 15 - Versionamento
O Problema do Mundo Real
O catálogo da nossa livraria cresceu muito. Retornar um array com 10.000 livros de uma vez na rota GET /livros está travando o banco de dados e causando timeouts na aplicação.
Sobrecarga do Banco
Consulta retorna 10.000+ registros simultaneamente
Timeout
Requisições demoram mais de 30s para responder
App Mobile
Aplicativo móvel travando ao carregar lista
Impacto da Performance
10K+
Registros
30s+
Tempo Resposta
100%
CPU Usage
A Solução Proposta
Para resolver o problema de performance, precisamos implementar paginação na API. No entanto, isso representa uma Breaking Change - mudar de um array simples para um objeto com metadados de paginação.
V1 Legada
Manter /api/v1/livros
V2 Nova
Criar /api/v2/livros
Estratégia: Manter a v1 intacta para compatibilidade e criar a v2 com paginação.
