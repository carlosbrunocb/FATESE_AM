Avaliação do Item I - Teste de Caixa Branca

INCORRETO
Análise da Assertiva I
Item I: INCORRETO
Aplicar caixa branca em calc(), f1(), f2() e usar resultado em teste de mesa
Justificativa Principal
Problema: Teste de caixa branca requer acesso obrigatório ao código-fonte para analisar estrutura interna, caminhos lógicos e cobertura de código.

Restrição: O enunciado afirma explicitamente que "não há nenhum tipo de documentação ou código fonte além do mostrado".
Impossibilidade Técnica
Sem acesso ao código-fonte das rotinas calc(), f1() e f2(), é impossível:
  • Analisar caminhos de execução
  • Calcular cobertura de código
  • Identificar condições de contorno
Exemplo Prático
1 // Sem código-fonte de calc(), f1(), f2()
2 // Não é possível aplicar caixa branca
3 SUBROTINA xis()
4 i = 0
5 ENQUANTO (i < Gn) FACA
6 // calc(i) - função opaca
7 SE (calc(i) <= Gn) ENTAO
8 f1(i) // função opaca
9 SENAO
10 f2(i) // função opaca
Caixa Preta: Testa entradas/saídas sem ver estrutura interna
Caixa Branca: Requer código-fonte para análise
