Avaliação do Item III - Teste de Caixa Preta em Código Objeto

CORRETO
Análise da Assertiva III
Item III: CORRETO
Aplicar caixa preta isoladamente ao código objeto de calc(), f1(), f2() antes de testar xis()
Justificativa Principal
Viabilidade: É perfeitamente possível aplicar testes de caixa preta em código objeto (binários, executáveis, DLLs), pois você só precisa da interface para enviar dados e observar o retorno.

Abordagem: Testar isoladamente as sub-rotinas calc(), f1() e f2() antes de integrá-las com xis() reduz incerteza e expõe efeitos colaterais.
Vantagens da Abordagem
  • Isola defeitos cedo no ciclo de teste
  • Facilita diagnóstico de problemas
  • Permite identificar alterações indevidas em Gn
Integração Bottom-up
calc()
f1()
f2()
xis()
1 // Teste de caixa preta em código objeto
2 // Testar cada sub-rotina isoladamente
3 test(calc, [1, 2, 3], // entradas)
4 assert(resultado_esperado, // validação)
5 monitor(Gn, // verificar alterações)
Isolamento: Testa cada módulo separadamente
Validação: Verifica comportamento esperado
