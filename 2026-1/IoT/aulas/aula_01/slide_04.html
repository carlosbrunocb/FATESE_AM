Arquitetura IoT em Camadas

Aplicação (Nuvem/Edge)

Bancos de dados, APIs, dashboards, analytics, processamento e visualização de dados

Rede (Conectividade)

Wi-Fi, Bluetooth LE, Ethernet, LoRa; Protocolos: MQTT, HTTP, CoAP

Percepção (Borda)

Sensores, atuadores, MCUs (ESP32), condicionamento de sinais

Fluxo de Dados

Sensor → MCU/Gateway: Coleta e pré-processamento local

MCU → Broker/Serviço: Transmissão via protocolos IoT

Broker → Aplicação: Armazenamento e análise

Aplicação → Usuário: Visualização e controle

Considerações Importantes

Segurança: TLS/SSL, autenticação, criptografia

Energia: Otimização para dispositivos battery-powered

Latência: Trade-off entre cloud e edge processing

Escalabilidade: Arquitetura preparada para crescimento
