Lógica de Botões: Bounce e Debounce

Lidando com a imperfeição física dos componentes eletrônicos

O Problema (Bounce)

Botões são componentes mecânicos. Quando você aperta o botão, as plaquinhas de metal lá dentro "quicam" microscopicamente antes de estabilizar.

Para o microcontrolador (que lê em microssegundos), um único aperto humano parece 5, 10 ou 20 apertos rápidos consecutivos.

← Ruído Mecânico

A Solução (Debounce por Software)

Para corrigir isso, instruímos o código a ignorar as leituras muito rápidas e sucessivas.

A lógica é simples: "Se o botão foi apertado, espere ~50 milissegundos. Se depois disso ele continuar apertado, então foi um aperto real."

← Sinal Limpo
// Lógica de Debounce usando millis() aplicada na Atividade do Semáforo (Slide 23)
if (digitalRead(botaoPin) == LOW) {
    if (millis() - ultimoTempoDebounce > 50) { // 50ms de trava
        // Ação real executada aqui! O aperto foi confirmado.
        iniciarTravessiaPedestre();
        ultimoTempoDebounce = millis(); // Reseta o cronômetro
    }
}
