Estrutura Básica do Código

O esqueleto fundamental de uma aplicação ESP32 para leitura de sensores analógicos.

leitura_sensor.ino

// Definição de pinos (Hardware)
const int pinoPot = 34; // GPIO 34 (Somente Entrada)

void setup() {
// 1. Inicializa comunicação Serial
Serial.begin(115200);

// 2. Configura pino como entrada
pinMode(pinoPot, INPUT);
}

void loop() {
// 3. Leitura Analógica (0 - 4095)
int leitura = analogRead(pinoPot);

Serial.print("Valor Pot: ");
Serial.println(leitura);

delay(500); // Pequena pausa
}

void setup()

Executado uma única vez ao ligar. Local ideal para iniciar a Serial e definir modos dos pinos (INPUT/OUTPUT).

void loop()

Executado infinitamente. Aqui ocorre a leitura dos sensores (analogRead) e a lógica de decisão.

Boas Práticas

1. Use const int para pinos (evita mudanças acidentais).
2. Use o Serial Monitor para depurar (debug).
3. Comente o código para explicar "o porquê" das decisões.
