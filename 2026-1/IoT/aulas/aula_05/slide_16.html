Base teórica

PWM no ESP32
(LEDC e analogWrite)

Periféricos

Frequência & resolução

Periférico LEDC

O ESP32 possui o LEDC (LED Controller) para gerar PWM de forma flexível e eficiente.

Canais

Permite controlar múltiplas saídas PWM (vários canais).

Timers

Timers configuráveis para ajustar frequência.

Uso típico

Dimming de LED, velocidade de motor (com driver) e geração de sinais com duty controlado.

Resolução (até 20 bits)

A resolução do duty pode chegar a 20 bits (dependendo da frequência configurada).

Passos

Mais bits ⇒ mais níveis de brilho/controle.

Trade-off

Frequência maior costuma reduzir a resolução.

Regra prática

Escolha a frequência pensando no atuador (LED/motor/servo) e ajuste a resolução para manter suavidade.

analogWrite (Arduino)

Em versões atuais do core Arduino-ESP32, analogWrite() oferece uma API simples para PWM.

Faixa

Valor 0–255 ≈ duty 0–100%.

Frequência

Tipicamente em torno de ~5 kHz (padrão), variando por configuração.

Quando usar

Para começar rápido (ex.: LED respirando). Para controle avançado, configure o LEDC diretamente.

Checklist de projeto (PWM)

Antes de escolher frequência e duty, valide o atuador e o circuito.

Percepção

LED: evite flicker com frequência adequada.

Potência

Use driver/MOSFET quando a carga exigir corrente.

Conexão com a prática

No Wokwi, usaremos analogWrite para o LED e a biblioteca ESP32Servo para pulsos do servo.

Ideia central

O ESP32 oferece PWM “na mão” via LEDC (controle fino) e uma API simples via analogWrite() (rápida para começar).

Frequência + resolução = qualidade do controle

Dica: se precisar de mais controle (frequência/resolução), use as APIs LEDC; se precisar de simplicidade, use analogWrite.
