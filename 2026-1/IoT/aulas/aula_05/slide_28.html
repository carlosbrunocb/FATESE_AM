Aula 05 — Atuadores e PWM

Atividade — exercícios práticos (Wokwi + Arduino IDE)

Exercícios propostos

A ideia é consolidar o loop de controle (ler sensor → decidir → acionar atuador), usando ADC, PWM e servo.

1

LDR → Brilho do LED

Leia o LDR no ADC e controle o duty do LED.

ADC → PWM

Entrada

0..4095

Saída

0..255

Dica

map()

brilho = map(ldr, 0, 4095, 0, 255)

analogWrite(ledPin, brilho)

2

Potenciômetro → Ângulo do Servo

Use um pot para comandar o ângulo do servo em tempo real.

ADC → write()

Entrada

0..4095

Saída

0..180°

Lib

ESP32Servo

ang = map(pot, 0, 4095, 0, 180)

meuServo.write(ang)

3

Frequência do PWM (LED)

Alterar a frequência do PWM e observar flicker (LED) e percepção visual.

Experimento

Teste

200 Hz

Teste

1 kHz

Teste

5 kHz

Registre: há tremulação? Qual configuração ficou mais confortável?

4

Fading com “curva suave”

Trocar o incremento linear por uma curva (ex.: quadrática ou seno) para suavizar a percepção.

Efeito

Sugestão: use uma função para converter t em brilho (0..255) e comparar com o fading “reto”.

brilho = (x*x) / 255

Onde x varia de 0 a 255.

5

Botão abre/fecha (debounce)

Criar uma tranca com servo: ao pressionar um botão, alternar entre 0° (fechado) e 90° (aberto). Implementar debounce para evitar múltiplos acionamentos.

Estado + tempo

Dica 1

Use uma variável bool aberto para guardar o estado atual.

Dica 2

Debounce com millis() e um intervalo (ex.: 30–80 ms).

Critério

No clique válido: aberto = !aberto e atualiza o servo.

Critérios de entrega

Sugestão de organização para facilitar avaliação e versionamento.

Estrutura

Um projeto por exercício ou um projeto único com pastas: /ex1_ldr_led, /ex2_pot_servo, etc.

README mínimo

Inclua: objetivo, circuito (print do Wokwi), como executar e o que observar.

Commits

Faça commits por etapa: circuito, leitura ADC, mapeamento, ação.

Dica para todos os exercícios

Se o sinal oscilar, aplique um filtro simples (média de N leituras) antes de mapear para PWM/ângulo.
