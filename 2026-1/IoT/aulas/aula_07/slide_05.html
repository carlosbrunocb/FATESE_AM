ISRs: Regras de Ouro e Padrão FreeRTOS

As 6 Regras de Ouro da ISR

Curta e rápida: Apenas limpe flags ou defina notificações.
Sem delay(): NUNCA use delays ou bloqueios.
Evite Serial: Serial.print() pode causar travamentos.
Volatile: Use em variáveis alteradas pela ISR e lidas fora.
IRAM_ATTR: Garante execução rápida a partir da RAM.
Sem malloc/free: Evite alocação dinâmica (heap).
// Exemplo simples e rápido (Sem RTOS) volatile bool flag_botao = false; void IRAM_ATTR isrHandler() { flag_botao = true; }

Modos de Disparo (Triggers)

RISING
Dispara na borda de subida (LOW → HIGH)
FALLING
Dispara na borda de descida (HIGH → LOW)
CHANGE
Dispara em qualquer mudança de estado

Padrão ISR → Task (FreeRTOS)

Processamento Adiado (Deferred Processing)
1. Evento de HW Pino dispara
2. ISR IRAM_ATTR xTaskNotifyFromISR
3. Task no Loop Executa ação
// Task Handle Global TaskHandle_t TaskProcessaHandle = NULL; // 1. A ISR (Executa rápido e acorda a Task) void IRAM_ATTR isrHandler() { BaseType_t xHigherPriorityTaskWoken = pdFALSE; // Notifica a Task enviando sinal (0) e ação definida xTaskNotifyFromISR(TaskProcessaHandle, 0, eSetValueWithoutOverwrite, &xHigherPriorityTaskWoken); // Força a troca de contexto se a Task for prioritária if (xHigherPriorityTaskWoken) { portYIELD_FROM_ISR(); } } // 2. A Tarefa (Fica bloqueada até receber o sinal) void vTaskProcessa(void *pvParameters) { uint32_t valorRecebido; while(1) { // Aguarda a notificação da ISR (Consome 0% CPU) xTaskNotifyWait(0x00, ULONG_MAX, &valorRecebido, portMAX_DELAY); // Aqui fazemos o trabalho pesado (Serial, Wi-Fi, etc) Serial.println("Interrupcao tratada na Task!"); } }
