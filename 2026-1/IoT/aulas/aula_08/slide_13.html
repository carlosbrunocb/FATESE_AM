Parte 2: Conexão Automática e Resiliência
Implementando reconexão automática e tratamento de erros
Auto-reconexão
Objetivo
Conectar o ESP32 via WiFi.begin() e monitorar o estado com WiFi.status(). Implementar reconexão automática ao detectar perda de link (status != WL_CONNECTED), garantindo resiliência para projetos de campo.
Início
WiFi.begin()
Aguardar
WL_CONNECTED
Conectado
WiFi.localIP()
Monitor
Loop + Reconnect
Status da Conexão
Status
WL_CONNECTED
RSSI
-45 dBm
Tempo
2.5s
Segurança
WPA2
Implementação
void conectarWiFi() {
WiFi.begin(ssid, password);
while (WiFi.status() != WL_CONNECTED) {
delay(500);
Serial.print(".");
}
Serial.println(WiFi.localIP());
}
Nota para Wokwi
No simulador, utilize o SSID "Wokwi-GUEST" sem senha. A conexão é instantânea, mas na vida real pode levar 2-10 segundos.
Observações Práticas
•
Timeout: Projete timeouts adequados (2-10s) para associação/DHCP
•
Backoff: Evite reconectar em alta frequência
•
Logs: Use Serial para diagnóstico durante comissionamento
•
Latência: Associação + DHCP podem levar 2-10s
