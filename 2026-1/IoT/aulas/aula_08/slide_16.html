Código Limpo em IoT
Princípios de organização e boas práticas para ESP32
Clean Code
Três Princípios Principais
Organização
Separar responsabilidades em funções específicas
Feedback
Mensagens Serial claras e contextuais
Tratamento de Erros
Verificar status e implementar retries
Benefícios: Manutenção facilitada, menos falhas, melhor OTA
Exemplo: Código Mal Estruturado (Anti-Padrão)
❌ Evitar
1 void setup() {
2 while (WiFi.status() != WL_CONNECTED) {
3 delay(500); // Sem timeout!
4 }
5 Serial.println("Conectado"); // Sem contexto
6 }
Exemplo: Código Bem Estruturado (Padrão)
✓ Recomendado
1 bool conectarWiFi(const char* ssid, const char* pass, int timeoutMs) {
2 WiFi.begin(ssid, pass);
3 int tentativas = 0;
4 while (WiFi.status() != WL_CONNECTED && tentativas < timeoutMs/500) {
5 delay(500); tentativas++;
6 }
7 return WiFi.status() == WL_CONNECTED;
8 }
