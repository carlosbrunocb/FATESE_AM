Princípios da Arquitetura REST

6 princípios fundamentais para APIs RESTful

1. Recursos (URIs)
Identificação por URIs

Cada recurso é identificado por uma URI única. Exemplo: /v1/sensores/42

GET /api/v1/dispositivos
2. Representações
JSON, XML, etc.

Recursos são representados em formatos padronizados para o cliente.

{"status": "ativo"}
3. Stateless
Sem estado no servidor

Cada requisição contém toda a informação necessária para ser processada.

Idempotente
4. Cacheable
Performance e Reuso

As respostas devem se auto-declarar cacheáveis para evitar chamadas repetitivas.

Cache-Control: max-age=3600
5. Interface Uniforme
Padronização HTTP

Uso consistente de métodos HTTP (GET, POST, PUT, DELETE).

GET POST
6. Layered System
Escalabilidade

O cliente não sabe se está conectado ao servidor final ou a um intermediário.

Independência
Boas práticas: nomes no plural, versionamento e códigos HTTP consistentes
