Modelo HTTP: Cliente-Servidor (Request/Response)
Arquitetura tradicional para comunicação web
Funcional
Diagrama de Arquitetura HTTP
Cliente
Navegador/App
Request →
← Response
Servidor
Web Server
Fluxo de Comunicação:
1. Cliente envia requisição HTTP para o servidor
2. Servidor processa a requisição
3. Servidor retorna resposta HTTP
4. Conexão é fechada após resposta
Características HTTP
Conexões sob demanda
Nova conexão para cada requisição
Cabeçalhos extensos
Protocolo sem estado (stateless)
Pull de dados
Via polling/long-polling
Limitações
Menos eficiente para telemetria frequente
