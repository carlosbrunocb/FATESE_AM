Firmware de Controle do ESP32 (Wokwi)

📄 firmware_mqtt.ino (Trecho Principal) 
void callback(char* topic, byte* payload, unsigned int length) {
  String mensagem = "";
  for (int i = 0; i < length; i++) {
    mensagem += (char)payload[i];
  }

  // Executa o acionamento físico
  if (mensagem == "1") {
    digitalWrite(pinoRele, HIGH);
    client.publish(topico_estado, "LIGADO"); 
  } 
  else if (mensagem == "0") {
    digitalWrite(pinoRele, LOW);
    client.publish(topico_estado, "DESLIGADO");
  }
}

Nota: O loop principal deve processar client.loop() constantemente.

O papel da Função Callback

É o gerenciador de eventos do MQTT. Quando uma nova mensagem de comando chega do tópico assinado (/set), esta função é disparada automaticamente de modo assíncrono.

Acionamento Físico Real

O comando digitalWrite(pinoRele, HIGH) altera a saída física de tensão do chip ESP32 para energizar o LED ou fechar o Relé de acionamento.

Publicação de Confirmação

Após a alteração do pino físico, o código imediatamente publica de volta o estado real de sucesso para /state. Isso garante que a interface do usuário se atualize apenas em caso de operação bem sucedida.
