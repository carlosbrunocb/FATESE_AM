Tratamento de Respostas HTTP e Erros
Como lidar com diferentes códigos de resposta e falhas na comunicação ESP32-Cloud
Códigos 2xx - Sucesso
Indicam que a requisição foi processada com sucesso pelo servidor.
  • 200 OK: Requisição bem-sucedida, resposta normal
  • 201 Created: Recurso criado com sucesso
  • 202 Accepted: Requisição aceita, processamento pendente
// Exemplo de tratamento de sucesso if (statusCode == 200) { Serial.println("Dados enviados com sucesso!"); }
Sucesso
Códigos 4xx - Erro do Cliente
Indicam problemas na requisição enviada pelo cliente.
  • 400 Bad Request: Formato JSON inválido
  • 401 Unauthorized: Token de autenticação inválido
  • 403 Forbidden: Acesso negado ao recurso
  • 404 Not Found: Endpoint não encontrado
// Verificar erros 4xx if (statusCode >= 400 && statusCode < 500) { Serial.println("Erro na requisição!"); }
Erro Cliente
Códigos 5xx - Erro do Servidor
Indicam problemas internos no servidor de destino.
  • 500 Internal Server Error: Erro interno do servidor
  • 502 Bad Gateway: Gateway inválido
  • 503 Service Unavailable: Serviço indisponível
  • 504 Gateway Timeout: Timeout na resposta
// Implementar retry com backoff if (statusCode >= 500) { delay(5000); // Esperar 5s antes de retry }
