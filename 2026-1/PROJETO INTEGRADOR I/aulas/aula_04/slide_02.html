O que você deve dominar ao final desta aula

Projeto Integrador I

Diferenciar Feature Principal vs Secundárias

Separar o núcleo do negócio (o que faz o sistema existir) das “perfumarias” que podem ficar para versões futuras (MVP).

Pergunta-guia: “Sem isso, o sistema ainda faz sentido?”

Definir RF e RNF com clareza e testabilidade

Escrever RF como ações (o que o sistema faz) e RNF como qualidades/restrições (como deve ser).

RF: “O sistema deve permitir…”

RNF: métricas (ex.: ≤ 3s)

RNF: segurança / tecnologia

Priorizar simplicidade e viabilidade (Anti-NASA)

Tomar decisões de escopo para entregar um MVP realizável no semestre: fluxo principal ponta a ponta funcionando.

Corte integrações complexas nesta fase (foco no essencial)

Mapa mental do conteúdo

Da ideia → requisitos → MVP

1

Feature Principal

O “porquê” do sistema existir

2

RF (O QUE faz)

Ações e fluxos do usuário

3

RNF (COMO deve ser)

Qualidade, restrições e padrões

Exemplos rápidos (teóricos)

iFood: principal = pedir comida • secundária = chat com suporte

Uber: principal = solicitar viagem • secundária = foto de perfil

Ao final, seu grupo terá uma Feature Principal, uma lista mínima de RF e RNF prontos para virar User Stories.

Saída esperada: requisitos claros, curtos e viáveis (MVP do PI I).

Simples • Testável • Entregável
