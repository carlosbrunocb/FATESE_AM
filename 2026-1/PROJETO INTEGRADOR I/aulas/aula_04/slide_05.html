Requisitos Não Funcionais (RNF)

Como o sistema deve ser (qualidade, restrições e padrões)

Projeto Integrador I

Conceito

RNF descreve qualidades (ex.: desempenho, usabilidade), restrições (ex.: web, idioma, tecnologia) e padrões que o sistema deve cumprir.

Pergunta-guia: “Qual a regra de qualidade/limite que não pode falhar?”

Formato (curto, claro e mensurável)

Use identificadores: RNF01, RNF02… e tente incluir métrica/critério sempre que possível.

“carregar em ≤ 3s”

“responsivo”

“POO conforme disciplina”

“PT-BR”

Exemplos de RNF (teóricos)

RNF01 (Usabilidade)

A interface deve permitir concluir o fluxo principal em até 3 cliques.

RNF02 (Desempenho)

As páginas principais devem carregar em até 3 segundos.

RNF03 (Tecnologia)

O sistema deve ser desenvolvido usando POO (conforme requisitos da disciplina).

RNF04 (Idioma)

Toda a interface do usuário deve estar em Português-Brasil.

Dica: prefira RNF com critério verificável (métrica, limite, compatibilidade).

Checklist rápido (RNF bem escrito)

Use para revisar antes de entregar

Mensurável

Tem número, limite ou condição verificável?

Observável

Alguém consegue testar/inspecionar?

Viável no semestre

Evita “NASA” (IA complexa/inúmeras integrações)?

Exemplo completo

RNF05 (Compatibilidade): O sistema deve ser uma aplicação Web, acessível via Chrome e Firefox.

RNF define padrões de qualidade do MVP. Sem eles, o sistema até “funciona”, mas pode ser lento, difícil de usar ou inviável.
