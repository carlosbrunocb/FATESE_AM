Projeto Integrador I

resolver a dor

Se o fluxo mínimo não funciona, nenhuma feature extra “salva” o projeto.

Regras práticas (Anti-NASA)

Decisões de escopo para reduzir risco e aumentar entrega

Corte integrações complexas no MVP

Comece com dados simples (cadastro manual) e evolua depois.

Evite “10 APIs + IA + BI” no 1º release

Um público-alvo + uma plataforma

Ex.: Web primeiro. Mobile nativo pode virar “versão 2”.

Critério de “MVP pronto”

Fluxo principal ponta a ponta funcionando e testável.

RF escritos e numerados (RF01…) com ações claras.

RNF mínimos definidos (plataforma, desempenho, tecnologia) para evitar “NASA”.
