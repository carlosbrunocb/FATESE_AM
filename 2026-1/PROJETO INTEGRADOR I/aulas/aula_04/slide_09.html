Exemplo prático: EcoTrack (Resumo do Escopo)

Transformando a ideia em Feature Principal + RF + RNF (MVP viável)

Projeto Integrador I

Contexto do problema

Condomínios: falta de organização e engajamento na coleta seletiva

Moradores não sabem horários

Informação dispersa e pouca previsibilidade.

Administração sem dados

Sem registro do volume mensal e impactos.

Baixo engajamento

Sem feedback claro do benefício ambiental.

Feature Principal (Killer Feature)

Dashboard de impacto em tempo real: converte o peso reciclado em indicadores simples (ex.: “árvores poupadas”, “litros de água preservados”).

Valor ao usuário

Enxerga benefício real e entende “por que participar”.

Regra do slide 3

Se tirar isso, o sistema perde a razão de existir.

No MVP, foque em: registrar volume → mostrar impacto. Gamificação e push podem virar versão futura.

Esquema do “impacto” (visão de requisitos)

Entrada

Peso mensal (kg) registrado pelo síndico/admin

Regra simples

Conversão por fator (tabela/constante no sistema)

Saída

Dashboard + gráfico (últimos 3 meses)

Requisitos Funcionais (RF)

O QUE o sistema faz (ações e fluxos)

RF01 — O sistema deve permitir que o administrador cadastre os horários de recolha seletiva.

RF02 — O sistema deve permitir que o morador visualize o cronograma do seu bloco.

RF03 — O sistema deve permitir busca no “Guia de Descarte” por nome do material (ex.: “lâmpada”).

RF04 — O sistema deve permitir que o síndico registre o peso total de recicláveis no mês.

RF05 — O sistema deve gerar um gráfico simples comparando o volume dos últimos 3 meses.

Padrão de escrita: 1 ação por requisito (curto e testável).

Requisitos Não Funcionais (RNF)

COMO deve ser (qualidade e restrições)

RNF01 (Usabilidade)

Encontrar o horário de recolha em no máximo 3 cliques.

RNF02 (Disponibilidade)

Aplicação Web, acessível via Chrome e Firefox.

RNF03 (Desempenho)

Dashboard de impacto carrega em até 3 segundos.

RNF04 (Tecnologia)

Desenvolvido com POO (conforme requisitos da disciplina).

RNF05 (Idioma)

Interface em Português-Brasil.

Bom RNF é mensurável (≤ 3s, 3 cliques) e reduz risco do MVP.

O que pode virar “versão futura”? (secundárias)

Exemplos do próprio EcoTrack para aplicar o Anti‑NASA no semestre.

Ranking / gamificação

Notificações push

Foto/Perfil completo

Chat / suporte

Frase para cortar excesso

“Foque no que dói mais no seu cliente agora.”

Saída do grupo (hoje)

1 Feature Principal + 5 RF + 3+ RNF no Documento de Escopo.

Dica: use este formato (Feature Principal + RF + RNF) como modelo para o seu projeto.

Simples • Testável • Entregável
