Conceito central

Secundária não é “perfumaria”

Ela viabiliza a jornada

Conceitos-chave

Funcionalidade Principal (Killer Feature)

Núcleo que entrega o valor central e justifica o produto.

Funcionalidade Secundária

Habilita, acelera ou torna possível perceber o valor da principal.

Jornada do Usuário

O valor só é percebido se o usuário completar o fluxo até o resultado.

Ponto de atenção

Secundária indispensável é aquela que conecta a principal ao resultado real (valor entregue).

Esquema rápido: valor acontece no fluxo

Principal

Killer Feature

Secundárias

Suporte

Valor

Resultado

Se o usuário não chega ao resultado, a “Killer Feature” vira uma promessa não comprovada.

Base teórica para priorizar

MVP (Eric Ries)

Menor conjunto para validar valor com usuários reais e aprender rápido.

Modelo Kano (visão prática)

Must-have (essencial), Performance (diferencial), Delighters (agradável).

Trade-off: Simplicidade × Abrangência

Cada “extra” aumenta custo, risco e tempo de entrega. MVP foca no valor validável.

Exemplo (teórico → prático)

Se a principal é “match por afinidade”, uma secundária “chat abrigo–adotante” pode ser must-have, pois viabiliza a conversão do match em ação.

Perguntas para classificar uma secundária

Essencial Diferencial NASA

Sem isso, o sistema perde o sentido? (quebra ou não entrega o valor)

Isso prova nosso diferencial? (melhor que o concorrente, de forma simples)

É tecnologia pela tecnologia? (entra no roadmap, não no MVP)
