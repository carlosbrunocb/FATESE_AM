Passo a passo (CLI)

1) Clonar e entrar

git clone <url-do-repo>

cd <pasta>

2) Criar develop

git checkout -b develop

git push -u origin develop

3) Criar feature

git checkout -b feature/match-algoritmo

4) Commit e push

git add .

git commit -m \"feat(match): algoritmo inicial\"

git push -u origin feature/match-algoritmo

5) Abrir PR

feature/* → develop

Solicite 1 revisão e descreva como testar.

Checklist do Pull Request

Link para a Issue

Ex.: “Closes #12” para fechar automaticamente.

Como testar

2–3 passos para validar o RF do MVP.

Evite “PR gigante”

Se ficou grande, quebre em partes menores.

Objetivo: reduzir retrabalho e manter velocidade para entregar o MVP.

Por que isso importa?

Um fluxo simples (feature → PR → develop → main) evita “código quebrado” e acelera a entrega do recorte do MVP.
