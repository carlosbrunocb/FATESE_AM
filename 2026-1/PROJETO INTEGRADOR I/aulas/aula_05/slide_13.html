Transformem o MVP em tarefas

1 Issue por RF (com critério de aceitação) + Kanban simples (To do/Doing/Done).

Protejam a main

Trabalho via branch + PR para evitar “quebrar a entrega”.

Lembrete anti-NASA

Se ao remover uma funcionalidade secundária o usuário ainda resolve o problema principal, ela vai para o roadmap, não para o MVP.
