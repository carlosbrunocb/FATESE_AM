Manifesto Ágil — Valores Fundamentais

Os 4 pilares que guiam o desenvolvimento ágil de software
2001 17 signatários
Indivíduos e Interações
Mais que processos e ferramentas
Pessoas são mais importantes que processos. A comunicação face a face e a colaboração direta são essenciais para o sucesso do projeto.
Aplicação no Projeto Integrador
• Comunicação direta entre membros do time
• Reuniões diárias (Daily Scrum) para alinhamento
• Decisões rápidas sem burocracia
Software em Funcionamento
Mais que documentação abrangente
Entregar valor funcional é prioridade. Documentação é importante, mas software funcionando é a medida real de progresso.
Aplicação no Projeto Integrador
• MVP funcionando > documentos extensos
• Entregas incrementais de valor
• Validação frequente com usuários
Colaboração com o Cliente
Mais que negociação de contratos
Parceria ativa com stakeholders. O cliente deve participar ativamente do processo de desenvolvimento.
Aplicação no Projeto Integrador
• Professor como cliente (PO)
• Validações frequentes de requisitos
• Feedback contínuo e adaptação
Responder a Mudanças
Mais que seguir um plano
Adaptação ao invés de resistência. Mudanças nos requisitos são bem-vindas, mesmo em estágios avançados.
Aplicação no Projeto Integrador
• Backlog adaptável a cada aprendizado
• Flexibilidade para mudanças
• Iterações curtas e feedback rápido
Princípio Central
Valorizar indivíduos e interações sobre processos e ferramentas
Métrica de Sucesso
Software funcionando como medida real de progresso
Colaboração
Cliente e equipe trabalhando juntos diariamente
Estes valores são fundamentais para o sucesso do Projeto Integrador
