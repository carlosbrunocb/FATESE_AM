Definição, formato e boas práticas

O que é uma User Story?
Uma User Story descreve, de forma simples, a necessidade de um usuário para gerar valor ao negócio.
Definição de User Story
Uma User Story é uma descrição informal, em linguagem natural, do que um usuário quer fazer em um sistema para obter algum valor. Diferente de um requisito técnico, a User Story foca na necessidade de quem usa o sistema.
Formato Padrão: "Como/Quero/Para"
Como [Persona], quero [Ação], para [Benefício/Valor]
Linguagem do Usuário
Evitar jargão técnico
Convite à Conversa
Diálogo entre negócio e time
Foco na Necessidade
Não apenas o "o que"
Valor Claro
Benefício no "para"
Exemplo Prático
Como Adotante, quero responder ao Quiz de Afinidade, para receber sugestões de pets que combinem com meu espaço e tempo.
Persona: Adotante Ação: Responder Quiz Valor: Match de Pets
Critérios de Aceitação:
• Quiz deve ter 8+ perguntas
• Resultados em tempo real
• Filtro por compatibilidade
