Técnicas de Delimitação de Escopo
Quatro dimensões para definir as fronteiras do projeto

Aula 10 - Projeto Integrador I

Delimitação efetiva requer definir claramente quando, onde, o que e como o sistema será implementado. Cada dimensão de delimitação protege o projeto contra o "Efeito NASA" e garante entrega focada.

1. Delimitação Temporal
Quando o sistema será implementado
Define o período de implementação, fases do projeto e cronograma de entregas. Estabelece marcos temporais claros e deadlines realistas.
Exemplos de aplicação:
Fase 1: 3 meses Fase 2: 2 meses Testes: 1 mês
2. Delimitação Geográfica
Onde o sistema será implementado
Especifica a localização física ou virtual onde o sistema operará. Define alcance geográfico e ambientes de uso.
Exemplos de aplicação:
Campus universitário Escritório central Cidade X
3. Delimitação Funcional
O que o sistema fará e não fará
Define as funcionalidades incluídas e excluídas do sistema. Estabelece limites claros do escopo funcional.
Exemplos de aplicação:
Cadastro de usuários Relatórios Não inclui: pagamentos
4. Delimitação Tecnológica
Quais tecnologias serão usadas
Especifica as tecnologias, plataformas e ferramentas utilizadas. Define limitações técnicas e compatibilidades.
Exemplos de aplicação:
React + Node.js MySQL API REST
