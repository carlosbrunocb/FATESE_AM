11 · A REGRA DE OURO
As 3 leis da escrita de requisitos
Se uma destas falhar, o requisito é refeito — não negociável.
LEI 01
RF começa com
“O sistema deve permitir / gerar”
Toda função do sistema é descrita por uma ação esperada diante de uma interação do ator.
EXEMPLOS
✓ O sistema deve permitir cadastro
✓ O sistema deve gerar relatório
✓ O sistema deve calcular score
✓ O sistema deve exibir lista
LEI 02
RNF descreve
QUALIDADE ou RESTRIÇÃO
Toda tecnologia base, limite de tempo ou norma de segurança vira RNF — nunca RF.
EXEMPLOS
✓ Desenvolvido em React + Node
✓ Resposta em até 3 s
✓ Senhas com bcrypt
✓ Compatível com Chrome 100+
LEI 03
ATOMICIDADE
Uma função por ID
Cada requisito descreve apenas uma função. Se aparecer a palavra “e” juntando ações — quebre em dois IDs.
PROIBIDO
✗ "O sistema deve fazer login, cadastrar pet e gerar relatório"

→ Vira 3 IDs separados
