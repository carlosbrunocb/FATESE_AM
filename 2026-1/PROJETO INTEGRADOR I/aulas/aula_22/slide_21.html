21 · QUADRO 7 · MATRIZ DE RNF
5.3.2  Requisitos Não Funcionais
Restrições tecnológicas, premissas de segurança e critérios de performance
ID
DESCRIÇÃO DO REQUISITO NÃO FUNCIONAL
CATEGORIA
PRIORIDADE
[RNF-01]
A interface de usuário deve ser construída seguindo o padrão Mobile First, garantindo responsividade em telas de smartphones.
USABILIDADE
ALTA
[RNF-02]
As senhas dos usuários devem ser armazenadas no banco de dados de forma criptografada, utilizando algoritmos de hashing (ex: bcrypt).
SEGURANÇA
ALTA
[RNF-03]
O processamento do algoritmo de match (RF-03) deve retornar o resultado na tela do usuário em um tempo máximo de 3 segundos.
PERFORMANCE
MÉDIA
[RNF-04]
O sistema web deve ser compatível com as versões mais recentes de Chrome, Safari e Firefox.
PORTABILIDADE
BAIXA
Quadro 7 — Matriz de Requisitos Não Funcionais (Dossiê AdoptMatch, Cap. 5.3.2). Note como RNF-03 referencia RF-03 — rastreabilidade explícita.
