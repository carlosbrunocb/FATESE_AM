Conceito Formal

Extensão do modelo cascata que enfatiza que para cada fase de desenvolvimento existe uma fase de teste correspondente. Garante alta rastreabilidade entre o que foi pedido e o que foi testado.

Exemplo: Sistema Bancário

Contexto: Core Banking (Compensação)

  • Regras BCB ↔ Teste de Aceite

  • Fluxo de Transação ↔ Teste de Sistema

  • API de Saldo ↔ Teste de Integração

Ponto de Atenção: Rigidez

Assim como o Cascata, é difícil acomodar mudanças de escopo no meio do caminho. Exige documentação robusta desde o início.
