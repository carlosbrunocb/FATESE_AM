Exemplo Teórico de QA

Implementando prevenção em um cenário de desenvolvimento

Cenário do Projeto

Uma equipe de desenvolvimento backend está iniciando a construção de uma API financeira crítica. O objetivo do QA é estabelecer barreiras de qualidade antes mesmo da primeira linha de código ser mesclada.

1. Padrões de Codificação

Definições:

  • Naming: Variáveis em snake_case, Classes em PascalCase.
  • Estilo: PEP8 (Python) ou Google Style Guide.
  • Complexidade: Nenhuma função com complexidade ciclomática > 10.

2. Code Review

Checklist Obrigatório:

  • O código está legível e comentado?
  • Existem testes unitários para a nova lógica?
  • Entradas de usuário foram tratadas (Segurança)?
  • Não há credenciais "hardcoded"?

3. Definition of Done (DoD)

Critérios de Aceite:

  • Todos os testes automatizados passando (Green).
  • Cobertura de código ≥ 80%.
  • Análise estática (Sonar) sem bugs críticos.
  • Documentação da API atualizada.

Efeito Esperado

A aplicação rigorosa destes processos de QA reduz a variabilidade do código entre diferentes desenvolvedores, previne defeitos recorrentes de lógica e segurança, e acelera o onboarding de novos membros, pois as regras do jogo estão claras.
