Exemplo Prático: Pipeline CI/CD

Automatizando a garantia de qualidade com GitHub Actions

1. Lint & Style

flake8

2. Testes Unitários

pytest

3. Cobertura & SAST

SonarQube

4. Build & Merge

Docker
.github/workflows/ci.yml

name: CI Pipeline

on: [push, pull_request]

jobs:

quality-check:

runs-on: ubuntu-latest

steps:

- uses: actions/checkout@v2

- name: Run Lint

run: flake8 src/

- name: Run Tests

run: pytest --cov=src/ --cov-fail-under=80

Quality Gates (Critérios)

  • Cobertura Mínima

    O código deve ter pelo menos 80% de cobertura de testes.

  • Zero Bugs Críticos

    Análise estática (SAST) não deve encontrar vulnerabilidades de alta severidade.

Proteção de Branch

Configuração no GitHub para impedir merge de código ruim:

Require status checks to pass before merging
Require pull request reviews before merging
