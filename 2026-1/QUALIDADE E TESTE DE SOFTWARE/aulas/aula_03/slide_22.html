ISO/IEC 25010: Modelos de Qualidade

Série SQuaRE (Systems and software Quality Requirements and Evaluation)

As 8 Características de Qualidade

  • Adequação Funcional
  • Eficiência de Desempenho
  • Compatibilidade
  • Usabilidade
  • Segurança
  • Manutenibilidade
  • Portabilidade
  • Confiabilidade

O que ela regulamenta?

Esta norma define um modelo de qualidade de produto e um modelo de qualidade em uso. Ela não diz "como" desenvolver, mas estabelece vocabulário comum e critérios para avaliar se um software tem qualidade. Ela substituiu a antiga ISO 9126, expandindo e detalhando as características não-funcionais.

Importância para QA e Testes

Para Quality Assurance (QA)

Serve como base para Engenharia de Requisitos. O QA usa a 25010 para garantir que requisitos não-funcionais (como segurança e performance) sejam definidos antes mesmo do código ser escrito.

Para Quality Control (QC)

Atua como um checklist de testes. O testador não foca apenas se o software "funciona" (funcionalidade), mas cria casos de teste para verificar se é seguro, rápido e fácil de manter.

Cenário: Um app bancário funciona (transfere dinheiro), mas demora 30 segundos para logar. Pela ISO 25010, ele falha na característica Eficiência de Desempenho, logo, não tem qualidade aceitável.
