GitHub Pull Request (PR)

github.com/projeto/pulls
O que é?

Solicitação formal para integrar alterações de código. É o coração da colaboração assíncrona.

Para que serve?

Rastreabilidade, controle de qualidade (code review) e CI/CD antes do merge final.

Fluxo de Trabalho

1. Criação da Branch

Desenvolva em uma branch isolada com commits pequenos.

2. Abertura do PR

Crie o PR com título claro, descrição e checklist de tarefas.

3. Code Review

Revisores comentam, sugerem mudanças e aprovam.

4. Merge & Close

Após aprovação e testes (CI), o código é integrado à main.
