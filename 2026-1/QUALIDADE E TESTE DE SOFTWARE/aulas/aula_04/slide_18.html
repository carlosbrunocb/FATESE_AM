Atividade Prática: The Human Linter

1. Preparação

Formar duplas (Dev & Revisor).

Criar repositório no GitHub.

Criar branches separadas para cada funcionalidade.

2. A Missão

A

Escrever uma função simples em Python contendo propositalmente 2 erros (1 de lógica, 1 de estilo/PEP8).

B

Abrir Pull Request. O parceiro deve revisar apenas olhando (sem rodar o código).

3. Critérios de Avaliação

Clareza e educação nos comentários do PR.

Uso adequado do checklist de revisão.

As sugestões são objetivas e resolvem o problema?

4. Entregáveis

Link do Pull Request no GitHub.

Lista dos problemas encontrados documentados.

Commit final com as correções aplicadas.

Desafio Mental:

Tente encontrar os erros sem usar nenhuma ferramenta automática. Compare o que você achou "no olho" vs. o que o VS Code aponta depois!
