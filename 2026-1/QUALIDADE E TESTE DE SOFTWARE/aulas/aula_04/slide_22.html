GitHub Pull Request

1. Feature Branch

Crie uma branch separada da main para trabalhar na sua funcionalidade ou correção.

2. Open PR

Faça o push e abra o Pull Request no GitHub descrevendo o que foi feito.

3. Code Review

Colegas revisam, comentam e solicitam mudanças. O CI roda os testes aqui.

4. Merge

Após aprovação e testes verdes, o código é integrado à branch principal.
