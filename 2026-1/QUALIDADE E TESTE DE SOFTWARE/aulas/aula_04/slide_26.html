Glossário: Conceitos

Walkthrough

Revisão técnica informal onde o autor guia o time pelo código ou documentação para coletar feedback e encontrar erros.

SAST

Static Application Security Testing. Teste de "caixa branca" que analisa o código-fonte em busca de vulnerabilidades de segurança sem executá-lo.

CI/CD

Continuous Integration / Continuous Delivery. Prática de automatizar a integração de código, testes e deploy frequente.

PRs (Pull Requests)

Solicitação para mesclar alterações de uma branch para outra. É o momento crucial para Code Review colaborativo.

DoD (Definition of Done)

Lista de critérios que uma tarefa deve atender para ser considerada "pronta" (ex: código revisado, testes passando, documentado).

Quality Gate

Um "portão" de qualidade no pipeline. Se o código não atingir métricas mínimas (ex: 80% de cobertura), o deploy é bloqueado.
