Metodologia TDD

O Ciclo Red-Green-Refactor

A essência do Test Driven Development em três passos contínuos

TDD
Passo 1
1

RED (Vermelho)

Escreva um pequeno teste para uma funcionalidade que ainda não existe.

Objetivo: Ver o teste falhar (garante que o teste detecta ausência da lógica).

Passo 2
2

GREEN (Verde)

Escreva o código mínimo necessário apenas para fazer o teste passar.

Objetivo: Sair do estado de erro rapidamente. Não foque em perfeição agora.

Passo 3
3

REFACTOR (Refatorar)

Melhore o código (limpeza, padrões, remoção de duplicidade) garantindo que os testes continuem passando.

Objetivo: Melhorar a qualidade do código com segurança (rede de proteção).

Repita este ciclo para cada nova pequena funcionalidade.
