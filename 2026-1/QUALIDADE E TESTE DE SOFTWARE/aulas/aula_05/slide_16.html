Aula 05 TDD Prático

Evolução e Refatoração

Novos Requisitos

1. Parametrização

Ao invés de repetir código para testar 18, 20 e 17, usamos @pytest.mark.parametrize. Isso injeta múltiplos cenários em uma única função de teste, tornando o código mais limpo e abrangente.

2. Testando Erros

O TDD também define o que não deve acontecer. Criamos um teste que espera uma exceção (ValueError) caso a idade seja negativa, garantindo robustez.

3. Refatoração (Green)

Para fazer o teste de erro passar, alteramos a implementação (sistema.py) adicionando uma validação explícita, sem quebrar os testes anteriores de maioridade.

tests/test_idade_v2.py
Novos Testes
import pytest from sistema import eh_maior_de_idade @pytest.mark.parametrize("idade, esperado", [ (18, True), (17, False), (21, True) ]) def test_fronteira_idade(idade, esperado): assert eh_maior_de_idade(idade) is esperado def test_idade_negativa_erro(): with pytest.raises(ValueError): eh_maior_de_idade(-5)
src/sistema.py
Implementação
def eh_maior_de_idade(idade: int) -> bool: # Validação adicionada para passar no novo teste (Refactor) if idade < 0: raise ValueError("Idade não pode ser negativa") return idade >= 18
