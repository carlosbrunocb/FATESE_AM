Gerenciamento de Dependências

Pip e Venv — Isolamento

Venv (Virtual Environment)

Cria um ambiente isolado para cada projeto. Isso evita conflitos entre versões de bibliotecas instaladas no sistema operacional e em outros projetos.

Pip (Package Installer)

É o gerenciador de pacotes padrão do Python. Usado para instalar, atualizar e remover bibliotecas (como o Pytest) do Python Package Index (PyPI).

Por que usar?

Garante reprodutibilidade: o arquivo requirements.txt gerado pelo pip permite que qualquer desenvolvedor instale as mesmas versões exatas das bibliotecas.

bash — Comandos Essenciais
# 1. Criar um ambiente virtual chamado "venv"
$ python -m venv venv
# 2. Ativar o ambiente (Linux/Mac)
$ source venv/bin/activate
# 2b. Ativar o ambiente (Windows)
C:\> venv\Scripts\activate
# 3. Instalar bibliotecas (ex: pytest)
(venv) $ pip install pytest
Successfully installed pytest-7.4.0 iniconfig-2.0.0 ...
# 4. Gerar lista de dependências
(venv) $ pip freeze > requirements.txt
