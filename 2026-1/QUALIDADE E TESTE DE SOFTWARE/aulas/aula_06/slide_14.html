Passo 2 — Ambiente Virtual

O que é VENV?

O módulo venv (Virtual Environment) cria ambientes leves e isolados. Cada ambiente possui sua própria instalação de binários do Python e um conjunto independente de bibliotecas.

Por que é Essencial?

Isolamento: Evita conflitos de versão entre projetos diferentes.

Limpeza: Não instala pacotes de teste no Python global do sistema.

Reprodutibilidade: Garante que todos da equipe usem as mesmas versões.

bash — setup venv
# 1. Criar o ambiente virtual (nome padrão: venv)
$ python -m venv venv
# 2a. Ativar no Linux ou macOS
$ source venv/bin/activate
# 2b. Ativar no Windows (PowerShell/CMD)
$ venv\Scripts\activate
# 3. Confirmar se o ambiente está ativo
(venv)$ which python
/home/user/lab_qualidade_software/venv/bin/python
(venv)$ pip --version
pip 23.0 from .../venv/lib/python3.10/site-packages/pip (python 3.10)
