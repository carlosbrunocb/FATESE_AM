src/clima.py 
import requests

def verificar_cancelamento_evento(cidade):
    """
    Decide se cancela evento baseado na chuva.
    Faz requisição HTTP real (lenta/instável).
    """
    url = f"https://api.clima.com/previsao/{cidade}"
    
    # Ponto Crítico: Dependência externa!
    resposta = requests.get(url)
    
    dados = resposta.json()
    
    if dados["tempo"] == "Chuva":
        return "Evento Cancelado"
        
    return "Evento Confirmado"

Teste com Pytest-Mock

Sem Internet
tests/test_clima.py 
def test_deve_cancelar_evento_se_chover(mocker):
    # 1. Arrange (Preparação do Mock)
    # Criamos um "dublê" para a resposta
    mock_response = mocker.Mock()
    # Definimos o que o .json() deve retornar
    mock_response.json.return_value = {"tempo": "Chuva"}
    
    # "Patcheamos" o requests.get para retornar nosso dublê
    # Isso intercepta a chamada antes de ir para a rede!
    mocker.patch("requests.get", return_value=mock_response)
    
    from src.clima import verificar_cancelamento_evento
    
    # 2. Act (Ação) - Chama a função (agora isolada)
    resultado = verificar_cancelamento_evento("Manaus")
    
    # 3. Assert (Verificação)
    assert resultado == "Evento Cancelado"

Conceito Chave: Monkey Patching

O mocker.patch substitui dinamicamente o objeto requests.get pelo nosso Mock durante a execução do teste. Ao final do teste, o pytest restaura o objeto original automaticamente.
