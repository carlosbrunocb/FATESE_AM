Topo: Sistema / E2E 10% dos testes

Simula o usuário real. É caro de manter e lento de executar. Minimizar uso de Mocks aqui para garantir que o sistema funciona no mundo real.

Meio: Integração 20% dos testes

Foco na comunicação entre módulos (ex: API ↔ Banco). Usa Mocks e Stubs para isolar dependências externas lentas ou instáveis.

Base: Unidade 70% dos testes

Testa a lógica isolada (funções, classes). Extremamente rápido. Use Mocks livremente para simular qualquer dependência externa.

Dica de Ouro: Mocks com Parcimônia

Mocks são excelentes para Integração (velocidade), mas perigosos em E2E (falso positivo). Se você mockar tudo no teste de sistema, não estará testando se o software realmente funciona em produção!
