O que é Teste de Caixa-Preta?

Técnica de teste funcional que valida o comportamento do sistema a partir de entradas e saídas observáveis, sem analisar a implementação interna.

Base

Requisitos e regras de negócio

Caixa-preta = “o quê” (comportamento)

Ignora “como” (código, arquitetura)

Modelo mental: Entrada → Sistema → Saída

O testador interage pela interface (UI/API/serviço) e verifica se a saída corresponde ao resultado esperado do requisito.

Diagrama (visão funcional)

Black-box

Entradas

dados, ações

Sistema

(opaco)

Saídas

mensagens, dados

Pergunta-guia: “Dada esta entrada, qual saída o requisito prevê?”

Características principais

Casos de teste são derivados de critérios de aceitação, domínios de dados e comportamento esperado.

Entrada

Válida / inválida

formatos, faixas, categorias

Saída

Comportamento observado

status, mensagens, dados

Onde é mais usado?

Testes de aceitação (UAT) e validação funcional

Confirma se o sistema atende às regras do negócio.

Testes de API / integração por contrato

Entrada/saída, códigos de resposta e payloads.

Detecção de falhas em regras e validações

Especialmente com EP e BVA (tema da aula).

Limitação: não garante cobertura de caminhos internos do código (complementa caixa-branca).
