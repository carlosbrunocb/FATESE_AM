Execução Seletiva com Markers
Filtrando e executando testes de regressão de forma inteligente
Em prática
Exemplo de Marcação de Testes
Python
import pytest
from src.calculadora import somar, dividir, multiplicar

# Marcadores de regressão
@pytest.mark.regressao
@pytest.mark.critico
def test_soma_positiva():
    assert somar(2, 3) == 5

@pytest.mark.regressao
def test_multiplicacao():
    assert multiplicar(10, 5) == 50

@pytest.mark.slow
def test_calculo_complexo():
    # Teste que leva mais tempo
Dica: Use @pytest.mark para categorizar testes por tipo, criticidade ou tempo de execução.
Comandos de Filtragem
CLI
pytest -m regressao -v
Executa apenas regressão
pytest -m "regressao and critico" -v
Combina marcadores
pytest -k "soma or divisao" -v
Filtro por nome
Configuração pytest.ini
Config
# pytest.ini
[pytest]
markers =
    regressao: testes de regressão críticos
    critico: alta severidade/negócio
    smoke: verificação rápida de saúde
    slow: testes que levam mais tempo

filterwarnings =
    ignore::DeprecationWarning
Boas Práticas
• Registre markers no pytest.ini para evitar warnings
• Use execução seletiva em PRs
• Rode suíte completa em releases importantes
Recomendações
  • Use -m para filtrar por marcadores
  • Combine marcadores com and/or
  • Use -k para filtrar por nome de função
  • Priorize testes críticos em CI/CD
