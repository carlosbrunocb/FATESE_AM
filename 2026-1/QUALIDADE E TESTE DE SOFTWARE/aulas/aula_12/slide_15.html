User Story e Critérios de Aceite
Pagamento via PIX - Validação de requisitos funcionais e não funcionais
User Story
Como cliente,
quero pagar com PIX para que o pedido seja aprovado instantaneamente,
para que eu possa receber meu produto mais rápido.
Critérios de Aceite (CA)
1
O QR Code deve ser gerado em até 2 segundos em 95% das tentativas Alta
2
Status do pedido deve mudar para "Pago" imediatamente após confirmação do banco Alta
3
Exibir mensagem clara se banco não responder em 60s Média
4
Registrar no log ID da transação do PSP e timestamp Média
5
Tela de pagamento deve atender acessibilidade (contraste AA) Baixa
Formato BDD (Given/When/Then)
Dado que estou na tela de pagamento de um pedido de R$ 50,00
E possuo uma conta PIX válida cadastrada
Quando seleciono PIX e confirmo o pagamento
Então o QR Code deve ser exibido em até 2 segundos
E o status do pedido deve ser atualizado para "Pago"
Regras de Negócio
Se valor = R$ 0,00
→ Ocultar opção PIX
Se 3 falhas consecutivas
→ Marcar como pendente
Se pagamento cancelado
→ Invalidar QR Code
Se banco não responde
→ Timeout em 60s
