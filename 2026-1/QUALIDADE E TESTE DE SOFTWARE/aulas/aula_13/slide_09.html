Como o WebDriver Funciona

Fluxo de funcionamento em 5 passos sequenciais

Em execução
1
Criar Sessão de Navegador
O script inicializa uma nova instância do navegador, estabelecendo uma sessão de comunicação entre o código e o WebDriver.
webdriver.Chrome()
2
Enviar Comandos HTTP
O script envia comandos via protocolo HTTP/bi-direcional para o driver, especificando ações como clique, digitação ou navegação.
HTTP/JSON
3
Driver Aciona Engine do Navegador
O WebDriver interpreta os comandos e os traduz para ações nativas do navegador, acionando a engine de renderização.
ChromeDriver
4
DOM é Manipulado e Renderizado
O navegador executa as ações no DOM, atualizando a interface e renderizando as mudanças visuais na página web.
DOM Update
5
Respostas Retornam ao Script
O navegador retorna estados, resultados e dados para o script, permitindo validações e assertions.
driver.title
