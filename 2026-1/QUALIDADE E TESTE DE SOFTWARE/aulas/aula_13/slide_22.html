Boas Práticas em Automação

Recomendações para criar testes robustos e manteníveis

Prática
Locators Estáveis
Use ID ou data-test-id quando disponíveis
Evite XPaths absolutos e frágeis
Prefira CSS Selectors sobre XPath
By.ID, "login-btn"
Esperas Explícitas
Use WebDriverWait ao invés de time.sleep()
Defina timeouts adequados (10-30s)
Aguarde condições específicas
WebDriverWait(driver, 10)
Page Object Model
Separe a lógica da página em classes
Crie métodos para cada ação
Reutilize elementos entre testes
class LoginPage:
Tratamento de Erros
Capture exceções específicas
Tire screenshots em caso de falha
Implemente retries quando necessário
try/except blocks
Logs e Evidências
Registre logs detalhados
Capture screenshots de falhas
Documente resultados de testes
driver.save_screenshot()
Testes Independentes
Cada teste deve ser autossuficiente
Evite dependências entre testes
Resete estado após cada teste
setUp() / tearDown()
Dica: Sempre mantenha seus testes atualizados com as mudanças da aplicação e revise regularmente a cobertura de testes.
