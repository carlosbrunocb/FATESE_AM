Contextualização Teórica Qualidade e Teste de Software

O Desafio da Web Assíncrona

Como o AJAX, React, Angular e Vue impactam diretamente a estabilidade dos testes de software.

Por que testes falham sem motivo aparente?

Em aplicações modernas, ao interagir com a interface, a página não é recarregada por completo. Apenas pedaços do DOM (Document Object Model) são atualizados de forma assíncrona após chamadas de API ao servidor.

As Exceções Temidas na Automação:

  • NoSuchElementException: Tentativa de ler ou interagir com um elemento que ainda não foi renderizado na árvore do DOM.
  • ElementNotInteractableException: O elemento está presente no HTML, mas ainda não está visível, habilitado ou pronto para receber cliques na tela.
👻

Isso cria os chamados Flaky Tests (testes instáveis que ora passam, ora falham sem alteração no código de produção).

A Corrida de Corrida de Impedância (Race Condition)

[Cenário de Teste Tradicional na Automação]
1. Script: Clica no botão "Entrar"
2. Browser: Dispara requisição HTTP assíncrona ao servidor da API...
3. Script (Tentativa imediata): Procura campo "Logout" para validar
>> ERRO: NoSuchElementException! (A API ainda não respondeu, a página não mudou)

Como resolver sem inserir atrasos fixos desnecessários?

Utilizar atrasos cegos (time.sleep(5)) é considerado uma péssima prática na automação profissional de testes, pois atrasa a execução geral da pipeline e não garante a resolução do problema caso a rede oscile.
