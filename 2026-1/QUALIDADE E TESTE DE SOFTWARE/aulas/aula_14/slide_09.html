Mapeamento de Elementos Qualidade e Teste de Software

Estratégias de Seletores (Locators)

Como escolher a melhor estratégia de localização para evitar a quebra dos seus scripts.

Principais Métodos de Localização (By)

Seletor Sintaxe Selenium Estabilidade
By.ID By.ID, "username" Máxima (Invejável)
By.NAME By.NAME, "password" Alta
By.CSS_SELECTOR By.CSS_SELECTOR, "button[type='submit']" Alta / Média
By.XPATH By.XPATH, "//div[@id='login']/button" Média (Usar com cautela)
By.CLASS_NAME By.CLASS_NAME, "flash.success" Baixa (Muito volátil)

A Regra de Ouro do Mapeamento

1. Priorize IDs e Names

IDs de elementos devem ser semanticamente únicos em uma página HTML bem escrita. Mapear por ID garante que seu teste não quebrará se a folha de estilo ou a ordem dos elementos mudarem.

2. Evite XPaths Absolutos

XPaths estruturais longos como /html/body/div[2]/div/div/div/form/button quebram com facilidade com qualquer mudança trivial de layout. Utilize caminhos relativos robustos.

3. Use Atributos Customizados

Em ambientes empresariais robustos, é comum criar identificadores específicos para automações de testes de front-end, como o data-testid="login-submit".
