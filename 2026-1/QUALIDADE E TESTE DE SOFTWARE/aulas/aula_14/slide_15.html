Desafio Extra

Mapeando o Fluxo de Exceção (Login Inválido)

Seu objetivo é criar uma nova função de teste ou estender o script atual para simular o comportamento de um usuário real digitando credenciais inválidas.

Requisitos do Desafio:

  1. Enviar intencionalmente dados incorretos: Usuário "usuario_errado".
  2. Mapear e aguardar o elemento de mensagem de erro gerado pelo sistema.
  3. Realizar uma asserção assert para garantir que o texto de erro "Your username is invalid!" é exibido na tela.

💡 Dica técnica: O seletor do alerta de erro permanece o mesmo do alerta de sucesso? Verifique no DevTools!

Por que testar fluxos negativos?

Um software robusto não é apenas aquele que funciona no caminho feliz. Ele deve gerenciar, tratar e exibir erros de forma limpa e informativa para os usuários, sem quebrar internamente.

Apoio Supervisionado

O professor estará circulando pelas bancadas do laboratório de informática para guiar e apoiar problemas de seleção com XPath e CSS Selector customizados para esse desafio.
