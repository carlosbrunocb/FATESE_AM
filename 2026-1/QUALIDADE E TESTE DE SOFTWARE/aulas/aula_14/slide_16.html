Procedimentos e Avaliação

Como sua entrega prática será monitorada, avaliada e integrada à sua nota processual (AV2).

Critérios de Avaliação Contínua

1

Versionamento no GitHub

O script desenvolvido em aula deve obrigatoriamente passar por um commit e push para o repositório individual criado para o projeto de testes.

2

Assertividade dos Seletores

Uso correto de seletores de alta estabilidade (ID, Name, CSS Selector relativos) em vez de XPaths absolutos voláteis que quebram facilmente.

3

Resiliência do Script

Implementação correta do sincronismo assíncrono (WebDriverWait) evitando o uso de pausas cegas como time.sleep().

Entrega e Próximos Passos

Esta atividade laboratorial prática é de extrema relevância, pois comporá a nota processual complementar da avaliação da segunda unidade (AV2), dando apoio de fundamentação técnica para a entrega final do seu projeto integrador.

Comandos Git sugeridos no final:

git add test_login_fluxo.py
git commit -m "feat: automacao fluxo login e logout aula 14"
git push origin main
🎯

Garante a portabilidade: confira se o arquivo .gitignore está configurado corretamente para ignorar a pasta venv/.
