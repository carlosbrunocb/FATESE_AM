Execução e Análise no Locust

Como executar testes de carga e analisar os resultados
1
Iniciar Teste
Configurar parâmetros
Defina número de usuários, taxa de spawn e tempo de execução.
# Iniciar com 500 usuários
locust -f locustfile.py
# Acessar localhost:8089
users = 500
spawn_rate = 10
2
Monitorar Métricas
RPS, Latência, Erros
Acompanhe em tempo real: RPS, latência (p95/p99), taxa de erros.
# Métricas principais
RPS: Requisições/seg
Latency: p95/p99
Failures: 4xx/5xx
3
Identificar Gargalos
Pontos de falha
Observe degradação de performance e identifique limites.
# Analisar gargalos
CPU > 80%
Memory > 90%
Response > 3s
4
Análise de Erros
HTTP 4xx/5xx
Verifique tipos de erros e causas de falha.
# Tipos de erro
4xx: Client Error
5xx: Server Error
Timeout: > 30s
5
Gerar Relatório
Exportar resultados
Exporte relatórios detalhados para análise posterior.
# Gerar relatório
locust -f locustfile.py
--csv resultado.csv
--html relatorio.html
6
Otimização
Melhorias de performance
Implemente melhorias baseadas nos resultados.
# Otimizações
Cache: Redis
DB: Indexação
CDN: Static files

1,250
Latência
45ms
Usuários
500
Erros
0.2%
