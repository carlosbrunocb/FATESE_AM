Definition of Done (DoD): O Acordo Coletivo

A Definition of Done (DoD) é o checklist formal de critérios que uma funcionalidade precisa obrigatoriamente cumprir para ser considerada verdadeiramente "Pronta" pelo time.

Diferente dos Critérios de Aceitação (específicos de cada história), a DoD se aplica de forma horizontal a todas as tarefas da Sprint, elevando os padrões de qualidade técnicos da organização de forma consistente.

CHECKLIST TÍTICO DE UMA DoD ÁGIL
[✓] Código revisado via Pull Request por outro desenvolvedor
[✓] Testes Unitários criados e executando com sucesso
[✓] Cobertura de Testes (Coverage) acima de 80%
[✓] Funcionalidade testada em ambiente de Staging / Homologação
[✓] Pipeline de testes de Integração Contínua (CI) verde (sem falhas)

A Definição de "Pronto"

FASE DE ESPECIFICAÇÃO
User Story (História)
Foco em valor e critério de aceitação de negócio
FASE DE CONSTRUÇÃO
Codificação & Automação
Devs escrevem lógica, QAs criam testes funcionais
FATOR DE CORTE OBRIGATÓRIO
Definition of Done (DoD)
Nenhuma funcionalidade pode passar para Produção se não atender 100% à DoD estabelecida coletivamente.
ENTREGA FINAL
Produção / Deployment
Incremento liberado de forma limpa e confiável
