O Funcionamento da Esteira de CI/CD

A esteira de CI/CD (Continuous Integration / Continuous Delivery) é o mecanismo de engenharia que automatiza a passagem do código pelos estágios de qualidade até chegar ao usuário.

O papel do QA é projetar os gatilhos, as baterias de teste corretas para cada estágio da esteira e garantir feedbacks eficientes que permitam lançamentos sem riscos.

CI - Integração Contínua

Foco no desenvolvedor. Executa compilação, análise de qualidade estática (linters) e testes unitários instantaneamente a cada commit. Garante que novos códigos não quebrem as funções existentes.

CD - Entrega Contínua

Foco na publicação. Automação do empacotamento, envio para ambientes de homologação (Staging), execução de testes integrados e posterior implantação automatizada em Produção.

[REGRA DE OURO] O pipeline quebrado interrompe imediatamente o fluxo de entrega. Nenhum deploy ocorre até que o código seja corrigido.

Estágios do Pipeline de Automação

Passo 1
Git Push / Pull Request
Desenvolvedor envia código para a branch central.
Passo 2
Build & Análise Estática
Servidor compila o código e roda linters de formatação.
Passo 3
Execução de Testes Automatizados
Fase Crítica: Pytest executa toda a suite de testes.
Passo 4
Deploy Automatizado (CD)
Se aprovado, o código vai para homologação ou produção.
