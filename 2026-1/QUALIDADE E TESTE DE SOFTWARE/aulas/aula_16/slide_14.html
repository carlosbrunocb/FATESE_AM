PRÁTICA LABORANTE

Laboratório Prático: Visão Geral e Estrutura

Vamos implementar, na prática, uma esteira básica de Integração Contínua (CI) utilizando o repositório GitHub da calculadora em Python que desenvolvemos na Aula 11.

O objetivo é simular o dia a dia de uma empresa DevOps: cada push ou pull request na branch principal irá disparar testes automatizados na nuvem em uma máquina virtual limpa.

PASSO 1: ESTRUTURA DO REPOSITÓRIO LOCAL

Certifique-se de que a estrutura do seu projeto local do Git esteja exatamente com a árvore de diretórios ocultos exigida pelo GitHub Actions: 

meu-projeto-calculadora/
├── .github/
│   └── workflows/
│       └── ci-testes.yml  <-- Criar este arquivo
├── src/
│   └── calculadora.py
└── tests/
    └── test_calculadora.py

Passo a Passo Prático

Passo 2: Criação das Pastas Ocultas

No diretório raiz da sua calculadora, crie a pasta .github e, dentro dela, crie a subpasta workflows.

Passo 3: Escrever o ci-testes.yml

Crie o arquivo ci-testes.yml e cole a configuração YAML que define as etapas e passos da execução automática.

Passo 4: Push e Acompanhamento

Realize o commit e envie as alterações para o seu repositório remoto no GitHub. Assista ao pipeline executar em tempo real na aba Actions do GitHub.
