Configuração da Esteira: ci-testes.yml

Análise Lógica do Script

O formato YAML dita os passos que o servidor do GitHub irá executar a cada modificação do código de forma transparente.

on: push / pull_request
Define os gatilhos: dispara a esteira quando há push ou alteração de código na branch principal.
runs-on: ubuntu-latest
Determina o ambiente: inicia uma máquina virtual Linux vazia para rodar a aplicação isoladamente.
actions/checkout@v3
Baixa os códigos do repositório para o disco rígido da máquina virtual recém-criada.
pytest -v
Comando final de QA: roda toda a suíte de testes de forma verbosa (-v), exibindo sucesso ou erro detalhadamente.
CONTEÚDO DO ARQUIVO: .github/workflows/ci-testes.yml YAML SPEC 
name: Pipeline de Testes Contínuos (CI)

# Gatilho: Executa sempre que houver um push ou PR na branch main
on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  executar-testes:
    runs-on: ubuntu-latest

    steps:
    # Passo 1: Baixa o código do repositório para a máquina virtual
    - name: Copiar o código do repositório
      uses: actions/checkout@v3

    # Passo 2: Configura o ambiente com a linguagem Python 3
    - name: Configurar o Python no ambiente
      uses: actions/setup-python@v4
      with:
        python-version: '3.10'

    # Passo 3: Instala as dependências de teste (Pytest)
    - name: Instalar dependências de QA
      run: |
        python -m pip install --upgrade pip
        pip install pytest

    # Passo 4: Executa a suite de testes de forma automatizada
    - name: Rodar os testes com Pytest
      run: |
        pytest -v
