PRÁTICA LABORANTE

Laboratório Prático: Executando a Cultura DevOps

1. ENVIANDO O TRABALHO (TERMINAL)

Agora que o arquivo YAML foi criado e salvo na pasta correta, execute a sequência de comandos Git no seu terminal para enviar as alterações para nuvem: 

$ git add .
$ git commit -m "feat: adicionando esteira de CI"
$ git push origin main
2. ACOMPANHAMENTO VISUAL NO GITHUB

Acesse o seu repositório no site do GitHub e clique na aba "Actions".

  • Observe a máquina virtual ligando.
  • Assista ao passo-a-passo da instalação do Python e Pytest.
  • Se o pipeline exibir Verde: Sucesso total na qualidade.
  • Se exibir Vermelho: Um teste quebrou; investigue os logs!

Procedimentos de Avaliação

Avaliação de Processo

Os alunos serão avaliados pela correta configuração do arquivo YAML e pela execução bem-sucedida do pipeline na nuvem.

Fechamento do Módulo V

Esta aula amarra as pontas soltas da automação. Escrever um script de teste local só tem valor real de mercado se ele for integrado a uma rotina automática e contínua de desenvolvimento (CI/CD).

Dica Importante

Se o seu pipeline falhar, clique no job do workflow e examine a saída do Pytest. Ele indicará exatamente qual teste quebrou e qual linha de código gerou o erro de qualidade.
