Exemplo Prático 1: Cálculo em E-Commerce

Caso prático de release-gate para consolidação dos conceitos de Densidade

Cenário Proposto

Durante a fase final de homologação de uma nova release de um e-commerce, o time de engenharia de testes (QA) detectou e confirmou 45 bugs válidos. O relatório de análise estática indicou que a release possui um tamanho de 15.000 linhas de código (15 KLOC).

Cálculo Detalhado

Densidade = 45 Defeitos / 15 KLOC = 3.0

Resultado: 3,0 defeitos por mil linhas de código (Defects / KLOC)
🚫

Tomada de Decisão (Release-Gate)

Métrica Aceitável da Empresa: ≤ 1.0 bug / KLOC
Métrica Obtida na Release: 3.0 bugs / KLOC
Status do Release-Gate: REPROVADO (BLOCKING)

Análise do Professor Carlos: Com 3 bugs por KLOC, a release está 300% acima do patamar tolerável de risco. A versão atual está extremamente instável. O Go-to-Market deve ser barrado imediatamente e o código deve retornar para a esteira de correção e regressão massiva dos desenvolvedores antes de ir para produção.

Resultado: liberação bloqueada por evidência analítica quantitativa
