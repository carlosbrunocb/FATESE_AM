Densidade Ponderada pela Gravidade dos Defeitos

Como refinar a medição aplicando pesos diferentes baseados no impacto do erro

Por que usar pesos de severidade?

Contar um erro de grafia ou layout (Severidade Baixa) com o mesmo peso de uma falha de banco de dados que corrompe transações (Severidade Crítica) gera distorções.

A Densidade Ponderada atribui pontuações para cada bug de acordo com seu impacto sistêmico real na aplicação.

Fórmula Avançada: Densidade Ponderada = Σ (Nº de Bugs × Peso Severidade) / KLOC

Escala Padrão de Pesos da Indústria

Severidade Peso Definição / Impacto real
Crítico (Critical) 5.0 Trava o sistema todo, gera perda de dados ou brecha grave de segurança.
Alto (High) 3.0 Funcionalidade core não executa e não há contorno simples.
Médio (Medium) 1.0 Problemas visíveis, mas com workarounds viáveis.
Baixo (Low) 0.5 Erros visuais mínimos ou pequenos bugs de grafia.

Nota: Permite calcular o "Índice de Toxicidade" real de uma release, evitando distorções em auditorias.
