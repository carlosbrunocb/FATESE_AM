Modelo OSI – 7 camadas

Ideia central

Cada camada tem uma responsabilidade; juntas, entregam dados ponta a ponta.

Regra prática

Para enviar, o dado “desce” (encapsula). Para receber, “sobe” (desencapsula).

Onde você atua mais

Camadas 7–6–5 (mais “perto do código”)

Você define protocolo, formato de dados, autenticação, compressão e criptografia (via bibliotecas).

HTTP

headers • métodos • status

TLS (HTTPS)

certificado • cifra • handshake

Sessão

login • tokens • state

Mas problemas comuns aparecem na Camada 4

Porta errada, timeout, conexão resetada, retransmissão e performance são sintomas “de transporte”.

Exemplo

8080 vs 80

Efeito

connection refused

Dica de estudo

Use este slide como “legenda” para identificar onde cada informação (porta, IP, MAC) entra no pacote.
