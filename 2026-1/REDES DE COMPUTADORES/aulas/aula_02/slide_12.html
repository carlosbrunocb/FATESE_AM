Atividade prática

Simulação do fluxo de dados

app → transporte → rede → enlace → bits

encapsula / desencapsula

Objetivo

Encenação guiada: transformar uma requisição HTTP em segmento TCP, pacote IP, quadro Ethernet/Wi‑Fi e, por fim, bits — e depois fazer o caminho de volta (servidor).

trabalho em grupo

Papéis (grupos)

Cada grupo “representa” uma parte da pilha.

4–6 alunos/grupo

Navegador (App)

HTTP / URL

Mensagem

Cria a requisição GET e escolhe headers.

SO (TCP)

portas / seq / ACK

Segmento

Anexa porta origem/destino e garante entrega (TCP).

Roteador (IP)

saltos / gateway

Pacote

Encaminha pelo IP destino; “próximo salto” muda a cada rede.

Switch/Enlace

MAC / FCS

Quadro

Entrega no enlace local e valida integridade (FCS).

Servidor (App)

Faz o caminho inverso (desencapsula) e interpreta HTTP para responder.

Materiais

Para “montar” o pacote em sala

cartões

Cartões de headers

HTTP • TCP • IP • Ethernet (+ FCS)

Apoio visual

setas, fita/ímãs, quadro e canetas

Opcional

projetor com pilha OSI/TCP-IP

Entregáveis

O que cada grupo deve produzir

1) Linha do tempo do pacote

Ordem correta: Mensagem → Segmento → Pacote → Quadro → Bits.

2) “Legenda” dos campos

Onde entram: Porta, IP, MAC e FCS.

3) Mini-relato

“O que o dev vê” vs. “o que a rede adiciona”.

Roteiro

Execução (descida e subida)

1

Aplicação cria a mensagem

Escrever uma requisição: GET /index.html + headers.

2

Transporte adiciona TCP

Definir porta origem (efêmera) e porta destino (80/443).

3

Rede adiciona IP

Colar IP origem/destino e explicar o papel do gateway.

4

Enlace/Física: quadro e bits

Adicionar MAC do próximo salto + FCS e “converter” em bits.

Subida (no servidor): remover headers

Validar FCS → ler IP → entregar ao TCP → app interpreta HTTP e responde.

Perguntas de reflexão (ADS)

Debate rápido em sala

1) 8080 vs 80: em qual camada está o erro?

Resposta esperada: Transporte (portas TCP/UDP).

2) HTTPS “embaralha” onde?

Resposta esperada: Aplicação/Apresentação (TLS).

3) Como ver os headers “de verdade”?

Resposta esperada: captura no Wireshark.
