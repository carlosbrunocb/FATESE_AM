Aula 05

IPv4

Broadcast e faixa de hosts úteis

Como identificar o último IP da rede e o intervalo utilizável

Base teórica • Bits de host = 1 (broadcast) / intervalo de hosts

Definições rápidas

Depois de achar a rede, vem o broadcast e os hosts

Regra por bits

Endereço de broadcast

É o endereço para enviar para todos os hosts da sub-rede. Para calculá-lo: todos os bits de host = 1.

Rede = bits de host 0 → Broadcast = bits de host 1

Faixa de hosts úteis

Hosts utilizáveis ficam entre o endereço de rede e o broadcast:

Primeiro host

Rede + 1

Último host

Broadcast − 1

Atenção

Em IPv4 tradicional, rede e broadcast não podem ser atribuídos a hosts. (Em cenários especiais, existem exceções, mas não é o padrão em LANs.)

Dica de prova/lab

Se a máscara for /24, o broadcast normalmente termina em .255 (host de 8 bits).

Exemplo: 10.1.5.200/16

Rede já calculada: 10.1.0.0

/16 = 255.255.0.0

1

Identifique o tamanho da porção de host

Em /16, os 16 últimos bits são de host (os dois últimos octetos).

2

Calcule o broadcast

Coloque todos os bits de host = 1 → últimos dois octetos viram 255.255.

Broadcast = 10.1.255.255

3

Determine a faixa de hosts úteis

Primeiro host

10.1.0.1

Último host

10.1.255.254

Total de hosts úteis: 2(32 - 16) − 2 = 216 − 2 = 65.534

Resumo do exemplo

Rede

10.1.0.0

Broadcast

10.1.255.255

Hosts

.1 a .254

Checklist mental

Broadcast = host 1 | Hosts úteis = (rede+1) até (broadcast−1)

host: 000…0 → 111…1
