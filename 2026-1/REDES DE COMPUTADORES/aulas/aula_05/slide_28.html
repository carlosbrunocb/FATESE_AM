Prática: identifique seu IP local

Coleta de dados reais (IP, máscara/prefixo, gateway e DNS) e verificação básica

Mãos na massa • Dados da sua máquina

Checklist de coleta

Anote e use no cálculo de rede/broadcast

passos

1

Execute o comando no seu sistema

Windows: ipconfig /all • Linux: ip a

2

Anote os 4 dados essenciais

IPv4

{seu IPv4}

Máscara / Prefixo

{255.255... ou /24}

Gateway

{seu gateway}

DNS

{DNS}

3

Calcule (no caderno)

Rede, broadcast e faixa de hosts úteis usando a máscara/prefixo.

4

Valide conectividade com ping

Teste o gateway (e não o broadcast).

ping {gateway}

Atenção

Algumas redes bloqueiam ICMP. Se o ping falhar, valide com outro teste (ex.: abrir o gateway no navegador, se aplicável) e verifique firewall.

Comandos rápidos

Copie e cole no terminal

referência

Windows

PowerShell / Prompt

ipconfig /all

ping {gateway}

Linux

iproute2

ip a

ip route

ping {gateway}

Interfaces

Se houver VPN/Wi‑Fi, escolha a interface ativa (status UP).

Boas práticas

Não compartilhe IP público em prints; prefira ocultar partes.

Dica

Se a máscara aparecer como /24, lembre: /24 = 255.255.255.0.

Entregável rápido:

Informe seu IPv4/prefixo + rede + broadcast + faixa de hosts (cálculo manual).
