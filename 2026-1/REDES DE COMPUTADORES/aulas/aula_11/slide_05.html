Resolução de Nomes DNS

Consultas Recursivas vs Iterativas

Processo de Resolução
Consulta Recursiva (Padrão do Cliente)

O resolver (servidor DNS do ISP) faz todo o trabalho de resolução para o cliente.

1
Cliente → Resolver (RD=1)
Cliente envia consulta com flag RD (Recursion Desired) = 1
2
Resolver pergunta ao Root
Servidor resolver consulta servidor raiz
3
Root retorna referência TLD
Servidor raiz aponta para servidor TLD
4
Resolver pergunta ao TLD
Consulta servidor TLD (.com, .br, etc)
5
TLD retorna referência autoritativo
Obtém endereço do servidor autoritativo
6
Resolver pergunta ao autoritativo
Consulta servidor autoritativo do domínio
7
Resolver devolve resposta ao cliente
RA=1 (Recursion Available) + cache para futuras consultas
Vantagem: Simplicidade para o cliente | Desafio: Carga no resolver
Consulta Iterativa (Cliente Itera)

O cliente (ou resolver) recebe respostas de referência e itera manualmente.

1
Cliente → Root (Iterativa)
Cliente pergunta diretamente ao servidor raiz
2
Root retorna referência TLD
Recebe lista de servidores TLD para .com
3
Cliente → TLD (Iterativa)
Cliente pergunta diretamente ao TLD
4
TLD retorna referência autoritativo
Recebe endereço do servidor autoritativo
5
Cliente → Autoritativo (Iterativa)
Cliente consulta servidor autoritativo
6
Autoritativo retorna resposta final
Recebe o registro DNS solicitado (A, CNAME, etc)
Vantagem: Controle do cliente | Desafio: Complexidade
