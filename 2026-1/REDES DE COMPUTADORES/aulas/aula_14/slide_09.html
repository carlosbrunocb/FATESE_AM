Mecanismo RTS/CTS
Processo RTS/CTS
1. Request to Send (RTS)
STA envia RTS ao AP com duração prevista da transmissão
2. Clear to Send (CTS)
AP responde com CTS, reservando o canal (NAV nos vizinhos)
3. Transmissão de Dados
STA envia dados para o AP
4. ACK de Confirmação
AP confirma recebimento com ACK
Diagrama de Rede
STA
RTS/CTS
AP
Duração RTS
~20 bytes
Duração CTS
~14 bytes
Timeout
2 x SIFS
Limite RTS
2346 bytes
Por que usar RTS/CTS?
Útil para quadros grandes e ambientes congestionados. O overhead é compensado pela redução de colisões.
