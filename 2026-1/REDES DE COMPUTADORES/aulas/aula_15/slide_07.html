1ª Geração: Filtragem de Pacotes (Stateless) Arquitetura

Como Funciona o Filtro Sem Estado?

O firewall Stateless (Sem Estado) analisa cada pacote de rede de forma totalmente isolada. Ele toma decisões de liberação ou bloqueio baseando-se apenas nas informações contidas nos cabeçalhos das camadas 3 (IP) e 4 (TCP/UDP/ICMP).

O filtro não tem memória ("estado"). Ele desconhece se o pacote é o início de uma nova conexão, parte de uma conexão legítima já estabelecida ou um tráfego forjado.

Vantagens & Limitações Clarass

  • Extrema Rapidez: Processamento na velocidade do hardware (ASICs), ideal para bordas de alta performance.
  • Vulnerável a Spoofing: Permite pacotes forjados com facilidade.
  • Regras Complexas: Para conexões de ida e volta, é obrigatório criar regras explícitas para ambos os sentidos.

A Tabela de Decisão do Filtro

Exemplo prático de regras aplicadas em um roteador de borda stateless:

Regra IP Origem Porta Or. IP Destino Porta Des. Ação
R1 Qualquer * 192.168.1.50 80 (HTTP) Permitir
R2 10.0.0.99 * Qualquer 22 (SSH) Permitir
R3 Qualquer * 192.168.1.100 23 (Telnet) Negar
R4 (Padrão) Qualquer * Qualquer * Negar (Implícito)
Nota de Engenharia:

Para que o servidor Web retorne os dados ao cliente na porta 80, o administrador deve criar uma regra permitindo portas de retorno (ex: >1023), criando brechas de segurança perigosas.
