3ª Geração: Application Gateway (Proxy) Arquitetura

Inspeção Completa na Camada 7

O Application Gateway (Proxy de Aplicação) opera no nível mais alto do Modelo OSI (Camada 7 - Aplicação). Em vez de simplesmente deixar os pacotes passarem, o proxy rompe a conexão física direta entre a origem e o destino.

O cliente interno fecha uma conexão TCP com o Proxy; este, por sua vez, valida os dados e inicia uma nova conexão separada com o servidor de destino na internet. O tráfego direto não existe.

Controle de Conteúdo Sem Igual

  • Filtro Semântico: Impede comandos SQL maliciosos escondidos em portas normais (80/443).
  • Anonimato Completo: O IP real do servidor interno nunca é revelado à internet, apenas o IP do proxy.

Como o Proxy Intermedeia

Diferença estrutural na transferência de dados:

Arquitetura de Conexões Duplas:
[Máquina Aluno] --(Conexão A)--> [Proxy Firewall] --(Conexão B)--> [Servidor Web]

O Proxy reconstrói o payload da aplicação para validação profunda, protegendo ativamente os endpoints de exploits complexos.

Os Custos da Camada de Aplicação

  • Alto Overhead de Hardware: Exige muita memória e processamento para descriptografar e analisar o conteúdo completo de cada arquivo e transação.
  • Incompatibilidade de Protocolos: Um proxy precisa ser desenvolvido especificamente para cada protocolo de aplicação (HTTP, FTP, SMTP, DNS). Protocolos novos ou personalizados falham.
