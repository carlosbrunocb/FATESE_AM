O que são Máscaras Curinga (Wildcard Masks)? Conceitos de Rede

Definição e Semântica Inversa

Ao contrário das máscaras de sub-rede tradicionais (usadas para definir IPs de interfaces), as Máscaras Curinga (Wildcard Masks) são usadas em ACLs para especificar quais bits de um endereço IP devem ser inspecionados ou ignorados.

Elas operam de forma estritamente inversa à lógica de uma máscara de sub-rede convencional.

As Duas Regras Binárias do Curinga

  • Bit 0 (Match): Significa "Verifique este bit". Ele deve ser idêntico ao IP configurado.
  • Bit 1 (Don't Care): Significa "Não me importo". O valor deste bit pode ser qualquer um (0 ou 1).

Visualização de Bits do Curinga

Exemplo prático de aplicação para a sub-rede 192.168.10.0 / 24:

Endereço IP da Rede:
192.168.10.0
Máscara de Sub-rede:
255.255.255.0
Máscara Curinga (Inversa):
0.0.0.255
Análise Binária do Octeto Curinga:
Octeto Decimal 0 (Primeiros 3 octetos) 255 (Último octeto)
Valor Binário 00000000 11111111
Significado Verificação Obrigatória (Match) Qualquer valor é aceito (Ignore)
Resultado Prático: O firewall irá testar se o tráfego começa obrigatoriamente com 192.168.10. e ignorará qualquer valor no último octeto (de .1 a .254).
