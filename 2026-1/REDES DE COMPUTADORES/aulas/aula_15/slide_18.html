Sintaxe Cisco: ACL Extended (Estendida) Comandos & CLI

Estrutura Geral de Comandos

A ACL Estendida oferece filtros avançados de multifatores, permitindo analisar origem, destino, protocolo e portas específicos:

Sintaxe básica do comando: Router(config)# access-list [100-199] {permit|deny} [Protocolo] [IP_Origem] [Curinga_Origem] [IP_Destino] [Curinga_Destino] [operador] [Porta]

Parâmetros de Controle:

  • Protocolos: ip, tcp, udp, icmp, ospf, etc..
  • Operadores de Porta: eq (igual), neq (diferente), gt (maior), lt (menor), range.
  • established: Permite a passagem de pacotes apenas se a conexão TCP já estiver estabelecida.

Exemplos de Configuração Prática

! 1. Bloquear acesso HTTP (porta 80) do Host para o Servidor Router(config)# access-list 101 deny tcp host 192.168.1.10 host 192.168.2.50 eq 80 ! 2. Permitir acesso HTTPS (porta 443) da sub-rede para qualquer destino Router(config)# access-list 101 permit tcp 192.168.1.0 0.0.0.255 any eq 443 ! 3. Bloquear tráfego de PING (ICMP echo) de qualquer origem/destino Router(config)# access-list 101 deny icmp any any echo ! 4. Permitir todo o restante do tráfego IP Router(config)# access-list 101 permit ip any any ! 5. Associar a regra à interface (Entrada/Inbound) Router(config)# interface GigabitEthernet0/1 Router(config-if)# ip access-group 101 in
Nota de Segurança: Ao final da ACL, o comando oculto deny ip any any bloquearia todo tráfego IP se o comando permit ip any any não estivesse configurado.
