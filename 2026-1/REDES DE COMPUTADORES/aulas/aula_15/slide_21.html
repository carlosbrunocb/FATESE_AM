Configuração Lógica: Cisco IOS CLI Comandos Práticos

Passo a Passo de Configuração

Para aplicar as políticas de segurança no roteador central Cisco, devemos seguir três etapas principais no modo de configuração global:

  • Etapa 1 (Definição do Bloqueio): Escrevemos a linha da ACL estendida que nega o acesso HTTP do PC alvo para a porta 80 do Servidor Web.
  • Etapa 2 (Permissão Geral): Inserimos a regra que permite todo o restante do tráfego IP de qualquer origem para qualquer destino.
  • Etapa 3 (Associação Física): Acessamos o modo de configuração da interface Gig0/0 (LAN de Origem) e aplicamos a ACL no fluxo de entrada (inbound).

Entradas de Comando no Cisco Terminal

Router> enable Router# configure terminal Router(config)# Router(config)# ! Passo 1: Nega HTTP do host para o servidor Router(config)# access-list 101 deny tcp host 192.168.1.10 host 192.168.2.50 eq 80 Router(config)# Router(config)# ! Passo 2: Permite todo o restante do tráfego Router(config)# access-list 101 permit ip any any Router(config)# Router(config)# ! Passo 3: Entrar na interface mais próxima da origem Router(config)# interface GigabitEthernet0/0 Router(config-if)# ip access-group 101 in Router(config-if)# end Router# write memory ! Salva as configurações
Validação Rápida da ACL: Para visualizar se as regras foram cadastradas e verificar as contagens de hits dos pacotes de dados, execute o comando:
Router# show access-lists.
