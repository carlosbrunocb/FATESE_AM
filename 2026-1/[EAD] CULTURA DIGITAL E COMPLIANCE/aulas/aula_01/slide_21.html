Pergunta-guia: “qual ferramenta vira critério de aceite no PR?”

Coluna A

Categorias

O que controlar

Qualidade & estilo

Linters, formatters e padrões de código para consistência e manutenibilidade.

Segurança (AppSec)

SAST/DAST/IAST, SCA (dependências) e scanning de segredos no repositório.

Governança & operações

Vault/KMS, DLP, SIEM, RBAC/MFA e trilhas de auditoria em ambientes.

Políticas como código

Regras versionadas (ex.: OPA) + logs de auditoria para evidências contínuas.

Heurística: se não gera evidência (log/relatório), vira “controle invisível”.

Coluna B

Exemplos

Como implementar

Git hooks + CI

Pre-commit, PR checks e quality gates para bloquear problemas antes do merge.

OWASP ASVS

Checklist de requisitos de segurança para alinhar controles ao nível de risco.

OPA (Policy-as-Code)

Regras versionadas para validar infraestrutura e deploy (ex.: permissões, redes).

HashiCorp Vault

Gestão de segredos (API keys, tokens) com rotação e controle de acesso.

OpenTelemetry

Observabilidade (logs/métricas/traces) para auditoria técnica e resposta a incidentes.

Dica prática: trate achados como bugs no backlog e registre evidências no repositório.

Takeaway para o Dev

Compliance fica “operacional” quando vira automação + evidência dentro do pipeline.
