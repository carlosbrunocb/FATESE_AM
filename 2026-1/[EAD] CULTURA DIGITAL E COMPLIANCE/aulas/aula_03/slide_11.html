WCAG 2.2 - 4 Princípios POUR

Perceptível, Operável, Compreensível e Robusto

Perceptível (Perceivable)
Alternativas de mídia e contraste
Definição

Informações e componentes da interface devem ser apresentados de forma que os usuários possam percebê-los.

Exemplo: Fornecer textos alternativos para imagens, legendas para vídeos, e contraste mínimo de 4.5:1 para texto normal.

Checklist
  • Alternativas de texto para imagens (alt text)
  • Legendas e transcrições para áudio/vídeo
  • Contraste adequado (4.5:1 para texto)
  • Não usar apenas cor para transmitir informação
Operável (Operable)
Navegação por teclado e foco
Definição

Componentes da interface devem ser operáveis por todos os usuários, incluindo aqueles que não usam mouse.

Exemplo: Todos os elementos interativos acessíveis por teclado, com foco visível e navegação lógica.

Checklist
  • Tudo acessível por teclado (Tab, Enter, Espaço)
  • Foco visível nos elementos interativos
  • Evitar conteúdo que pisque (pode causar crises)
  • Oferecer controle de tempo
Compreensível (Understandable)
Linguagem clara e consistente
Definição

Informações e operação da interface devem ser compreensíveis para todos os usuários.

Exemplo: Linguagem clara, rótulos explícitos em formulários, mensagens de erro com sugestões de correção.

Checklist
  • Linguagem clara e consistente
  • Rótulos e instruções explícitos
  • Mensagens de erro com sugestão de correção
  • Previsibilidade de navegação
Robusto (Robust)
HTML semântico e ARIA
Definição

Conteúdo deve ser robusto o suficiente para ser interpretado por diversas tecnologias assistivas.

Exemplo: HTML semântico correto, uso adequado de ARIA, compatibilidade com leitores de tela.

Checklist
  • HTML semântico (header, nav, main, footer)
  • ARIA usado corretamente
  • Compatibilidade com tecnologias assistivas
  • Testes cross-browser/dispositivos
