Cycle Time - Conceito e Cálculo

Medindo o tempo efetivo de trabalho no fluxo de desenvolvimento

Definição de Cycle Time

O que é Cycle Time?
Tempo efetivo de trabalho desde o início da execução até a conclusão (In Progress → Done). Diferente do Lead Time, o Cycle Time mede apenas o período de trabalho ativo, excluindo esperas e filas.
Fórmula de Cálculo
Cycle Time = Data de Conclusão − Data de Início (In Progress)
Exemplo: Início em 05/05 e conclusão em 08/05 = 3 dias
Lead Time
Tempo total desde solicitação até entrega (inclui esperas)
Cycle Time
Tempo de trabalho ativo (exclui esperas)
Quando usar Lead Time
Para promessas de entrega ao cliente
Quando usar Cycle Time
Para medir eficiência interna

Como medir no GitHub Projects

1
Capturar Início
Registrar quando o card entra em In Progress
2
Automação
Status muda para In Progress → preencher "Iniciado em"
3
Registrar Conclusão
Quando Done, preencher "Concluído em"
4
Calcular
Cycle Time = Concluído em − Iniciado em
Dica: Use campos customizados no GitHub Projects para registrar datas de início e fim, depois exporte para calcular automaticamente.
