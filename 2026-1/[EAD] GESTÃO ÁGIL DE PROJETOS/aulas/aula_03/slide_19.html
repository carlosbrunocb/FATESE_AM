Prática Guiada no GitHub Projects

Simulação de impedimentos e gestão visual de bloqueios

Objetivos da Demonstração

Simular um impedimento real
Criar uma situação de bloqueio técnico e sinalizar no Kanban
Aplicar label de bloqueio
Criar e aplicar label vermelha "Impedimento/Blocked" na Issue
Documentar o problema
Comentar na Issue descrevendo o bloqueio e mencionando o Scrum Master
Criar Issue de melhoria
Gerar Issue de "Melhoria de Processo" saída da retrospectiva

Passos da Demonstração

1
Abrir GitHub Projects do Projeto Integrador
2
Mover User Story para "In Progress"
3
Criar label "Impedimento" e aplicar na Issue
4
Comentar problema e marcar Scrum Master
5
Criar Issue de melhoria de processo

Ferramentas Utilizadas

GitHub Projects
Board Kanban
GitHub Issues
Gestão de tarefas
Labels
Categorização
Comentários
Comunicação
Resultado Esperado
Board com item bloqueado visível, comentário detalhado registrado e Issue de melhoria criada e priorizada.
