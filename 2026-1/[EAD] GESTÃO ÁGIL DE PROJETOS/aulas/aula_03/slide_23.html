Parte 1 - Simulando um Impedimento (Gargalo)

Passo a passo completo para simular e documentar um impedimento no GitHub Projects

1
Abrir GitHub Projects e Mover User Story

Abra o GitHub Projects do Projeto Integrador e mova pelo menos uma User Story para a coluna In Progress.

Abrir Projeto
Localizar Story
Mover para In Progress
2
Simular Problema Técnico

Imaginem que o Desenvolvedor travou nesta tarefa. Exemplos de problemas:

Erro no banco de dados
API fora do ar
Falta de acesso
Dependência externa
3
Criar Label de Impedimento

Crie uma Label de cor vermelha chamada Impedimento ou Blocked

// Criar label no GitHub
Settings > Labels > New label
Name: "Impedimento"
Color: #D73A4A (vermelho)
Description: "Tarefa bloqueada por impedimento"

Checklist de Ações

✅ Mover Story In Progress
✅ Criar Label Impedimento
✅ Aplicar Label Na Issue
✅ Comentar Detalhar problema
✅ Marcar SM @nome

Papel do Scrum Master

Identificar Impedimentos
Remover Bloqueios
Facilitar Resolução
Importante: Não iniciar novo trabalho até desbloquear a tarefa atual.
